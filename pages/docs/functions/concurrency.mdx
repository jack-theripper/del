# Управление параллелизмом

Ограничьте количество одновременно выполняемых шагов для вашей функции с помощью параметров конфигурации [`concurrency`](/docs/reference/functions/create#configuration). Установка необязательного параметра `key` ограничивает параллелизм для каждого уникального значения выражения.

<Callout>
**Важно:** Ограничения параллелизма распространяются на выполнение _шагов_, а не на общее количество запусков функции. Функции, находящиеся в состоянии ожидания (sleep), ожидающие событий или приостановленные между шагами, **не** учитываются в лимите параллелизма. Это означает, что одновременно может выполняться значительно больше запусков функции, чем позволяет ваш лимит параллелизма. [Подробнее о том, как работает параллелизм →](/docs/guides/concurrency#how-concurrency-works)
</Callout>

[Прочитайте руководство по параллелизму для получения дополнительной информации, включая принцип работы и ограничения](/docs/guides/concurrency).

<CodeGroup>
```ts {{ title: "Simple" }}
export default inngest.createFunction(
  {
    id: "sync-contacts",
    concurrency: {
      limit: 10,
    },
  }
  // ...
);
```
```ts {{ title: "Multiple keys" }}
inngest.createFunction(
  {
    id: "unique-function-id",
    concurrency: [
      {
         // Используем лимит параллелизма на уровне аккаунта для этой функции,
         // используя ключ "openai" как виртуальную очередь. Любая другая функция,
         // выполняемая с тем же ключом "openai", учитывается в этом лимите.
         scope: "account",
         key: `"openai"`,
         // Если 10 шагов выполняются с ключом "openai", запуски этой функции
         // будут ожидать освобождения ресурсов перед выполнением.
         limit: 10,
      },
      {
         // Создаём ещё одну виртуальную очередь параллелизма только для этой функции.
         // Это ограничивает все аккаунты одним одновременно выполняемым шагом
         // для этой функции на основе поля event.data.account_id.
         // "fn" — это область по умолчанию, поэтому это поле можно опустить.
         scope: "fn",
         key: "event.data.account_id",
         limit: 1,
      },
    ],
    triggers: { event: "ai/summary.requested" },
  },
  async ({ event, step }) => {
  }
);
```
</CodeGroup>

Установка ограничений `concurrency` особенно полезна для:

* Соблюдения лимитов частоты запросов к API — ограничьте параллелизм, чтобы не превышать квоты, установленные сторонними API.
* Ограничения операций с базой данных или количества подключений
* Предотвращения чрезмерного потребления ресурсов одним аккаунтом пользователя (см. параметр `key`)

Если вы хотите ограничить количество запусков функции за определённый период, возможно, больше подойдёт [параметр `rateLimit`](/docs/reference/functions/rate-limit).


## Конфигурация

<Properties>
  <Property name="concurrency" type="number | object | [object, object]">
    Параметры настройки параллелизма. Указание числа (`number`) — это сокращённая запись для установки свойства `limit`.

    <Properties nested={true}>
      <Property name="limit" type="number" required>
        Максимальное количество одновременно выполняемых шагов.
        Значение `0` или `undefined` равнозначно отсутствию ограничения.
        Максимальное значение определяется тарифным планом вашего аккаунта.
      </Property>
      <Property name="scope" type="'account' | 'env' | 'fn'">
        Область действия ограничения параллелизма, определяющая, управляется ли параллелизм для отдельной функции, в рамках среды или для всего аккаунта.

        * `fn` (по умолчанию): только запуски данной функции влияют на лимит параллелизма
        * `env`: все запуски в одной среде с одинаковым вычисленным значением ключа влияют на лимит параллелизма. Требует установки `key`, который вычисляется как имя виртуальной очереди.
        * `account`: каждый запуск с одинаковым вычисленным значением ключа влияет на лимит параллелизма во всех средах. Требует установки `key`, который вычисляется как имя виртуальной очереди.
      </Property>
      <Property name="key" type="string">
        Выражение, которое вычисляется в строку на основе запускающего события. Строка, возвращаемая выражением, используется как имя очереди параллелизма. Ключ обязателен при установке области `env` или `account`.

        Выражения определяются с использованием Common Expression Language (CEL), оригинальное событие доступно через точечную нотацию. Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions) для получения дополнительной информации. Примеры:

        * Ограничение параллелизма до `n` (через `limit`) для каждого ID клиента: `'event.data.customer_id'`
        * Ограничение до `n` для каждого пользователя на каждый ID импорта: `'event.data.user_id + "-" + event.data.import_id'`
        * Глобальное ограничение с использованием конкретной строки: `'"global-quoted-key"'` (обёрнуто в кавычки, так как выражение вычисляется как язык)
      </Property>
    </Properties>
  </Property>
</Properties>

<Callout>
**Параллелизм на уровне шагов:** Параметр concurrency управляет количеством одновременно выполняемых _шагов_, а не количеством запусков функции.

Поскольку функции часто приостанавливаются (во время ожидания, ожидания событий или между шагами), нередко одновременно выполняется значительно больше запусков, чем допускает лимит параллелизма. Однако в любой момент времени _активно выполняется_ не более заданного числа шагов.

Например, при `concurrency: 10` может быть 500 активных запусков функции, но только 10 шагов выполняют код в каждый конкретный момент.
</Callout>
