# Ссылки на функции

Используя [`step.invoke()`](/docs/reference/functions/step-invoke), вы можете напрямую вызвать одну функцию Inngest из другой и обработать результат. Вы можете использовать это вместе с `referenceFunction`, чтобы вызывать функции Inngest, расположенные в других приложениях, или чтобы избежать импорта зависимостей функций в рамках одного приложения.

```ts
// @/inngest/compute.ts
import { referenceFunction } from "inngest";
import { z } from "zod";
import { type computePi } from "@/inngest/computePi";

// Создаём локальную ссылку на функцию без импорта её зависимостей
export const computePi = referenceFunction<typeof computePi>({
  functionId: "compute-pi",
});

// Создаём ссылку на функцию в другом приложении
export const computeSquare = referenceFunction({
  appId: "my-python-app",
  functionId: "compute-square",
  // Схемы необязательны, но при указании задают типизацию вызова
  schemas: {
    data: z.object({
      number: z.number(),
    }),
    return: z.object({
      result: z.number(),
    }),
  },
});
```

```ts
// @/inngest/someFn.ts
import { computeSquare } from "@/inngest/compute"; // импортируем ссылку

// square.result имеет тип number
const square = await step.invoke("compute-square-value", {
  function: computeSquare,
  data: { number: 4 }, // входные данные типизированы и обязательны при необходимости
});
```

## Как использовать `referenceFunction`

Простейшая ссылка содержит только `functionId`. При использовании она вызовет функцию с указанным ID в том же приложении, из которого производится вызов.

Типы входных и выходных данных имеют значение `unknown`.

```ts
import { referenceFunction } from "inngest";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
  }),
});
```

При ссылке на функцию в другом приложении также укажите `appId`:

```ts
import { referenceFunction } from "inngest";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
    appId: "some-app",
  }),
});
```

Дополнительно можно указать `schemas` — набор [стандартных схем (Standard Schemas)](https://standardschema.dev/), обеспечивающих типизацию входных и выходных данных вызываемой функции.

<Callout>
В будущем это также будет _валидировать_ входные и выходные данные.
</Callout>

```ts
import { referenceFunction } from "inngest";
import { z } from "zod";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
    appId: "some-app",
    schemas: {
      data: z.object({
        foo: z.string(),
      }),
      return: z.object({
        success: z.boolean(),
      }),
    },
  }),
});
```

Даже если функции находятся в одном приложении, это также можно использовать для избежания импорта зависимостей одной функции в другую. Это особенно удобно для фреймворков вроде Next.js, где логика edge и serverless может находиться в одном месте, но требовать разных зависимостей.

```ts
import { referenceFunction } from "inngest";
import { type someInngestFn } from "@/inngest/someFn"; // импортируем только тип

await step.invoke("start-process", {
  function: referenceFunction<typeof someInngestFn>({
    functionId: "some-fn",
  }),
});
```

## Конфигурация

<Properties>
     <Property name="functionId" type="string" required>
          ID функции, на которую создаётся ссылка. Это может быть как локальный ID функции, так и ID функции из другого приложения.

          В последнем случае также необходимо указать `appId`. Если `appId` не указан, предполагается, что ID функции является локальным (будет использован ID приложения вызывающей стороны).
     </Property>
     <Property name="appId" type="string">
          ID приложения, которому принадлежит функция. Обязателен только в том случае, если указываемая функция находится в другом приложении.
     </Property>
     <Property name="schemas" type="object">
          Схемы указываемой функции, обеспечивающие типизацию входных данных `data` и возвращаемого значения `return` при вызове.

          Если не указаны и локальный тип функции не передаётся как обобщённый тип в `referenceFunction()`, схемы будут выведены как `unknown`.

          <Properties nested>
               <Property name="data" type="zod">
                    [Стандартная схема (Standard Schema)](https://standardschema.dev/) для типизации полезной нагрузки `data`, требуемой указываемой функцией.
               </Property>
               <Property name="return" type="zod">
                    [Стандартная схема (Standard Schema)](https://standardschema.dev/) для типизации возвращаемого значения указываемой функции при вызове.
               </Property>
          </Properties>
     </Property>
</Properties>
