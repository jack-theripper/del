# Фазы выпуска Inngest

На этой странице описано, как функции и продукты Inngest выпускаются через 3 отдельных фазы зрелости разработки и доступности для пользователей.

Процесс выпуска варьируется в зависимости от требований безопасности или масштабируемости новой функции или продукта:

Выпуски могут следовать нелинейному пути к GA — приведённая схема не является жёсткой, а лишь отражает то, как функции могут переходить от внутренней разработки к GA.
Кроме того (_не показано на схеме_), небольшие функции могут переходить от внутренней разработки сразу к GA при запуске.

Ниже вы найдёте подробное описание каждой фазы выпуска.


## Фазы выпуска


### Developer Preview (Предварительный просмотр для разработчиков)

Продукты или функции, выпущенные как Developer Preview, всё ещё находятся в разработке и предоставляются ранним пользователям для сбора обратной связи.
Это означает, что новые API или методы, доступные через наши API или SDK, помечены как экспериментальные и могут изменяться без строгого управления версиями или предварительного уведомления.

Краткая характеристика функций или продуктов в предварительном просмотре для разработчиков:

| Аспект | Подробности |
|--------|---------|
| API | **Предоставляет нефинальные API, которые могут изменяться без строгого управления версиями или уведомления** |
| Документация | Предоставляет ограниченную документацию, которая будет развиваться с учётом обратной связи пользователей |
| Использование в продакшене | **Не рекомендуется для использования в продакшене и не охватывается соглашением об уровне обслуживания (SLA) для корпоративного плана** |
| Ограничения | Могут иметь ограничения, специфичные для функции, например, ограничения на основе плана или использования |
| Цена | Ограниченная бесплатная доступность в фазе предварительного просмотра |

Функции или продукты, находящиеся в предварительном просмотре для разработчиков:

- [Realtime](/docs/features/realtime)
- [Connect](/docs/setup/connect)
- [AgentKit](https://agentkit.inngest.com/)



### Закрытая/открытая бета-версия

Продукты или функции в закрытой или открытой бета-версии приближаются к GA, но всё ещё требуют некоторой работы для соответствия желаемым стандартам качества (например, улучшение надёжности, нагрузочное тестирование масштабируемости). На этом этапе большинство продуктов или функций почти завершены и должны корректно обрабатывать рабочие нагрузки, но могут демонстрировать проблемы, о которых мы не знали во время раннего доступа.

Краткая характеристика функций или продуктов в бета-версии:

| Аспект | Подробности |
|--------|---------|
| API | Предоставляет почти финальные API, которые могут изменяться в соответствии со строгим управлением версиями и коммуникациями |
| Документация | Предоставляет расширенную документацию, включающую руководство по началу работы, справочные материалы и FAQ по граничным случаям |
| Использование в продакшене | **Не рекомендуется для использования в продакшене и не охватывается соглашением об уровне обслуживания (SLA) для корпоративного плана** |
| Поддержка | Предоставляет доступ к каналу для обратной связи (_в случае закрытой бета-версии_) |
| Ограничения | Могут иметь ограничения, специфичные для функции, например, ограничения на основе плана или использования |
| Цена | Ограниченная бесплатная доступность в фазе предварительного просмотра |

<Callout variant="info" skipSearchCrawler>
**Закрытая бета-версия vs. открытая бета-версия**

Функции или продукты, требующие нагрузочного тестирования масштабируемости, соображений безопасности или ранней обратной связи от клиентов, являются хорошими кандидатами для закрытой бета-версии, обеспечивая положительный опыт для всех участников.
Другие функции могут подходить для открытой бета-версии (например, функции SDK, улучшения Dev Server Inngest).

</Callout>

### Общая доступность (General Availability)

Продукт или функция в общей доступности считается безопасной, масштабируемой и стабильной, готовой для использования пользователями и клиентами на всех планах.

Краткая характеристика функций или продуктов в общей доступности:

| Аспект | Подробности |
|--------|---------|
| API | **Предоставляет финальные API, которые могут развиваться в соответствии со строгим управлением версиями и журналом изменений** |
| Документация | Предоставляет полную документацию: быстрые старты, примеры и демонстрации |
| Использование в продакшене | **Готова к использованию в продакшене пользователями и клиентами на всех планах и охватывается соглашением об уровне обслуживания (SLA) для корпоративного плана** |
| Ограничения | Ограничения должным образом задокументированы (_если есть_) |
| Цена | Цена переходит в права использования (entitlement) |

### Устаревание (Deprecated)

Устаревание функций или продуктов следует следующему плану на протяжении нескольких месяцев:

1. Добавление упоминаний об устаревании на соответствующих страницах документации
    1. Публикация руководств по миграции (при необходимости)
2. Уведомление пользователей и клиентов по электронной почте, в Discord и в приватных каналах поддержки Slack
3. Обновление SDK для выделения устаревших API (при необходимости)
4. Целевые напоминания по электронной почте за несколько недель до целевой даты завершения поддержки



## Часто задаваемые вопросы

### Как собирается обратная связь в фазе предварительного просмотра для разработчиков?

В фазах Developer Preview и открытой бета-версии мы собираем обратную связь от пользователей и клиентов через наш канал Discord.

Обратная связь в фазах GA и закрытой бета-версии собирается через наш приватный канал поддержки Slack, канал Discord и наш [портал поддержки](https://app.inngest.com/support).


### Как быть в курсе выпуска новых функций и продуктов?

Вы можете следить за нами в [Twitter](https://x.com/inngest) или [Discord](https://discord.gg/inngest), чтобы получать уведомления о выпуске новых функций и продуктов.

Вы также можете подписаться на нашу [рассылку](https://inngest.com/newsletter), чтобы получать уведомления о выпуске новых функций и продуктов.
