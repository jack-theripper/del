# Промежуточное ПО шифрования


Промежуточное ПО шифрования обеспечивает сквозное шифрование событий, вывода шагов и вывода функций. **На серверы Inngest отправляются только зашифрованные данные**: шифрование и расшифрование происходят внутри вашей инфраструктуры.

    ## Установка

    `EncryptionMiddleware` доступен как часть пакета `inngest_encryption`:

    ```py
    import inngest
    from inngest_encryption import EncryptionMiddleware

    inngest_client = inngest.Inngest(
        app_id="my-app",
        middleware=[EncryptionMiddleware.factory("my-secret-key")],
    )
    ```

    По умолчанию шифруются следующие данные:
    - Поле `encrypted` в `event.data`.
    - Возвращаемые значения `step.run`.
    - Возвращаемые значения функций.

    ## Установка

    Установите [пакет `@inngest/middleware-encryption`](https://www.npmjs.com/package/@inngest/middleware-encryption) ([GitHub](https://github.com/inngest/inngest-js/tree/main/packages/middleware-encryption#readme)) и настройте его следующим образом:

    ```ts
    import { encryptionMiddleware } from "@inngest/middleware-encryption";

    // Initialize the middleware
    const mw = encryptionMiddleware({
      // your encryption key string should not be hard coded
      key: process.env.MY_ENCRYPTION_KEY,
    });

    // Use the middleware with Inngest
    const inngest = new Inngest({
      id: "my-app",
      middleware: [mw],
    });
    ```

   По умолчанию шифруются следующие данные:
    - Все данные шагов
    - Весь вывод функций
    - Данные событий, помещённые внутрь `data.encrypted`

  </LanguageSection>

    ## Изменение зашифрованного поля `event.data`

    По умолчанию `event.data.encrypted` зашифровано. Все остальные поля передаются в открытом виде. Для шифрования другого поля укажите параметр `event_encryption_field`.
  </LanguageSection>

  <LanguageSection show="typescript">
    ## Изменение зашифрованного поля `event.data`

    Шифруются только отдельные части данных события. По умолчанию — только поле `data.encrypted`.

    Это можно настроить с помощью параметра `eventEncryptionField: string`.
  </LanguageSection>

  <LanguageSection show="typescript">
    ## Резервные ключи расшифрования

    Для попытки расшифрования с несколькими ключами установите параметр `fallbackDecryptionKeys`. Это полезно при ротации ключей, поскольку старые события могут быть зашифрованы другим ключом:

    ```ts
    // start out with the current key
    encryptionMiddleware({
      key: process.env.MY_ENCRYPTION_KEY,
    });

    // deploy all services with the new key as a decryption fallback
    encryptionMiddleware({
      key: process.env.MY_ENCRYPTION_KEY,
      fallbackDecryptionKeys: ["new"],
    });

    // deploy all services using the new key for encryption
    encryptionMiddleware({
      key: process.env.MY_ENCRYPTION_KEY_V2,
      fallbackDecryptionKeys: ["current"],
    });

    // once you are sure all data using the "current" key has passed, phase it out
    encryptionMiddleware({
      key: process.env.MY_ENCRYPTION_KEY_V2,
    });
    ```
