# Использование промежуточного ПО для внедрения зависимостей

Функции Inngest, работающие в одном приложении, часто используют общие экземпляры клиентов — например, клиенты баз данных или сторонних библиотек.

Ниже пример добавления клиента OpenAI ко всем функциям Inngest, что даёт им немедленный доступ без необходимости создавать клиент самостоятельно.

<LanguageSelector>
    <LanguageSection show="typescript-middleware">
        Можно использовать `dependencyInjectionMiddleware` для добавления аргументов во входные данные функции.

        Смотрите [пример TypeScript](?guide=typescript) для кастомного промежуточного ПО.

        ```ts
        import { dependencyInjectionMiddleware } from "inngest";
        import OpenAI from 'openai';

        const openai = new OpenAI();

        const inngest = new Inngest({
          id: 'my-app',
          middleware: [
            dependencyInjectionMiddleware({ openai }),
          ],
        });
        ```

        Теперь функции Inngest могут получать доступ к клиенту OpenAI через контекст:

         ```ts
        inngest.createFunction(
          { name: "user-create", triggers: { event: "app/user.create" } },
          async ({ openai }) => {
            const chatCompletion = await openai.chat.completions.create({
              messages: [{ role: "user", content: "Say this is a test" }],
              model: "gpt-3.5-turbo",
            });

            // ...
          },
        );

        ```

        <Callout>
        💡 Типы выводятся из вывода промежуточного ПО, поэтому функции Inngest увидят типизированное свойство `openai` в своих входных данных.
        </Callout>

        Изучите другие примеры на [странице примеров промежуточного ПО TypeScript SDK](/docs/reference/typescript/v4/middleware/examples).

        ### Расширенная мутация

        Когда промежуточное ПО выполняется, типы и данные в переданном `ctx` накладываются поверх значений по умолчанию, предоставленных библиотекой. Это позволяет безопасно и точно перезаписывать данные и типы.

        Например, здесь используется утверждение `const` для вывода литерального значения `foo`:

        ```ts
        // In middleware
        dependencyInjectionMiddleware({
          foo: "bar",
        } as const)

        // In a function
        async ({ event, foo }) => {
          //             ^? (parameter) foo: "bar"
        }
        ```

        ## Порядок промежуточного ПО и типы

        Промежуточное ПО выполняется в порядке, указанном при регистрации (смотрите [жизненный цикл промежуточного ПО](/docs/reference/typescript/v4/middleware/lifecycle)), что также влияет на типизацию.

        При выводе мутированных входных или выходных данных SDK применяет изменения каждого промежуточного ПО последовательно, так же как это происходит во время выполнения. Это означает, что для двух промежуточных ПО, добавляющих значение `foo` к входным аргументам, последнее выполненное будет видно как в типах, так и во время выполнения.
    </LanguageSection>

    <LanguageSection show="typescript">
        Наш `OpenAIMiddleware` использует [хук `transformFunctionInput`](/docs/reference/typescript/v4/middleware/lifecycle#transformfunctioninput) для мутации контекста функции:

        ```ts
        import { Middleware } from "inngest";
        import OpenAI from "openai";

        const openai = new OpenAI();

        class OpenAIMiddleware extends Middleware.BaseMiddleware {
          transformFunctionInput(args: Middleware.TransformFunctionInputArgs) {
            return {
              ...args,
              ctx: {
                ...args.ctx,
                // Anything passed via `ctx` will be merged with the function's arguments
                openai,
              },
            };
          }
        }
        ```

        Теперь функции Inngest могут получать доступ к клиенту OpenAI через контекст:

         ```ts
        inngest.createFunction(
          { name: "user-create", triggers: { event: "app/user.create" } },
          async ({ openai }) => {
            const chatCompletion = await openai.chat.completions.create({
              messages: [{ role: "user", content: "Say this is a test" }],
              model: "gpt-3.5-turbo",
            });

            // ...
          },
        );

        ```

        <Callout>
        💡 Типы выводятся из вывода промежуточного ПО, поэтому функции Inngest увидят типизированное свойство `openai` в своих входных данных.
        </Callout>

        Изучите другие примеры на [странице примеров промежуточного ПО TypeScript SDK](/docs/reference/typescript/v4/middleware/examples).

        ### Расширенная мутация

        Когда `transformFunctionInput()` возвращает новый `ctx`, типы и данные внутри этого `ctx` накладываются поверх значений по умолчанию, предоставленных библиотекой. Это позволяет безопасно и точно перезаписывать данные и типы.

        Например, здесь используется утверждение `const` для вывода литерального значения `foo`:

        ```ts
        // In middleware
        transformFunctionInput(args: Middleware.TransformFunctionInputArgs) {
          return {
            ...args,
            ctx: {
              ...args.ctx,
              foo: "bar",
            } as const,
          };
        }

        // In a function
        async ({ event, foo }) => {
          //             ^? (parameter) foo: "bar"
        }
        ```

        Поскольку возвращаемый объект `ctx` и значения по умолчанию объединяются, иногда хорошо выведенные типы перезаписываются более общими типами из промежуточного ПО. Частый пример — обработка данных события в промежуточном ПО.

        Чтобы обойти это, можно предоставить данные, но опустить тип, используя утверждение типа `as`. Например, здесь используется утверждение для добавления `foo` и изменения данных события без влияния на тип:

        ```ts
        transformFunctionInput(args: Middleware.TransformFunctionInputArgs) {
          const event = await decrypt(args.ctx.event);

          const newCtx = {
            ...args.ctx,
            foo: "bar",
            event,
          };

          return {
            ...args,
            // Don't affect the `event` type
            ctx: newCtx as Omit<typeof newCtx, "event">,
          };
        }
        ```

        При выводе мутированных входных или выходных данных SDK применяет изменения каждого промежуточного ПО последовательно, так же как это происходит во время выполнения. Это означает, что для двух промежуточных ПО, добавляющих значение `foo` к входным аргументам, последнее выполненное будет видно как в типах, так и во время выполнения.

    <LanguageSection show="python">
        Наш `OpenAIMiddleware` использует [хук `transform_input`](/docs/reference/python/middleware/lifecycle#transform-input) для внедрения контекста:

        ```py
        import inngest
        from openai import OpenAI

        class OpenAIMiddleware(inngest.Middleware):
            def __init__(
                self,
                client: inngest.Inngest,
                raw_request: object,
            ) -> None:
                self.openai = OpenAI(
                    # This is the default and can be omitted
                    api_key=os.environ.get("OPENAI_API_KEY"),
                )

            def transform_input(
                self,
                ctx: execution_lib.Context,
                function: function.Function,
                steps: step_lib.StepMemos,
            ) -> None:
                ctx.openai = self.openai  # type: ignore


        inngest_client = inngest.Inngest(
            app_id="my_app",
            middleware=[OpenAIMiddleware],
        )
        ```

        Теперь функции Inngest могут получать доступ к клиенту `openai` через контекст:

        ```py
            @inngest_client.create_function(
                fn_id="user-create",
                trigger=inngest.TriggerEvent(event="app/user.create"),
            )
            async def fn(ctx: inngest.Context):
                chat_completion = ctx.openai.chat.completions.create(
                    messages=[
                        {
                            "role": "user",
                            "content": "Say this is a test",
                        }
                    ],
                    model="gpt-3.5-turbo",
                )
        ```
