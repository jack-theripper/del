# Fetch: выполнение API-запросов и получение данных <VersionBadge version="TypeScript only" />

TypeScript SDK Inngest предоставляет API `step.fetch()` и утилиту `fetch()`, позволяющие выполнять запросы к сторонним API или получать данные устойчивым способом, передавая их на выполнение платформе Inngest:

- `step.fetch()` — это сокращённая запись для выполнения HTTP-запросов внутри функции Inngest, упрощающая запуск параллельных HTTP-запросов.
- Утилита `fetch()` может передаваться пакетам, принимающим пользовательскую реализацию `fetch`, например `axios`.

![Использование Fetch передаёт HTTP-запрос на платформу Inngest](/assets/docs/features/inngest-functions/steps-workflows/fetch/step-fetch.png)


## Использование `step.fetch()`

`step.fetch()` можно использовать для выполнения HTTP-запросов внутри функции Inngest.

`step.fetch()` передаёт HTTP-запрос на платформу Inngest, поэтому ваш сервис не должен быть активным и ожидать ответа.

```ts {{ title: "src/inngest/functions.ts" }}
import { inngest } from "./client";

export const retrieveTextFile = inngest.createFunction(
  { id: "retrieveTextFile", triggers: { event: "textFile/retrieve" } },
  async ({ step }) => {
    // The fetching of the text file is offloaded to the Inngest Platform
    const response = await step.fetch(
      "https://example-files.online-convert.com/document/txt/example.txt"
    );

    // The Inngest function run is resumed when the HTTP request is complete
    await step.run("extract-text", async () => {
      const text = await response.text();
      const exampleOccurences = text.match(/example/g);
      return exampleOccurences?.length;
    });
  }
);
```

`step.fetch()` полезен:
- В бессерверных средах для передачи долгих HTTP-запросов, которые могут вызвать таймаут.
- Как сокращённая запись для HTTP-запросов внутри функции Inngest, упрощающая запуск параллельных запросов через `Promise.all()`.
- Как лучшая практика для обеспечения устойчивости всех HTTP-запросов с возможностью их проверки на платформе Inngest или в Dev Server.

## Использование утилиты `fetch()`

Экспортируется совместимая с Fetch API функция, позволяющая сделать любые HTTP-запросы устойчивыми, если они вызываются внутри функции Inngest.

Например, класс `MyProductApi`, основанный на axios, может принимать параметр `fetch`:


```ts {{ title: "TypeScript" }}
import { fetch } from "inngest";

const api = new MyProductApi({ fetch });

// A call outside an Inngest function will fall back to the global fetch
await api.getProduct(1);

// A call from inside an Inngest function will be made durable and offloaded to the Inngest Platform
inngest.createFunction(
  { id: "my-fn", triggers: { event: "product/activated" } },
  async () => {
    await api.getProduct(1);
  },
);
```

<Callout variant="warning">
  ⚠️ `fetch()` и `step.run()`

  Вызовы `fetch()` от Inngest не следует выполнять внутри блоков `step.run()`.
  Иначе `fetch()` вернётся к глобальной реализации `fetch`.

  Почему? Утилита `fetch()` преобразует вызовы `fetch` в вызовы `step.run()`, [которые не могут быть вложенными](/docs/sdk/eslint#inngest-no-nested-steps).
</Callout>

### Внутри шагов

По умолчанию использование `fetch` от Inngest сохраняет всю функциональность запросов, выполненных вне эндпоинта, но гарантирует, что запросы внутри него будут устойчивыми.

<CodeGroup>
```ts {{ title: "TypeScript" }}
import { fetch as inngestFetch } from 'inngest';
import { generateText } from 'ai';
import { createAnthropic } from '@ai-sdk/anthropic';

// The AI SDK's createAnthropic objects can be passed a custom fetch implementation
const anthropic = createAnthropic({
  fetch: inngestFetch,
});

// NOTE - Using this fetch outside of an Inngest function will fall back to the global fetch
const response = await generateText({
  model: anthropic('claude-3-5-sonnet-20240620'),
  prompt: 'Hello, world!',
});

// A call from inside an Inngest function will be made durable
inngest.createFunction(
  { id: "generate-summary", triggers: { event: "post.created" } },
  async ({ event }) => {
    // This will use step.fetch automatically!
    const response = await generateText({
      model: anthropic('claude-3-5-sonnet-20240620'),
      prompt: `Summarize the following post: ${event.data.content}`,
    });
  },
);
```
</CodeGroup>

### Использование с AI SDK: отключение повторов AI SDK

<Callout variant="warning">
  **Важно:** При использовании `fetch` от Inngest с AI SDK отключите встроенный механизм повторов AI SDK и позвольте Inngest управлять повторами.
</Callout>

В AI SDK есть встроенная логика повторов, которая может конфликтовать с механизмом повторов Inngest, особенно при работе с долго выполняющимися моделями или бессерверными платформами с ограничениями по таймауту (например, 15-минутный максимальный таймаут Vercel).

**Почему это важно:**
- Когда повторы AI SDK включены одновременно с механизмом повторов Inngest, общая продолжительность повторов может превысить ограничения платформы по таймауту
- Для долго выполняющихся моделей (например, OpenAI's o3) это особенно проблематично
- При возникновении таймаута сообщения об ошибках могут вводить в заблуждение: Vercel отменяет запрос, но это отображается в Inngest как «Внутренняя ошибка сервера» (от Inngest, не Vercel), что затрудняет отладку

**Рекомендуемая конфигурация:**

<CodeGroup>
```ts {{ title: "generateText() with maxRetries: 0" }}
import { generateText } from 'ai';
import { createOpenAI } from '@ai-sdk/openai';
import { fetch as inngestFetch } from 'inngest';

const openai = createOpenAI({
  fetch: inngestFetch,
});

inngest.createFunction(
  { id: "generate-with-o3", triggers: { event: "content/generate" } },
  async ({ event }) => {
    // Disable AI SDK retries - let Inngest handle them instead
    const response = await generateText({
      model: openai('o3'),
      prompt: event.data.prompt,
      maxRetries: 0, // Critical: Set to 0 to let Inngest handle retries
    });

    return response;
  },
);
```

```ts {{ title: "streamText() with maxRetries: 0" }}
import { streamText } from 'ai';
import { createAnthropic } from '@ai-sdk/anthropic';
import { fetch as inngestFetch } from 'inngest';

const anthropic = createAnthropic({
  fetch: inngestFetch,
});

inngest.createFunction(
  { id: "stream-completion", triggers: { event: "completion/stream" } },
  async ({ event, step }) => {
    // Disable AI SDK retries for streaming operations
    const result = await streamText({
      model: anthropic('claude-3-5-sonnet-20240620'),
      prompt: event.data.prompt,
      maxRetries: 0, // Let Inngest handle retries
    });

    return result;
  },
);
```
</CodeGroup>

Устанавливая `maxRetries: 0` в вызовах AI SDK, вы:
- Избегаете проблем с таймаутом на бессерверных платформах
- Получаете более чёткие сообщения об ошибках при сбоях
- Используете механизм повторов Inngest, разработанный для долго выполняющихся операций
- Получаете преимущества наблюдаемости Inngest для отслеживания того, что произошло при каждой попытке

Тот же `fetch` также экспортируется как `step.fetch`, позволяя создавать API, изолированные внутри функции:

<CodeGroup>
```ts {{ title: "TypeScript" }}
import { createAnthropic } from '@ai-sdk/anthropic';
import { fetch as inngestFetch } from 'inngest';
import { generateText } from 'ai';

const anthropic = createAnthropic({
  fetch: inngestFetch,
});

inngest.createFunction(
  { id: "generate-summary", triggers: { event: "post.created" } },
  async ({ step }) => {
    const response = await generateText({
      model: anthropic('claude-3-5-sonnet-20240620'),
      prompt: `Summarize the following post: ${event.data.content}`,
    });
  },
);
```
</CodeGroup>

### Резервные варианты

По умолчанию корректно откатывается на глобальный `fetch`, если вызывается вне функции Inngest, хотя можно также установить пользовательский резервный вариант с помощью метода `config`:

<CodeGroup>
```ts {{ title: "TypeScript" }}
import { fetch } from "inngest";

const api = new MyProductApi({
  fetch: fetch.config({ fallback: myCustomFetch }),
});
```
</CodeGroup>

Можно также полностью отключить резервный вариант:

<CodeGroup>
```ts {{ title: "TypeScript" }}
import { fetch } from "inngest";

const api = new MyProductApi({
  fetch: fetch.config({ fallback: undefined }),
});
```
</CodeGroup>

### Как это работает

Функция `fetch` от Inngest использует базовые строительные блоки Inngest для бесшовного создания опционально устойчивого кода. При вызове она:

- Проверяет контекст, в котором выполняется
- Если не в функции Inngest — опционально использует резервный вариант; иначе:
- Сообщает о запросе в Inngest
- Inngest выполняет запрос
- Inngest продолжает функцию с полученным `Response`

Критически важно: ваш сервис не должен быть активным в течение всего вызова — функция продолжится, когда будет получен результат, при этом оставаясь устойчивой!
