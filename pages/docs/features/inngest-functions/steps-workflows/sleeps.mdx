# Сны (Sleeps)

Два метода шага, `step.sleep` и `step.sleepUntil`, позволяют приостановить выполнение функции на определённое время. Функция может «спать» секунды, минуты или дни — максимум один год.

Использование методов сна позволяет избежать необходимости запускать несколько cron-задач или использовать дополнительные очереди. Например, сны позволяют создать рабочий процесс онбординга пользователей, последовательно выполняющий действия во времени: сначала отправить приветственное письмо, затем каждый день в течение недели отправлять обучающие материалы.

## Как работают сны

`step.sleep` и `step.sleepUntil` указывают Inngest возобновить выполнение функции в будущем. Код не должен быть запущен в течение интервала сна, что позволяет использовать сны в любой среде, включая бессерверные платформы.

Функция, приостановленная спящим шагом, не влияет на ёмкость аккаунта — она не учитывается в лимите параллелизма плана. Спящая функция также не учитывается в [политике параллелизма](/docs/guides/concurrency), установленной для функции.

## Приостановка выполнения на заданное время


<LanguageSection show="typescript">

  Используйте `step.sleep()` для приостановки выполнения функции на определённый промежуток времени.


```ts
export default inngest.createFunction(
  { id: "send-delayed-email", triggers: { event: "app/user.signup" } },
  async ({ event, step }) => {
    await step.sleep("wait-a-couple-of-days", "2d");
    // Do something else
  }
);
```


Смотрите [справочник TypeScript для `step.sleep()`](/docs/reference/functions/step-sleep).

</LanguageSection>

<LanguageSection show="python">

  Используйте `step.sleep()` для приостановки выполнения функции на определённый промежуток времени.

  ```py
  @inngest_client.create_function(
      fn_id="my_function",
      trigger=inngest.TriggerEvent(event="app/my_function"),
  )
  async def fn(ctx: inngest.Context) -> None:
      await ctx.step.sleep("zzz", datetime.timedelta(seconds=2))
  ```

Смотрите [справочник Python для `step.sleep()`](/docs/reference/python/steps/sleep).

</LanguageSection>

## Приостановка выполнения до заданной даты


<LanguageSection show="typescript">

Используйте `step.sleepUntil()` для приостановки выполнения функции до определённого момента времени.


```ts
export default inngest.createFunction(
  { id: "send-scheduled-reminder", triggers: { event: "app/reminder.scheduled" } },
  async ({ event, step }) => {
    const date = new Date(event.data.remind_at);
    await step.sleepUntil("wait-for-scheduled-reminder", date);
    // Do something else
  }
);
```


Смотрите [справочник TypeScript для `step.sleepUntil()`](/docs/reference/functions/step-sleep-until).

</LanguageSection>

<LanguageSection show="python">

  Используйте `step.sleep_until()` для приостановки выполнения функции до определённого момента времени.

```py
@inngest_client.create_function(
    fn_id="my_function",
    trigger=inngest.TriggerEvent(event="app/my_function"),
)
async def fn(ctx: inngest.Context) -> None:
    await ctx.step.sleep_until(
        "zzz",
        datetime.datetime.now() + datetime.timedelta(seconds=2),
    )
```
Смотрите [справочник Python для `step.sleep_until()`](/docs/reference/python/steps/sleep-until).

</LanguageSection>

<Callout variant={'info'}>
  **Сны и история трассировки/логов**

  Вы можете заметить, что в представлении «Запуски функций» в Inngest Cloud не отображаются запуски функций, использующие сны длиннее лимита истории трассировки и логов вашего [плана Inngest](/pricing?ref=docs-sleeps), хотя функции всё ещё спят и продолжат выполняться в запланированное время. **Это известное ограничение** текущей панели управления, которое мы работаем исправить.

  В настоящее время:

  - Будьте уверены, что спящие функции *действительно* спят и возобновятся по расписанию, даже если они не отображаются в списке «Запуски функций».
  - Зная ID запуска функции, можно проверить его статус с помощью функции быстрого поиска Inngest Cloud (Ctrl-K или ⌘K) или [REST API](https://api-docs.inngest.com/docs/inngest-api/).
</Callout>
