# Ожидание события

Можно приостановить выполнение функции до получения определённого события.

Это полезный паттерн для реакции на конкретные действия пользователей (например, реализации [«Человека в цепочке»](/docs/ai-patterns/human-in-the-loop) в агентных AI-рабочих процессах) или ожидания событий во внешних системах.

Это рекомендуемый и предпочтительный способ приостановки и возобновления функции через события или сигналы, потому что:

* События распространяются на многие функции: можно возобновить множество запусков из одного события без изменения кода приложения.
* События чище: события разделяют код и функции, которые они возобновляют.
* События имеют журналы аудита: события хранятся в вашем OLAP-хранилище, автоматически обеспечивая журналы аудита и аналитику.

<LanguageSection show="typescript">

  Используйте `step.waitForEvent()` для ожидания получения определённого события перед продолжением. Возвращает `Promise`, который разрешается с полученным событием или `null`, если событие не получено в течение таймаута.


```ts
export default inngest.createFunction(
  { id: "send-onboarding-nudge-email", triggers: { event: "app/account.created" } },
  async ({ event, step }) => {
    const onboardingCompleted = await step.waitForEvent(
      "wait-for-onboarding-completion",
      {
        event: "app/onboarding.completed",
        timeout: "3d",
        // "async" is the received "ai/post.topic.selected" event here:
        if: `async.data.completionId == "${generatedTopics.completionId}"`,
      }
    );
    if (!onboardingCompleted) {
      // if no event is received within 3 days, onboardingCompleted will be null
    } else {
      // if the event is received, onboardingCompleted will be the event payload object
    }
  }
);
```


Смотрите [справочник TypeScript для `step.waitForEvent()`](/docs/reference/functions/step-wait-for-event).

Для простой задержки выполнения во времени используйте [`step.sleep()`](/docs/reference/functions/step-sleep).


## Примеры

### Динамические функции, ожидающие дополнительных действий пользователя

Ниже пример функции Inngest, создающей drip email-кампанию в стиле Intercom или Customer.io, персонализированную на основе:


```ts
export default inngest.createFunction(
  { id: "onboarding-email-drip-campaign", triggers: { event: "app/account.created" } },
  async ({ event, step }) => {
    // Send the user the welcome email immediately
    await step.run("send-welcome-email", async () => {
      await sendEmail(event.user.email, "welcome");
    });

    // Wait up to 3 days for the user to complete the final onboarding step
    // If the event is received within these 3 days, onboardingCompleted will be the
    // event payload itself, if not it will be null
    const onboardingCompleted = await step.waitForEvent("wait-for-onboarding", {
      event: "app/onboarding.completed",
      timeout: "3d",
      // The "data.userId" must match in both the "app/account.created" and
      // the "app/onboarding.completed" events
      if: `async.data.userId == "${event.data.userId}"`,
    });

    // If the user has not completed onboarding within 3 days, send them a nudge email
    if (!onboardingCompleted) {
      await step.run("send-onboarding-nudge-email", async () => {
        await sendEmail(event.user.email, "onboarding_nudge");
      });
    } else {
      // If they have completed onboarding, send them a tips email
      await step.run("send-tips-email", async () => {
        await sendEmail(event.user.email, "new_user_tips");
      });
    }
  }
);
```

<Callout variant={'warning'}>
**Предотвращение гонок условий**

Метод «ожидание события» начинает прослушивать новые события с момента выполнения кода. Это означает, что события, отправленные до выполнения функции, не будут обработаны ожиданием.

Чтобы избежать гонки условий, всегда проверяйте поток событий в ваших функциях.

_Примечание: В скором времени механизм «ожидания события» получит функцию «обратного просмотра», включающую события из заданного прошедшего периода времени._
</Callout>
