# Ожидание сигнала

Можно приостановить выполнение функции до получения определённого сигнала. Это похоже на [ожидание события](/docs/features/inngest-functions/steps-workflows/wait-for-event), но возобновляет только один запуск — через API-вызов, а не через событие. Использование этого метода вместо [ожидания события](/docs/features/inngest-functions/steps-workflows/wait-for-event) рекомендуется только в специфических сценариях.
В частности, `waitForSignal` имеет немного меньшую задержку, чем `waitForEvent`, поскольку API возобновления транзакционный, а не в конечном итоге согласованный через события.

В большинстве случаев следует использовать [ожидание события](/docs/features/inngest-functions/steps-workflows/wait-for-event).

Это полезный паттерн для реакции на конкретные действия пользователей (например, реализации «Человека в цепочке» в агентных AI-рабочих процессах) или ожидания событий во внешних системах.


<LanguageSelector>



<LanguageSection show="typescript">

  Используйте `step.waitForSignal()` для ожидания получения определённого сигнала перед продолжением. Возвращает `Promise`, который разрешается с данными сигнала или `null`, если событие не получено в течение таймаута.


```ts
export default inngest.createFunction(
  { id: "send-onboarding-nudge-email", triggers: { event: "app/account.created" } },
  async ({ event, step }) => {
    const approval = await step.waitForSignal(
      "wait-for-approval",
      {
        signal: "task/71651db4-9f27-466a-a6be-4759b9784b3c",
        timeout: "3d",
      }
    );
    if (!approval) {
      // if no event is received within 3 days, onboardingCompleted will be null
    } else {
      // if the event is received, onboardingCompleted will be the event payload object
    }
  }
);
```

Можно возобновлять функции через `step.sendSignal` в функциях или через клиент Inngest:

```ts

inngest.sendSignal({
  signal: "task/71651db4-9f27-466a-a6be-4759b9784b3c",
  data: "whatever-you-want", // this will be injected into the waitForSignal response, and can be any JSON-serializable data.
});

```

Смотрите [справочник TypeScript для `step.waitForSignal()`](/docs/reference/functions/step-wait-for-signal).

</LanguageSection>

  ```go
    })

    if err == step.ErrSignalNotReceived {
    }
		return nil, nil
	}

    // ...

    return nil, nil
  }
  ```
