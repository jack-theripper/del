# Отмена по таймауту

Можно принудительно отменять запуски, если они слишком долго ожидают начала выполнения или выполняются слишком долго. Параметр конфигурации `timeouts` позволяет автоматически отменять функции на основе двух свойств таймаута:

- `timeouts.start` — определяет, как долго функция может находиться в очереди до начала выполнения
- `timeouts.finish` — определяет, как долго функция может выполняться после запуска


В следующих примерах рассмотрим, как настроить свойство `timeouts` и как это работает.

<LanguageSelector>


## `timeouts.start` — Таймаут для запусков в очереди (до начала выполнения)

Запуски могут находиться в очереди ожидания из-за накопления задач вследствие ограничения параллелизма, настроек троттлинга или других задержек. Такие запуски можно автоматически отменять, если они слишком долго ожидают начала выполнения.

Свойство `timeouts.start` управляет этим таймаутом. В следующем примере запуски принудительно отменяются, если первый шаг не начал выполняться в течение 10 секунд:

<LanguageSection show="typescript">
```ts {{ title: "inngest/function.ts" }}
const scheduleReminder = inngest.createFunction(
  {
    id: "schedule-reminder",
    timeouts: {
      // If the run takes longer than 10s to start, cancel the run.
      start: "10s",
    },
    triggers: { event: "tasks/reminder.created" },
  },
  async ({ event, step }) => {
    await step.run('send-reminder-push', async () => {
      await pushNotificationService.push(event.data.reminder)
    })
  }
  // ...
);
```
</LanguageSection>

## `timeouts.finish` — Таймаут для выполняющихся запусков

Может потребоваться ограничить общую продолжительность запуска после его начала. Функции можно автоматически отменять, если они выполняются слишком долго.

Свойство `timeouts.finish` управляет этим таймаутом. В следующем примере запуски принудительно отменяются, если после начала выполнения прошло более 30 секунд:

<LanguageSection show="typescript">
```ts {{ title: "inngest/function.ts" }}
const scheduleReminder = inngest.createFunction(
  {
    id: "schedule-reminder",
    timeouts: {
      // If the run takes longer than 10s to start, cancel the run.
      start: "10s",
      // And if the run takes longer than 30s to finish after starting, cancel the run.
      finish: "30s",
    },
    triggers: { event: "tasks/reminder.created" },
  },
  async ({ event, step }) => {
    await step.run('send-reminder-push', async () => {
      await pushNotificationService.push(event.data.reminder)
    })
  }
  // ...
);
```
</LanguageSection>

## Советы

* `timeouts.start` ограничивает время ожидания запуска в очереди до начала первого шага
* Как только начинается первая попытка выполнения шага, свойство `timeouts.start` перестаёт действовать. Вместо него начинает отсчитываться таймаут `timeouts.finish`.
* После начала выполнения `timeouts.finish` ограничивает, как долго может выполняться запуск
* Оба свойства можно комбинировать для контроля общей продолжительности запуска функции
* Запуски, отменённые по таймауту, генерируют событие [`inngest/function.cancelled`](/docs/reference/system-events/inngest-function-cancelled)

## Ограничения

- Продолжительность шага не зависит от таймаутов. Например, 5-минутный таймаут не прервёт шаг досрочно по истечении 5 минут.
- Ограничение параллелизма может задержать применение таймаутов. Например, если функция настроена с 5-минутным таймаутом, но ограничение параллелизма задерживает запуск на 10 минут, отмена займёт 10 минут.
