# Отмена по событиям


Как вы уже узнали, функции можно запускать с помощью событий — точно так же можно отменять активные функции, отправив событие.

В качестве примера возьмём приложение-напоминалку, где пользователь может запланировать напоминание на будущее. Пользователь также может удалить напоминание, если передумал.

<LanguageSection show="typescript">

Откладывать выполнение кода на дни или недели легко с помощью `step.sleepUntil`, но нам нужен способ остановить функцию, если пользователь удалит напоминание, пока функция «спит».

При определении функции можно указать параметр `cancelOn`, который позволяет задать одно или несколько событий, при получении которых Inngest прекратит сон и пометит функцию как «Отменённую».

Вот функция планирования напоминаний с использованием `cancelOn`:


```ts {{ title: "inngest/syncContacts.ts" }}
const scheduleReminder = inngest.createFunction(
  {
    id: "schedule-reminder",
    cancelOn: [{
      event: "tasks/reminder.deleted", // The event name that cancels this function
      // Ensure the cancellation event (async) and the triggering event (event)'s reminderId are the same:
      if: "async.data.reminderId == event.data.reminderId",
    }],
    triggers: { event: "tasks/reminder.created" }
  },
  async ({ event, step }) => {
    await step.sleepUntil('sleep-until-remind-at-time', event.data.remindAt);
    await step.run('send-reminder-push', async ({}) => {
      await pushNotificationService.push(event.data.userId, event.data.reminderBody)
    })
  }
  // ...
);
```


Разберём, как это работает:

1. При каждом запуске функции создаётся слушатель отмены, ожидающий получения события `"tasks/reminder.deleted"`.
2. Условие `if` сообщает Inngest, что и триггерное событие (`"tasks/reminder.created"`), и событие отмены (`"tasks/reminder.deleted"`) должны иметь одинаковое значение `data.reminderId` в полезной нагрузке. Это гарантирует, что одно событие не отменит другое напоминание.

Подробнее о написании выражений читайте в нашем руководстве [по написанию выражений](/docs/guides/writing-expressions).

Вот пример двух событий, которые будут сопоставлены по полю `data.reminderId`:

### Советы

* Можно дополнительно указать `timeout`, чтобы разрешить отмену только в течение определённого периода времени.
* Можно настроить до пяти событий для отмены функции.
* Можно написать более сложное выражение сопоставления с помощью поля `if`.

Подробнее в полном [справочнике](/docs/reference/typescript/functions/cancel-on).

Вот функция планирования напоминаний с использованием `cancel`:

```py {{ title: "inngest/schedule_reminder.py" }}
@inngest_client.create_function(
    fn_id="schedule-reminder",
    trigger=inngest.TriggerEvent(event="tasks/reminder.created"),
    cancel=[inngest.Cancel(
                event="tasks/reminder.deleted", # The event name that cancels this function
                # Ensure the cancellation event (async) and the triggering event (event)'s reminderId are the same:
                if_exp="async.data.reminderId == event.data.reminderId"
            )],
)
async def fn(ctx: inngest.Context) -> None:
    # Step 1
    await ctx.step.sleep_until(
        "sleep-until-remind-at-time",
         ctx.event.data["remind_at"],
    )

    # Step 2
    await ctx.step.run("send-reminder-push", send_reminder_push)



async def send_reminder_push() -> None:
    pass
```

1. При каждом запуске функции создаётся слушатель отмены, ожидающий получения события `"tasks/reminder.deleted"`.
2. Условие `if` сообщает Inngest, что и триггерное событие (`"tasks/reminder.created"`), и событие отмены (`"tasks/reminder.deleted"`) должны иметь одинаковое значение `data.reminderId` в полезной нагрузке. Это гарантирует, что одно событие не отменит другое напоминание.
