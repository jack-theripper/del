# Отмена

Отмена — полезный механизм для предотвращения ненужных действий на основе предыдущих (например, пропуска генерации отчёта при удалении аккаунта) или остановки нежелательного запуска функции, состоящей из нескольких шагов (например, ошибка развёртывания, дубликаты).

Inngest позволяет отменять запущенные функции через API, панель управления или на основе событий:

## Анатомия отмены

Механизмы отмены Inngest предотвращают запуск запланированной функции или останавливают выполняемую функцию между последующими шагами (шагами сна или действия).

Обратите внимание:

- Отмена функции с активно выполняющимся шагом не остановит выполнение этого шага. Все активно выполняющиеся шаги будут завершены.
- Отмена набора запусков функций не предотвращает постановку новых запусков в очередь (например, в случае проблем с циклами). Вместо этого рассмотрите использование [паузы функций](/docs/guides/pause-functions).

Рассмотрим следующую функцию Inngest:

```ts {{ title: "inngest/scheduleReminder.ts" }}
const scheduleReminder = inngest.createFunction(
  {
    id: "schedule-reminder",
    cancelOn: [{ event: "tasks/deleted", if: "event.data.id == async.data.id" }],
    triggers: { event: "tasks/reminder.created" }
  },
  async ({ event, step }) => {
    // Шаг 1
    await step.sleepUntil('sleep-until-remind-at-time', event.data.remindAt);
    // Шаг 2
    await step.run('send-reminder-push', async ({}) => {
      await pushNotificationService.push(event.data.userId, event.data.reminderBody)
    })
  }
  // ...
);
```

Рассмотрим два варианта отмены через UI массовой отмены панели управления:

<Row>
  <Col>

   **Отмена запуска до или после Шага 1**

    1. Запуск функции подхватывается Inngest
    2. Шаг 1 обрабатывается, запускается сон до следующей недели
    3. Через три дня поступает запрос на отмену
    4. Запуск функции отменяется (Шаг 2 пропускается)


  </Col>
  <Col>

    **Отмена запуска во время выполнения Шага 2**

    1. Запуск функции подхватывается Inngest
    2. Шаг 1 обрабатывается, запускается сон до следующей недели
    3. Через неделю поступает запрос на отмену, но Шаг 2 уже начат
    4. Шаг 2 выполняется до завершения
    5. Запуск функции помечается как «отменённый»

  </Col>
</Row>

Все отменённые запуски функций можно повторить с помощью [UI повтора функций](/docs/platform/replay) в платформе.

## Обработка отменённых функций

Отменённые запуски функций могут потребовать дополнительных действий: очистки базы данных или удаления временных ресурсов. Это можно сделать, используя системное событие [`inngest/function.cancelled`](/docs/reference/system-events/inngest-function-cancelled).

Смотрите [этот полный пример](/docs/examples/cleanup-after-function-cancellation) использования данного события в вашей системе для очистки после отмены запуска функции.
