# React-хуки / Next.js <VersionBadge version="TypeScript SDK v3.32.0+" />

Realtime предоставляет React-хук [`useInngestSubscription()`](#use-inngest-subscription-api-reference), обеспечивающий полностью типизированный опыт подписки на каналы.

`useInngestSubscription()` безопасно подписывается на каналы, используя токен подписки, полученный с сервера.

В Next.js это реализуется как серверное действие (server action), возвращающее токен, который затем используется клиентом для подписки:



```tsx {{filename: "src/actions.ts"}}
"use server";
// securely fetch an Inngest Realtime subscription token from the server as a server action
export async function fetchSubscriptionToken(): Promise<Realtime.Token<typeof helloChannel, ["logs"]>> {
  const token = await getSubscriptionToken(getInngestApp(), {
    channel: helloChannel(),
    topics: ["logs"],
  });

  return token;
}
```


```tsx {{filename: "src/App.tsx"}}
"use client";

import { useInngestSubscription } from "@inngest/realtime/hooks";
import { getSubscriptionToken, Realtime } from "@inngest/realtime";
import { getInngestApp } from "@/inngest";
import { helloChannel } from "@/inngest/functions/helloWorld";
// import the server action to securely fetch the Realtime subscription token
import { fetchRealtimeSubscriptionToken } from "./actions";

export default function Home() {
  // subscribe to the hello-world channel via the subscription token
  // `data` is fully typed based on the selected channel and topics!
  const { data, error } = useInngestSubscription({
    refreshToken: fetchRealtimeSubscriptionToken,
  });

  return (
    <div>
      <h1>Realtime</h1>
      {data.map((message, i) => (
        <div key={i}>{message.data}</div>
      ))}
    </div>
  )
}
```

## Справочник API `useInngestSubscription()`


### Параметры

- `enabled?: boolean` — включена ли подписка хука.
- `bufferInterval?: number` — если установлено значение выше `0`, вывод будет обновляться каждые `n` миллисекунд. Помогает при очень активных потоках, которые могут перегрузить интерфейс.
- `token?: Realtime.Subscribe.Token` — токен для подписки (смотрите [Подписка с клиента](/docs/features/realtime#subscribe-from-the-client)).
- `refreshToken?: () => Promise<Realtime.Subscribe.Token>` — функция, вызываемая если токен отсутствует или хук был повторно включён, а предыдущий токен истёк.

<Warning>
  Требуется параметр `token` или `refreshToken`.
</Warning>

### Возвращаемые значения

- `data: Array<Realtime.Message>` — все сообщения, полученные по подписке в хронологическом порядке.
- `latestData: Realtime.Message` — ярлык для последнего полученного сообщения. Полезно для потоков, где каждое сообщение является последним состоянием сущности.
- `freshData: Array<Realtime.Message>` — если активен `bufferInterval`, это будет последний пакет сообщений, освобождённых из буфера. Если `bufferInterval` неактивен, это всегда последнее сообщение.
- `error: Error | null` — если не равно `null`, указывает на ошибку с подпиской.
- `state: InngestSubscriptionState` — текущее состояние подписки: `"closed"`, `"error"`, `"refresh_token"`, `"connecting"`, `"active"` или `"closing"`.
- `clear: () => void` — функция для очистки всех накопленных данных сообщений из внутреннего состояния. Включает массивы `data`, `freshData` и `latestData`. Не влияет на соединение или состояние ошибки.
