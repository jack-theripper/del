## Формат полезной нагрузки события

Полезная нагрузка события — это JSON-объект, который должен содержать свойства `name` и `data`.

### Обязательные свойства

* `name` — тип или категория события. Поле `name` используется для [запуска функций](/docs/functions). Например: `app/user.created` или `billing/invoice.paid`. Смотрите [советы по именованию событий](#tips-for-event-naming) ниже.
* `data` — любые данные, которые вы хотите связать с событием. Данные будут сериализованы в JSON. Например, при отправке события об оплаченном счёте можно включить `id` счёта, `amount` и `customerId` в свойство `data`. Свойство `data` может содержать любой вложенный JSON-объект, включая объекты и массивы.

### Необязательные свойства

* `id` — уникальный идентификатор события для предотвращения дублирующихся событий. Подробнее о [дедупликации](/docs/events#deduplication).
* `ts` — временная метка события в миллисекундах с начала эпохи Unix. Если не указана, будет установлена равной времени получения события Inngest.
* `v` — версия полезной нагрузки события. Полезна для отслеживания изменений структуры полезной нагрузки со временем. Например: `"2024-01-14.1"`.

```py {{ title: "Pydantic: представление типов" }}
import inngest
import pydantic
import typing

TEvent = typing.TypeVar("TEvent", bound="BaseEvent")

class BaseEvent(pydantic.BaseModel):
    data: pydantic.BaseModel
    id: str = ""
    name: typing.ClassVar[str]
    ts: int = 0

    @classmethod
    def from_event(cls: type[TEvent], event: inngest.Event) -> TEvent:
        return cls.model_validate(event.model_dump(mode="json"))

    def to_event(self) -> inngest.Event:
        return inngest.Event(
            name=self.name,
            data=self.data.model_dump(mode="json"),
            id=self.id,
            ts=self.ts,
        )

class InvoicePaidEventData(pydantic.BaseModel):
    customerId: str
    invoiceId: str
    amount: int
    metadata: dict

class InvoicePaidEvent(BaseEvent):
    data: InvoicePaidEventData
    name: typing.ClassVar[str] = "billing/invoice.paid"
```

### Советы по именованию событий

Имена событий используются для запуска функций. Рекомендуем использовать единое соглашение об именовании для всех событий — это упростит поиск и запуск функций в будущем. Несколько советов:

* **Объект-Действие**: Используйте паттерн «Объект-Действие» — существительное и глагол. Это удобно для группировки связанных событий: `account.created`, `account.updated`, `account.deleted`.
* **Прошедшее время**: Используйте глагол в прошедшем времени для действия: `uploaded`, `paid`, `completed`, `sent`.
* **Разделители**: Используйте точечную нотацию и/или подчёркивания для разделения слов: `user.created` или `blog_post.published`.
* **Префиксы**: Используйте префиксы для группировки связанных событий: `api/user.created`, `billing/invoice.paid`, `stripe/customer.created`. Особенно удобно при наличии нескольких приложений, отправляющих события в Inngest.

Не существует единственно правильного способа именовать события. Главное — придерживаться одного подхода и использовать соглашение, которое логично для вашего приложения.
