# Neon

Inngest позволяет запускать функции из обновлений вашей базы данных Neon Postgres.

## Преимущества запуска функций из событий базы данных

Разделяя триггеры функций и логику приложения, события инициируются обновлениями базы данных, а не зависят от инструментации в коде для их отправки. Это гарантирует, что вы не пропустите событие при манипуляции данными в приложении. Такое разделение создаёт чистый уровень абстракции между операциями с базой данных и кодом, выполняемым асинхронно.

Кроме того, поскольку события базы данных поступают в систему Inngest для постановки новых функций в очередь, это может устранить необходимость в архитектурных паттернах, таких как [паттерн транзакционного журнала исходящих сообщений (transactional outbox)](https://microservices.io/patterns/data/transactional-outbox.html).

### Использование функций Inngest с триггерами базы данных

Помимо архитектурных преимуществ, некоторые функции Inngest отлично сочетаются с триггерами базы данных:

- [**Fan-out**](/docs/guides/fan-out-jobs) — используйте одно событие базы данных для запуска нескольких функций параллельно. Например, `pg/users.inserted` может запустить функцию отправки приветственного письма и функцию начала пробного периода в Stripe.
- [**Пакетная обработка**](/docs/guides/batching) — события базы данных можно объединять в пакеты для более эффективной обработки множества обновлений. Например, множество небольших обновлений можно агрегировать или эффективно выполнять массовые операции через сторонние API с поддержкой пакетной обработки, такие как Shopify.
- [**Управление потоком**](/docs/guides/flow-control) — объединяйте триггеры базы данных с функциями управления потоком: троттлингом, дебаунсингом или ограничением частоты запросов для лучшего управления ресурсами. Например, используйте троттлинг при работе с лимитами частоты запросов к сторонним API или дебаунсинг для операций, которые могут выполняться часто, чтобы избежать избыточной работы.

## Как это работает
После подключения Neon к Inngest любые изменения данных в вашей базе данных будут автоматически отправлять новые события в ваш аккаунт Inngest.

<Info>
  Интеграция Neon в настоящее время поддерживает только одно подключение к базе данных для production-среды Inngest.
  Другие среды пока не поддерживаются.
</Info>

## Подключение Neon к Inngest
Для подключения Neon потребуются некоторые изменения конфигурации на стороне Postgres и проекта Neon.

Установка интеграции Neon в Inngest состоит из трёх шагов:

1. **Авторизация:** предоставьте учётные данные Postgres, чтобы Inngest мог получить доступ к базе данных для выполнения установки
2. **Включение логической репликации:** измените конфигурацию `wal_level` на `logical`
3. **Подключение базы данных Neon к Inngest**

Neon находится на странице интеграций в панели управления Inngest. Нажмите «Connect», чтобы начать процесс настройки:

### 1. Авторизация Inngest
<Callout>
Inngest не хранит ваши учётные данные. Не обновляйте страницу в процессе выполнения шагов, иначе учётные данные будут потеряны. В этом случае вам будет предложено авторизовать Inngest повторно.
</Callout>

Введите учётные данные Postgres и нажмите кнопку «Verify» для начала процесса проверки:
<ImageTheme
  light={'/assets/docs/features/events-triggers/neon/neon-authorize-step.png'}
  className="my-0"
  alt={'Шаг авторизации Neon на странице интеграций Inngest'}
/>


### 2. Включение логической репликации
Убедитесь, что в вашем проекте Neon включена логическая репликация.

Включите логическую репликацию автоматически через панель управления Neon:
<ImageTheme
  light={'/assets/docs/features/events-triggers/neon/neon-dashboard.png'}
  className="my-0"
  alt={'Настройки панели управления Neon с опцией включения логической репликации'}
/>

Или следуйте инструкциям в [руководстве Neon](https://neon.tech/docs/guides/logical-replication-postgres-to-neon#enable-logical-replication-in-the-source-neon-project) для нахождения и редактирования файла postgresql.conf.
После завершения вернитесь в Inngest и нажмите «Verify logical replication is enabled»:

<ImageTheme
  light={'/assets/docs/features/events-triggers/neon/neon-logical-replication-step.png'}
  className="my-0"
  alt={'Шаг проверки логической репликации на странице интеграций Inngest'}
/>

### 3. Подключение
Есть два способа подключения к базе данных Neon:

- Автоматически
- Вручную *(скоро)*

Inngest автоматически настроит и подключится к вашей базе данных Neon. Будет создана роль Postgres для репликации, предоставлен доступ к схеме для этой роли, создан слот репликации и публикация.
