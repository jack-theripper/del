Каждый из этих триггеров можно настраивать различными способами:

- **[Фильтрация триггеров событий](/docs/guides/writing-expressions)** — запускайте функцию только для подмножества подходящих отправленных событий.
- **[Отложенное выполнение](/docs/guides/delayed-functions)** — запускайте функцию в определённый момент времени в будущем.
- **[Пакетная обработка событий](/docs/guides/batching)** — обрабатывайте несколько событий в одной функции для большей эффективности.
- **[Несколько триггеров](/docs/guides/multiple-triggers)** — используйте одну функцию для обработки нескольких типов событий.


## Зачем использовать события?

Использование событий для запуска функций Inngest вместо прямых вызовов даёт большую гибкость:

- Одно событие может запускать несколько функций Inngest.
- События можно использовать для синхронизации запусков функций через [отмену](/docs/features/inngest-functions/cancellation) и [шаг «ожидание события»](/docs/reference/functions/step-wait-for-event).
- События позволяют запускать функции в нескольких приложениях.
- Похожие события можно группировать для более быстрой обработки.

События служат удобным связующим звеном между действиями вашего приложения (например, `user.signup`) и кодом приложения (например, `sendWelcomeEmail()` и `importContacts()`):
