# Промежуточное ПО (Middleware)

Промежуточное ПО позволяет запускать код в различные моменты жизненного цикла клиента Inngest — например, во время выполнения функции или при отправке события.

## Поддержка в SDK

Промежуточное ПО доступно в [TypeScript SDK](/docs/reference/typescript/v4/middleware/lifecycle) и [Python SDK](/docs/reference/python/middleware/lifecycle).

Поддержка в Go SDK планируется.

## Жизненный цикл промежуточного ПО

Промежуточное ПО можно регистрировать на уровне клиента Inngest или функций.

Добавление промежуточного ПО формирует общий «стек» промежуточного ПО. При регистрации нескольких промежуточных ПО SDK группирует и выполняет хуки каждого в следующем порядке:

1. Промежуточное ПО, зарегистрированное на **клиенте**, в порядке убывания
2. Промежуточное ПО, зарегистрированное на **функции**, в порядке убывания

Например:

<CodeGroup>
```ts {{ title: "TypeScript" }}
const inngest = new Inngest({
  id: "my-app",
  middleware: [
    logMiddleware, // This is executed first
    errorMiddleware, // This is executed second
  ],
});

inngest.createFunction(
  {
    id: "example",
    triggers: { event: "test" },
    middleware: [
      dbSetupMiddleware, // This is executed third
      datadogMiddleware, // This is executed fourth
    ],
  },
  async () => {
    // ...
  }
);
```
```py {{ title: "Python" }}
inngest_client = inngest.Inngest(
    app_id="my_app",
    middleware=[
        LogMiddleware, # This is executed first
        ErrorMiddleware # This is executed second
    ],
)

# ...


@inngest_client.create_function(
    fn_id="import-product-images",
    trigger=inngest.TriggerEvent(event="shop/product.imported"),
    middleware=[
        DbSetupMiddleware, # This is executed third
        DatadogMiddleware # This is executed fourth
    ],
)
async def fn(ctx: inngest.Context):
    # ...

```
</CodeGroup>

Подробнее о хуках промежуточного ПО и порядке их выполнения в разделе [«Создание промежуточного ПО»](/docs/features/middleware/create).
