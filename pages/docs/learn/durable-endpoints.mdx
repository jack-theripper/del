# Durable Endpoints <VersionBadge version="Beta" />

Durable Endpoints позволяют создавать или преобразовывать ваш API в отказоустойчивые эндпоинты, просто оборачивая критическую логику в [надежные шаги (durable steps)](/docs/learn/inngest-steps).

Durable Endpoints ведут себя как обычные эндпоинты. Ментальная модель остается прежней: запрос, ответ. Но каждый шаг добавляет трассировку, наблюдаемость и логику повторов с момента сбоя.


<Callout>
  Durable Endpoints доступны в SDK для TypeScript и Go.
</Callout>

## Когда использовать Durable Endpoints

**У вас есть эндпоинты, которые падают на полпути.** Любой эндпоинт с несколькими шагами, где сбой на шаге 3 означает, что работа на шагах 1 и 2 была напрасной. Вместо того чтобы писать логику try/catch повсюду или надеяться на лучшее, просто оберните свой код в шаги, и пусть выполнение возобновляется с того места, где оно остановилось.

**Вам нужна наблюдаемость без сложной настройки.** Если вы хотите видеть, что происходит в ваших эндпоинтах, без настройки множества внешних сервисов, Durable Endpoints дают вам это мгновенно.

**Вы уже используете Inngest.** Вы можете добавить надежность другим эндпоинтам без переработки всего в рабочий процесс или глубоких раздумий о логике событий.

Затем оберните ваш API-эндпоинт с помощью `inngest.endpoint` и перенесите критическую логику вашего эндпоинта в блоки `step.run`:

<CodeGroup>
```typescript {{ title: "Next.js" }}
import { step } from "inngest";
import { inngest } from "@/inngest/client";
import { NextRequest } from "next/server";

export const POST = inngest.endpoint(async (req: NextRequest) => {
  const { userId, data } = await req.json();

  // Шаг 1: Валидация и обогащение данных
  const enriched = await step.run("enrich-data", async () => {
    const user = await db.users.find(userId);
    return { ...data, account: user.accountId };
  });

  // Шаг 2: Обработка обогащенных данных
  const result = await step.run("process", async () => {
    return await processData(enriched);
  });

  // Шаг 3: Отправка уведомления
  await step.run("notify", async () => {
    await sendNotification(userId, result);
  });

  return Response.json({ success: true, result });
});
```
```typescript {{ title: "Bun" }}
import { Inngest, step } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({ id: "my-app", endpointAdapter });

Bun.serve({
  port: 3000,
  routes: {
    "/process": inngest.endpoint(async (req) => {
      const { userId, data } = await req.json();

      // Шаг 1: Валидация и обогащение данных
      const enriched = await step.run("enrich-data", async () => {
        const user = await db.users.find(userId);
        return { ...data, account: user.accountId };
      });

      // Шаг 2: Обработка обогащенных данных
      const result = await step.run("process", async () => {
        return await processData(enriched);
      });

      // Шаг 3: Отправка уведомления
      await step.run("notify", async () => {
        await sendNotification(userId, result);
      });

      return new Response(JSON.stringify({ success: true, result }));
    }),
  },
});
```
</CodeGroup>

Если `process` упадет, эндпоинт повторит попытку с этого шага. `enrich-data` не будет запускаться повторно.

**Прочтите [Справочник по Durable Endpoint TypeScript SDK](/docs/reference/typescript/durable-endpoints) для получения более подробной информации об использовании.**

### Использование шагов (Steps)

Durable Endpoints поддерживают все те же методы шагов, что и функции Inngest. См. [документацию по шагам](/docs/learn/inngest-steps) для получения полной справки:

- [`step.run()`](/docs/learn/inngest-steps#step-run): Надежно выполняет предоставленный блок, повторяя попытку в случае сбоя
- [`step.sleep()`](/docs/learn/inngest-steps#step-sleep): Приостанавливает выполнение на определенное время
- [`step.waitForEvent()`](/docs/learn/inngest-steps#step-wait-for-event-step-wait-for-event): Ожидает внешнего события

</LanguageSection>

<LanguageSection show="go">


Чтобы начать использовать шаги в ваших API-эндпоинтах, вы должны сначала настроить промежуточное ПО (middleware) для перехвата HTTP-запросов.

```go
import (
	"context"

	"github.com/inngest/inngestgo/step"
	"github.com/inngest/inngestgo/stephttp"
)

func setuphttp() {
	// provider добавляет поддержку inngest к http-обработчикам
	provider := stephttp.Setup(stephttp.SetupOpts{
		Domain: "api.example.com", // добавьте сюда ваш домен api.
	})

  // provider позволяет оборачивать отдельные http-обработчики через `provider.servehttp`,
  // и предоставляет stdlib-совместимое промежуточное ПО через `provider.middleware`
  http.HandleFunc("/users", provider.ServeHTTP(handleUsers))

  // или через middleware, например, с chi:
  r := chi.NewRouter()
  r.Use(provider.Middleware)
  r.Get("/users", handleUsers)
}
```

После добавления промежуточного ПО вы можете настраивать функции и выполнять шаги непосредственно в REST-эндпоинтах:

```go
import (
	"context"

	"github.com/inngest/inngestgo/step"
	"github.com/inngest/inngestgo/stephttp"
)

func handleUsers(w http.ResponseWriter, r *http.Request) {
	ctx := r.Context()

	stephttp.Configure(ctx, stephttp.FnOpts{
		// Настройте ID функции, удалив ID из URL:
		ID: "/users/{id}"
	})

	// Шаг 1: Аутентификация (с полной наблюдаемостью)
	auth, err := step.Run(ctx, "authenticate", func(ctx context.Context) (*AuthResult, error) {
		// Вы можете связывать шаги как обычно...
		return nil, nil
	})
	if err != nil {
		http.Error(w, "Authentication failed", http.StatusUnauthorized)
		return
	}
	// ...
}
```

## Запрос к Durable Endpoint

Durable Endpoints ведут себя как обычные API-эндпоинты при успешном выполнении. Вы можете запрашивать их из фронтенда (_или бэкенда_), используя `fetch()` или вашу любимую библиотеку для запросов:

Однако, когда сбой вызывает повторные попытки, Durable Endpoint возвращает перенаправление (redirect) на специальный эндпоинт в Inngest Cloud для опроса финального результата.

Вот пример кода, обрабатывающего как прямой результат, так и результат после перенаправления при повторах:

```typescript
function handleError(error) {
  // ...
}

async function handleResult(result) {
  const result = await res.json()
  // ...
}

fetch(`/api/your-durable-endpoint`)
  .then((res) => {
    if (res.redirected) {
      // следуем за перенаправлением
      fetch(res.url)
        .then(handleResult)
        .catch(handleError);
    } else {
      handleResult(res)
    }
  })
  .catch(handleError);
```

<Callout>
  Поскольку Durable Endpoint перенаправляет запрос на специальный эндпоинт в облаке Inngest, `fetch()` не может просто автоматически следовать за этим перенаправлением (_из-за политики CORS_).
  Вместо этого вам нужно получить URL перенаправления (`res.url`) и инициировать новый запрос `fetch()`.
</Callout>

## Ограничения

Durable Endpoints в данный момент находятся в стадии бета-тестирования. Применяются следующие ограничения:

- **Управление потоком (Flow control) не поддерживается** — такие функции, как ограничения параллелизма (concurrency limits) и ограничение частоты запросов (rate limiting), недоступны для Durable Endpoints
- **Тело POST-запроса пока не поддерживается** — предпочитайте использовать строки запроса (query strings) для передачи данных. Поддержка тела POST-запроса появится скоро
- **Только стандартные HTTP-ответы** — Durable Endpoints должны возвращать стандартный HTTP-ответ, а не поток SSE
