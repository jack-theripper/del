# Шаги Inngest (Inngest Steps)

Шаги — это фундаментальные строительные блоки в функциях Inngest. Каждый шаг представляет собой отдельную задачу (или другую единицу работы) внутри функции, которая может быть выполнена независимо.

Шаги имеют решающее значение, поскольку они позволяют функциям выполнять определенные задачи контролируемым и последовательным (или параллельным) образом. Вы можете создавать сложные рабочие процессы, объединяя простые дискретные операции.

На этой странице вы узнаете о преимуществах использования шагов и получите обзор доступных методов шагов.

## Преимущества использования шагов

- **Повышенная надежность**: структурированные шаги обеспечивают точный контроль и обработку каждой задачи внутри функции.
- **Обработка ошибок**: захват и управление ошибками на уровне шага означает лучшее восстановление после сбоев.
- **Механизм повторов**: неудачные шаги могут быть перезапущены и восстановлены независимо, без повторного выполнения других успешных шагов.
- **Независимое тестирование**: каждый шаг можно тестировать и отлаживать независимо от других.
- **Улучшенная читаемость кода**: модульный подход облегчает навигацию по коду и рефакторинг.

Если вы хотите узнать больше о том, как выполняются шаги Inngest, посетите страницу ["Как выполняются функции Inngest"](/docs/learn/how-functions-are-executed).

## Анатомия шага Inngest

Первым аргументом каждого метода шага Inngest является `id`. Каждый шаг рассматривается как дискретная задача, которую можно индивидуально перезапускать, отлаживать или восстанавливать. Inngest использует ID для мемоизации состояния шага между версиями функций.

<LanguageSelector>

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "import-product-images", triggers: { event: "shop/product.imported" } },
  async ({ event, step }) => {
    const uploadedImageURLs = await step.run(
      // ID шага
      "copy-images-to-s3",
      // другие аргументы, в данном случае: обработчик
      async () => {
        return copyAllImagesToS3(event.data.imageURLs);
    });
  }
);
```

</LanguageSection>

<LanguageSection show="go">

```go
import (
  "github.com/inngest/inngestgo"
  "github.com/inngest/inngestgo/step"
)

inngestgo.CreateFunction(
	client,
	// конфиг
	inngestgo.FunctionOpts{
		ID: "import-product-images",
	},
	// триггер (событие или cron)
	inngestgo.EventTrigger("shop/product.imported", nil),
	// функция обработчик
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		// Здесь находится бизнес-логика
		// Оборачивая код в шаги, он будет автоматически перезапущен при сбое
		s3Urls, err := step.Run("copy-images-to-s3", func() ([]string, error) {
			return copyAllImagesToS3(input.Event.Data["imageURLs"].([]string))
		})
		if err != nil {
			return nil, err
		}

		return nil, nil
	},
)
```

</LanguageSection>

<LanguageSection show="python">

```python
import inngest
from src.inngest.client import inngest_client

@inngest_client.create_function(
    fn_id="import-product-images",
    event="shop/product.imported"
)
async def import_product_images(ctx: inngest.Context):
    uploaded_image_urls = await ctx.step.run(
        # ID шага
        "copy-images-to-s3",
        # другие аргументы, в данном случае: обработчик
        lambda: copy_all_images_to_s3(ctx.event.data["image_urls"])
    )
```

</LanguageSection>

</LanguageSelector>

ID также используется для идентификации функции в системе Inngest.

SDK Inngest также записывает счетчик для каждого уникального ID шага. Счетчик увеличивается каждый раз, когда вызывается один и тот же шаг. Это позволяет запускать один и тот же шаг в цикле без изменения ID.

<Callout>
Обратите внимание, что каждый шаг выполняется как **отдельный HTTP-запрос**. Чтобы обеспечить эффективное и правильное выполнение, помещайте любую недетерминированную логику (например, вызовы БД или API) внутри вызова `step.run()`.
</Callout>

## Доступные методы шагов

<LanguageSelector>

### step.run()

Этот метод выполняет определенный фрагмент кода. Код внутри `step.run()` автоматически перезапускается, если он выдает ошибку. Когда `step.run()` успешно завершается, ответ сохраняется в состоянии запуска функции, и шаг не будет запускаться повторно.

Используйте его для запуска синхронного или асинхронного кода как перезапускаемого шага в вашей функции.

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "import-product-images", triggers: { event: "shop/product.imported" } },
  async ({ event, step }) => {
    // Здесь находится бизнес-логика
    // Оборачивая код в шаги, он будет автоматически перезапущен при сбое
    const uploadedImageURLs = await step.run("copy-images-to-s3", async () => {
      return copyAllImagesToS3(event.data.imageURLs);
    });
  }
);
```

</LanguageSection>

<LanguageSection show="go">

```go

  "github.com/inngest/inngestgo"
  "github.com/inngest/inngestgo/step"

			return copyAllImagesToS3(input.Event.Data["imageURLs"].([]string))
		})

			return nil, err
		}

		return nil, nil
	},

```

</LanguageSection>


<Callout>
`step.run()` действует как транзакция на уровне кода. Весь шаг должен завершиться успешно, чтобы считаться выполненным.
</Callout>

### step.sleep()

Этот метод приостанавливает выполнение на указанный срок. Несмотря на то, что это похоже на `setInterval`, ваша функция не работает в это время (вы не используете вычислительные ресурсы). Inngest берет на себя планирование за вас. Используйте его для добавления задержек или ожидания в течение определенного времени перед продолжением. Максимально функции могут "спать" до одного года (семь дней для [бесплатных тарифных планов](/pricing)).

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "send-delayed-email", triggers: { event: "app/user.signup" } },
  async ({ event, step }) => {
    await step.sleep("wait-a-couple-of-days", "2d");
    // Сделать что-то еще
  }
);
```

</LanguageSection>

<LanguageSection show="go">

```go
import (
  "github.com/inngest/inngestgo"
  "github.com/inngest/inngestgo/step"
)

inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{
		ID: "send-delayed-email",
	},
	inngestgo.EventTrigger("app/user.signup", nil),
	// функция обработчик
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		step.Sleep("wait-a-couple-of-days", 2*time.Day)
		return nil, nil
	},
)
```

</LanguageSection>

### step.sleepUntil() / step.sleep_until()

Этот метод приостанавливает выполнение до определенной даты и времени. Любая строка даты и времени в формате, принимаемом объектом Date, например `YYYY-MM-DD` или `YYYY-MM-DDHH:mm:ss`. Максимально функции могут "спать" до одного года (семь дней для [бесплатных тарифных планов](/pricing)).

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "send-scheduled-reminder", triggers: { event: "app/reminder.scheduled" } },
  async ({ event, step }) => {
    const date = new Date(event.data.remind_at);
    await step.sleepUntil("wait-for-the-date", date);
    // Сделать что-то еще
  }
);
```

</LanguageSection>

<LanguageSection show="go">

Go SDK не имеет метода `sleepUntil`. Используйте `step.Sleep()` с рассчитанной продолжительностью.

</LanguageSection>

<LanguageSection show="python">

```python
import inngest
from src.inngest.client import inngest_client
from datetime import datetime

@inngest_client.create_function(
    fn_id="send-scheduled-reminder",
    trigger=inngest.TriggerEvent(event="app/reminder.scheduled")
)
async def send_scheduled_reminder(ctx: inngest.Context):
    date = datetime.fromisoformat(ctx.event.data["remind_at"])
    await ctx.step.sleep_until("wait-for-the-date", date)
    # Сделать что-то еще
```

### step.waitForEvent() / step.wait_for_event()

Этот метод приостанавливает выполнение запуска до тех пор, пока не будет получено определенное событие.

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "send-onboarding-nudge-email", triggers: { event: "app/account.created" } },
  async ({ event, step }) => {
    const onboardingCompleted = await step.waitForEvent(
      "wait-for-onboarding-completion",
      { event: "app/onboarding.completed", timeout: "3d", if: "event.data.userId == async.data.userId" }
    );
    // Сделать что-то еще
  }
);
```

</LanguageSection>

<LanguageSection show="go">

```go
import (
  "github.com/inngest/inngestgo"
  "github.com/inngest/inngestgo/errors"
  "github.com/inngest/inngestgo/step"
)

inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{
		ID: "send-delayed-email",
	},
	inngestgo.EventTrigger("app/user.signup", nil),
	// функция обработчик
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		// Опрос потока событий для новых событий. Функция остановится
		// и автоматически возобновится при обнаружении подходящего события
		// или при достижении тайм-аута.
		fn, err := step.WaitForEvent[FunctionCreatedEvent](
			ctx,
			"wait-for-activity",
			step.WaitForEventOpts{
				Name:    "Ожидание создания функции",
				Event:   "api/function.created",
				Timeout: time.Hour * 72,
				// Сопоставление событий, где user_id совпадает в асинхронном событии.
				If: inngestgo.StrPtr("event.data.user_id == async.data.user_id"),
			},
		)
		if err == step.ErrEventNotReceived {
			// Функция не была создана в течение 3 дней. Отправить последующее письмо.
			_, _ = step.Run(ctx, "follow-up-email", func(ctx context.Context) (any, error) {
				// ...
				return true, nil
			})
			return nil, nil
		}
		return nil, nil
	},
)
```

</LanguageSection>

### step.waitForSignal() / step.wait_for_signal()

Этот метод приостанавливает выполнение запуска до тех пор, пока не будет получен определенный сигнал через наш SDK или API. Сигналы должны быть уникальными для каждого запуска и предлагают API для возобновления конкретных запусков по уникальному сигналу, в отличие от событий.

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "send-onboarding-nudge-email", triggers: { event: "app/account.created" } },
  async ({ event, step }) => {
    const onboardingCompleted = await step.waitForSignal(
      "wait-for-specific-signal",
      { signal: "task/0e7d16d1-8335-4b93-9122-76e7cc6d9eb6", timeout: "3d" }
    );
    // Сделать что-то с данными сигнала
  }
);
```

</LanguageSection>

### step.invoke()

Этот метод используется для асинхронного вызова другой функции Inngest ([написанной на любом языке SDK](/blog/cross-language-support-with-new-sdks)) и обработки результата. Вызов других функций позволяет легко повторно использовать функциональность и компоновать их для создания более сложных рабочих процессов или задач типа map-reduce.

Этот метод имеет свою собственную конфигурацию, которая позволяет определять специфические настройки, такие как ограничения параллелизма.

<LanguageSection show="typescript">

```typescript
// Функция, которую мы будем вызывать в другом месте нашего приложения
const computeSquare = inngest.createFunction(
  { id: "compute-square", triggers: { event: "calculate/square" } },
  async ({ event }) => {
    return { result: event.data.number * event.data.number }; // Результат типизирован как { result: number }
  }
);

// В этой функции мы вызовем `computeSquare`
const mainFunction = inngest.createFunction(
  { id: "main-function", triggers: { event: "main/event" } },
  async ({ step }) => {
    const square = await step.invoke("compute-square-value", {
      function: computeSquare,
      data: { number: 4 }, // входные данные типизированы
    });

    return `Квадрат 4 равен ${square.result}.`; // square.result типизирован как number
  }
);
```

</LanguageSection>

<LanguageSection show="go">

```go
import (
  "github.com/inngest/inngestgo"
  "github.com/inngest/inngestgo/errors"
  "github.com/inngest/inngestgo/step"
)

inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{
		ID: "send-delayed-email",
	},
	inngestgo.EventTrigger("app/user.signup", nil),
	// функция обработчик
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		// Вызвать другую функцию и дождаться её результата
		result, err := step.Invoke[any](
			ctx,
			"invoke-email-function",
			step.InvokeOpts{
				FunctionID: "send-welcome-email",
				// Передать данные в вызываемую функцию
				Data: map[string]any{
					"user_id": input.Event.Data["user_id"],
					"email":   input.Event.Data["email"],
				},
				// Опционально: Установить ограничение параллелизма
				Concurrency: step.ConcurrencyOpts{
					Limit: 5,
					Key:   "user-{{event.data.user_id}}",
				},
			},
		)
		if err != nil {
			return nil, err
		}
		return result, nil
	},
)
```

</LanguageSection>

### step.sendEvent() / step.send_event()

Этот метод отправляет события в Inngest для вызова функций с соответствующим событием. Используйте `sendEvent()`, когда вы хотите запустить другие функции, но вам не нужно возвращать результат. Это полезно, например, в [функциях разветвления (fan-out functions)](/docs/guides/fan-out-jobs).

<LanguageSection show="typescript">

```typescript
export default inngest.createFunction(
  { id: "user-onboarding", triggers: { event: "app/user.signup" } },
  async ({ event, step }) => {
    // Сделать что-то
    await step.sendEvent("send-activation-event", {
      name: "app/user.activated",
      data: { userId: event.data.userId },
    });
    // Сделать что-то еще
  }
);
```

</LanguageSection>

<LanguageSection show="go">

Go SDK не имеет выделенного метода `step.sendEvent()`. Используйте метод `Send()` клиента Inngest внутри `step.Run()` вместо этого.

</LanguageSection>

<LanguageSection show="python">

```python
import inngest
from src.inngest.client import inngest_client

@inngest_client.create_function(
    fn_id="my_function",
    trigger=inngest.TriggerEvent(event="app/my_function"),
)
async def fn(ctx: inngest.Context) -> list[str]:
    return await ctx.step.send_event("send", inngest.Event(name="foo"))
```
