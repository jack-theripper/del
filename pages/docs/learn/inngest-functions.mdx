# Функции Inngest

Функции Inngest позволяют разработчикам запускать надежную фоновую логику — от фоновых задач до сложных рабочих процессов.
Функция Inngest состоит из 3 основных частей, которые предоставляют мощные инструменты для повторов, планирования и координации сложных последовательностей операций:

<LanguageSection show="typescript">

```ts
inngest.createFunction({
    id: "sync-systems",
    // Функция запускается событиями
    triggers: { event: "auto/sync.request" },
    // Легко добавьте троттлинг с помощью управления потоком
    throttle: { limit: 3, period: "1min"},
  },
  async ({ step }) => {
    // шаг будет перезапущен в случае ошибки
    const data = await step.run("get-data", async () => {
      return getDataFromExternalSource();
    });

    // Шаги могут повторно использовать данные из предыдущих
    await step.run("save-data", async () => {
      return db.syncs.insertOne(data);
    });
  }
);
```

</LanguageSection>

<LanguageSection show="python">

```python
@inngest_client.create_function(
    id="sync-systems",
    # триггер (событие или cron)
    trigger=inngest.TriggerEvent(event="auto/sync.request"),
)
def sync_systems(ctx: inngest.ContextSync) -> None:
    # шаг будет перезапущен в случае ошибки
    data = ctx.step.run("Get data", get_data_from_external_source)

    # Шаги могут повторно использовать данные из предыдущих
    ctx.step.run("Save data", db.syncs.insert_one, data)
```
