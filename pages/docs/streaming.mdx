# Стриминг

В некоторых окружениях SDK позволяет передавать ответы обратно в Inngest через стриминг, что значительно увеличивает максимальные таймауты на многих serverless-платформах — до 15 минут.

Пока мы расширяем поддержку стриминга для других платформ, на данный момент поддерживаются следующие:

- [Cloudflare Workers](/docs/learn/serving-inngest-functions#framework-cloudflare-workers)
- [Express](/docs/learn/serving-inngest-functions#framework-express)
- [Next.js на Vercel Fluid Compute или Edge Functions](/docs/learn/serving-inngest-functions#framework-next-js)
- [Remix на Vercel Edge Functions](/docs/learn/serving-inngest-functions#framework-remix)

## Включение стриминга

Выберите свою платформу выше и следуйте соответствующему разделу «Стриминг», чтобы включить его для вашего приложения.

Каждый обработчик serve в Inngest предоставляет параметр `streaming`, например:

<CodeGroup>
```ts
serve({
  client: inngest,
  functions: [...fns],
  streaming: true,
});
```
</CodeGroup>

Он может принимать одно из следующих значений:

- `false` — стриминг никогда не используется. Это значение по умолчанию.
- `true` — стриминг используется, если обработчик serve его поддерживает.
