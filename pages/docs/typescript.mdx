# TypeScript

Inngest SDK использует всю мощь TypeScript, предоставляя вам отличные преимущества при работе с событиями:

- 📑 **Автодополнение**<br/>
  Нажимайте <kbd className="dark:bg-slate-800 dark:text-white shadow-none border border-slate-600">Tab ↹</kbd> и получайте выведенные типы для каждого события.
- **Мгновенная обратная связь**<br/>
  Понимайте точно, где ваш код может <span className="error-text">вызвать ошибку</span>, ещё до сохранения файла.

Всё это вместе обеспечивает великолепный вывод типов на основе ваших реальных производственных данных.

## Использование типов

После генерации типов есть несколько способов использовать их для защиты ваших функций.

### Клиент `new Inngest()`

Мы можем использовать их при создании нового клиента Inngest через `new Inngest()`.

Это даёт мощный вывод типов: автодополнение имён событий при выборе того, на что реагировать, без необходимости самостоятельно искать имя и данные.

```ts {{ filename: "inngest/client.ts" }}
import { eventType, Inngest, staticSchema } from "inngest";

type UserSignup = {
  data: {
    email: string;
    name: string;
  };
};

export const userNewSignup = eventType("user/new.signup", {
  schema: staticSchema<UserSignup>(),
});

export const inngest = new Inngest({
  id: "my-app",
});
```


```ts {{ filename: "inngest/sendWelcomeEmail.ts" }}
import { inngest, userNewSignup } from "./client";

export default inngest.createFunction(
  { id: "send-welcome-email", triggers: [userNewSignup] },
  async ({ event }) => {
    // "event" полностью типизирован для обеспечения типобезопасности внутри этой функции
    return await email.send("welcome", event.data.email);
  }
);
```

### Отправка событий

TypeScript также будет проверять правильность формата ваших пользовательских событий — подробнее см. [Формат событий](/docs/reference/events/send).

Рекомендуем хранить клиент `new Inngest()` и типы в одном файле, например `/inngest/client.ts`, чтобы использовать его везде, где отправляете события.

Вот пример отправки события в обработчике Next.js API:

```ts {{ filename: "pages/api/signup.ts" }}
import type { NextApiRequest, NextApiResponse } from "next";
import { inngest } from "../../inngest/client";

export default function handler(req: NextApiRequest, res: NextApiResponse) {
  const user = createNewUser(req.body.email, req.body.password, req.body.name);

  // TypeScript предупредит, если типы не совпадают для полезной нагрузки события
  // и свойств объекта пользователя:
  await inngest.send({
    name: "user/new.signup",
    data: {
      email: user.email,
      name: user.name,
    }
  });
  res.status(200).json({ success: true });
}
```

### Использование с `waitForEvent`

При написании пошаговых функций вы можете использовать `waitForEvent`, чтобы приостановить текущую функцию до получения другого события или истечения таймаута — в зависимости от того, что произойдёт раньше. При объявлении типов через конструктор `Inngest`, `waitForEvent` использует все заданные вами типы:

```ts {{ filename: "inngest/client.ts" }}
import { eventType, Inngest, staticSchema } from "inngest";

type UserSignup = {
  data: {
    email: string;
    user_id: string;
    name: string;
  };
};
type UserAccountSetupCompleted = {
  data: {
    user_id: string;
  };
};

export const userNewSignup = eventType("user/new.signup", {
  schema: staticSchema<UserSignup>(),
});
export const userAccountSetupCompleted = eventType("user/account.setup.completed", {
  schema: staticSchema<UserAccountSetupCompleted>(),
});

export const inngest = new Inngest({
  id: "my-app",
});
```

```ts {{ filename: "inngest/onboardingDripCampaign.ts" }}
import { inngest, userNewSignup } from "./client";

export default inngest.createFunction(
  { id: "onboarding-drip-campaign", triggers: [userNewSignup] },
  async ({ event, step }) => {
    await step.run("send-welcome-email", async () => {
      // "event" будет полностью типизирован для обеспечения типобезопасности внутри этой функции
      return await email.send("welcome", event.data.email);
    });

    // We wait up to 2 days for the user to set up their account
    const accountSetupCompleted = await step.waitForEvent(
      "wait-for-setup-complete",
      {
        event: "user/account.setup.completed",
        timeout: "2d",
        // ⬇️ Сопоставляет оба события по одному свойству
        // Поскольку оба типа событий зарегистрированы выше, это сопоставление типобезопасно
        match: "data.user_id",
      }
    );

    if (!accountSetupCompleted) {
      await step.run("send-setup-account-guide", async () => {
        return await email.send("account_setup_guide", event.data.email);
      });
    }
  }
);
```

## Вспомогательные типы

TypeScript SDK экспортирует ряд вспомогательных типов, позволяющих обращаться к внутренним типам Inngest за пределами функций Inngest.

### GetEvents <VersionBadge version="v2.0.0+" />

Получите запись всех доступных событий для заданного клиента Inngest.

Рекомендуется использовать этот тип вместо прямого переиспользования собственных типов событий, так как Inngest добавляет дополнительные свойства и внутренние события, такие как `ts` и `inngest/function.failed`.

```ts
import { type GetEvents } from "inngest";
import { inngest } from "@/inngest";

type Events = GetEvents<typeof inngest>;
```

По умолчанию возвращаемые события не включают внутренние события с префиксом
`inngest/`, например `inngest/function.finished`.

Чтобы включить эти события
<VersionBadge version="v3.13.1+" />, передайте второй дженерик `true`:

```ts
type Events = GetEvents<typeof inngest, true>;
```

### GetFunctionInput <VersionBadge version="v3.3.0+" />

Получите аргумент, передаваемый в функции Inngest, для заданного клиента Inngest и, опционально, триггера события.

Полезно для построения фабрик функций и подобных абстракций.

```ts
import { type GetFunctionInput } from "inngest";
import { inngest } from "@/inngest";

type InputArg = GetFunctionInput<typeof inngest>;
type InputArgWithTrigger = GetFunctionInput<typeof inngest, "app/user.created">;
```

### GetStepTools <VersionBadge version="v3.3.0+" />

Получите объект `step`, передаваемый в функцию Inngest, для заданного клиента Inngest и, опционально, триггера события.

Является тонкой обёрткой над `GetFunctionInput<...>["step"]`.

```ts
import { type GetStepTools } from "inngest";
import { inngest } from "@/inngest";

type StepTools = GetStepTools<typeof inngest>;
type StepToolsWithTrigger = GetStepTools<typeof inngest, "app/user.created">;
```

### Inngest.Any / InngestFunction.Any <VersionBadge version="v3.10.0+" />

Некоторые экспортируемые классы имеют тип `Any` в своём пространстве имён, который представляет любой экземпляр этого класса без вывода типов и дженериков.

Полезно для типизации списков функций или фабрик, создающих примитивы Inngest.

```ts
import { type InngestFunction } from "inngest";

const functionsToServe: InngestFunction.Any[] = [];
```
