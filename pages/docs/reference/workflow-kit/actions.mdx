# Создание действий рабочего процесса

Пакет [`@inngest/workflow-kit`](https://npmjs.com/package/@inngest/workflow-kit) предоставляет [движок рабочих процессов (workflow engine)](/docs/reference/workflow-kit/engine), позволяющий создавать действия рабочего процесса на бэкенде. Эти действия позже передаются на фронтенд, чтобы конечные пользователи могли создавать свои собственные экземпляры рабочих процессов с помощью компонента [`<Editor />`](/docs/reference/workflow-kit/components-api).

Действия рабочего процесса определяются как два объекта с использованием типов [`EngineAction`](#passing-actions-to-the-workflow-engine-engine-action) (для бэкенда) и [`PublicEngineAction`](#passing-actions-to-the-react-components-public-engine-action) (для фронтенда).

<CodeGroup>


```ts {{ title: "src/inngest/actions-definition.ts" }}
import { type PublicEngineAction } from "@inngest/workflow-kit";

export const actionsDefinition: PublicEngineAction[] = [
  {
    kind: "grammar_review",
    name: "Выполнить проверку грамматики",
    description: "Использовать OpenAI для исправления грамматики",
  },
];

```


```tsx {{ title: "src/inngest/actions.ts" }}
import { type EngineAction } from "@inngest/workflow-kit";

import { actionsDefinition } from "./actions-definition";

export const actions: EngineAction[] = [
  {
    // Добавить оглавление
    ...actionsDefinition[0],
    handler: async ({ event, step, workflowAction }) => {
	      // реализация...
    }
  },
];
```

</CodeGroup>

В приведенном выше примере массив `actionsDefinition` будет передан через пропсы в [`<Provider />`](/docs/reference/workflow-kit/components-api), в то время как `actions` передаются в [`Engine`](/docs/reference/workflow-kit/engine).

<Callout>

**Зачем мне нужны два типа действий?**

Действия необходимо разделить на 2 отдельных объекта, чтобы избежать утечки реализаций обработчиков действий и их зависимостей на фронтенд:

## Передача действий в Workflow Engine: `EngineAction[]`

<Callout>
**Примечание**: Наследует свойства `PublicEngineAction`.
</Callout>

<Properties>
  <Property name="handler" type="function">
    Handler — это ваш код, который запускается при выполнении действия. Каждый обработчик функции получает один объект в качестве аргумента, который можно деструктурировать. Ключевыми аргументами являются `event` и `step`.
  </Property>
</Properties>

```ts {{ title: "src/inngest/actions.ts" }}
import { type EngineAction } from "@inngest/workflow-kit";

import { actionsDefinition } from "./actions-definition";

export const actions: EngineAction[] = [
{
    // Добавить оглавление
    ...actionsDefinition[0],
    handler: async ({ event, step, workflow, workflowAction, state }) => {
        // ...
    }
},
];
```

Подробную информацию о **свойствах единственного аргумента** функции `handler()` можно найти ниже:

### Свойства аргумента функции `handler()`

<Properties>
  <Property name="event" type="TriggerEvent">
    См. [определение свойства аргумента `event`](/docs/reference/functions/create#event) обработчика функции Inngest.
  </Property>
  <Property name="step" type="Step">
    См. [определение свойства аргумента `step`](/docs/reference/functions/create#step) обработчика функции Inngest.
  </Property>
  <Property name="workflow" type="Workflow">
    См. [формат экземпляра Workflow](/docs/reference/workflow-kit/workflow-instance).
  </Property>
  <Property name="workflowAction" type="WorkflowAction">
     WorkflowAction — это выполняемое действие с полностью интерполированными входными данными.

     Ключевые свойства:
        - `id: string`: Идентификатор действия внутри экземпляра рабочего процесса.
        - `kind: string`: Тип действия, указанный в [`PublicEngineAction`](#passing-actions-to-the-react-components-public-engine-action).
        - `name?: string`: Название, указанное в [`PublicEngineAction`](#passing-actions-to-the-react-components-public-engine-action).
        - `description?: string`: Описание, указанное в [`PublicEngineAction`](#passing-actions-to-the-react-components-public-engine-action).
        - `inputs?: string`: Ключ записи — это имя входного параметра EngineAction, а значение — значение переменной.
  </Property>
  <Property name="state" type="object">
    State представляет текущее состояние рабочего процесса, где результаты предыдущих действий записаны в виде пар ключ-значение.
  </Property>
</Properties>
