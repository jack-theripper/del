# Экземпляр рабочего процесса

Экземпляр рабочего процесса (workflow instance) представляет собой пользовательскую конфигурацию последовательности [действий рабочего процесса](/docs/reference/workflow-kit/actions), которая позже передается в [движок рабочих процессов (workflow engine)](/docs/reference/workflow-kit/engine) для выполнения.


Пример объекта экземпляра рабочего процесса:

```json
{
  "name": "Generate social posts",
  "edges": [
    {
      "to": "1",
      "from": "$source"
    },
    {
      "to": "2",
      "from": "1"
    }
  ],
  "actions": [
    {
      "id": "1",
      "kind": "generate_tweet_posts",
      "name": "Generate Twitter posts"
    },
    {
      "id": "2",
      "kind": "generate_linkedin_posts",
      "name": "Generate LinkedIn posts"
    }
  ]
}
```

<Callout>

**Как использовать объект экземпляра рабочего процесса**

Объекты экземпляра рабочего процесса предназначены для получения из редактора [`<Provider>`](/docs/reference/workflow-kit/components-api), сохранения в базе данных и загрузки в [движок рабочих процессов (Workflow Engine)](/docs/reference/workflow-kit/engine) с помощью загрузчика (loader).

Используйте этот справочник, если вам нужно обновить экземпляр рабочего процесса между этими этапами.

</Callout>

## `Workflow`

Экземпляр Workflow представляет собой объект со следующими свойствами:

<Properties>
  <Property name="name" type="string">
     Название конфигурации рабочего процесса, предоставленное конечным пользователем.
  </Property>
  <Property name="description" type="string">
     Описание конфигурации рабочего процесса, предоставленное конечным пользователем.
  </Property>
  <Property name="actions" type="WorkflowAction[]" required>
     См. справочник [`WorkflowAction`](#workflow-action) ниже.
  </Property>
  <Property name="edges" type="WorkflowEdge[]" required>
     См. справочник [`WorkflowEdge`](#workflow-edge) ниже.
  </Property>
</Properties>


## `WorkflowAction`

`WorkflowAction` представляет собой шаг экземпляра рабочего процесса, связанный с определенным [`EngineAction`](/docs/reference/workflow-kit/actions).

<Properties>
  <Property name="id" type="string">
     Идентификатор действия внутри экземпляра рабочего процесса. Он используется в качестве ссылки и должен быть уникальным внутри самого экземпляра.
  </Property>
  <Property name="kind" type="string" required>
      Тип действия, используемый для поиска определения `EngineAction`.
  </Property>
  <Property name="name" type="string" required>
     Name — это человекочитаемое название действия.
  </Property>
  <Property name="description" type="string">
     Description — краткое описание действия.
  </Property>
  <Property name="inputs" type="object">
     Inputs — это список настроенных входных данных для EngineAction.

    Ключ записи — это имя входного параметра EngineAction, а значение — значение переменной.

    Типы будут проверены на соответствие типу EngineAction перед сохранением и перед выполнением.

    Ссылки на входные данные для интерполяции имеют формат `"!ref($.<path>)"`, например `"!ref($.event.data.email)"`
  </Property>
</Properties>

## `WorkflowEdge`

`WorkflowEdge` представляет связь между двумя `WorkflowAction`.

<Properties>
  <Property name="from" type="string" required>
     `WorkflowAction.id` исходного действия.

     `"$source"` — это зарезервированное значение, используемое в качестве отправной точки экземпляра рабочего процесса.
  </Property>
  <Property name="to" type="string" required>
     `WorkflowAction.id` следующего действия.
  </Property>
  {/* <Property name="conditional" type="object">
     `WorkflowAction.id` of the next action.
  </Property> */}
</Properties>
