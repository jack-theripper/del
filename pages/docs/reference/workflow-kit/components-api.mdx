# API компонентов (React)

Пакет [`@inngest/workflow-kit`](https://npmjs.com/package/@inngest/workflow-kit) предоставляет набор React-компонентов, позволяющих в кратчайшие сроки создать интерфейс редактора рабочих процессов!

![workflow-kit-announcement-video-loop.gif](/assets/docs/reference/workflow-kit/workflow-demo.gif)

## Использование

```tsx {{ title: "src/components/my-workflow-editor.ts" }}
import { useState } from "react";
import { Editor, Provider, Sidebar, type Workflow } from "@inngest/workflow-kit/ui";

// импорт `PublicEngineAction[]`
import { actionsDefinitions } from "@/inngest/actions-definitions";

// ПРИМЕЧАНИЕ - Импорт CSS из JavaScript требует плагина сборщика, такого как PostCSS или CSS-модули
import "@inngest/workflow-kit/ui/ui.css";
import "@xyflow/react/dist/style.css";

export const MyWorkflowEditor = ({ workflow }: { workflow: Workflow }) => {
  const [workflowDraft, updateWorkflowDraft] =
    useState<typeof workflow>(workflow);

  return (
    <Provider
      workflow={workflowDraft}
      trigger={{ event: { name: 'blog-post.updated' } }}
      availableActions={actionsDefinitions}
      onChange={updateWorkflowDraft}
    >
      <Editor>
        <Sidebar position="right"></Sidebar>
      </Editor>
    </Provider>
  );
};
```


## Справочник

### `<Provider>`

<Callout>

`<Provider>` — это [контролируемый компонент (Controlled Component)](https://react.dev/learn/sharing-state-between-components#controlled-and-uncontrolled-components), который следит за обновлением `workflow={}`.

Убедитесь, что `workflow={}` обновляется на основе данных, полученных через `onChange={}`.

</Callout>


<Properties>
  <Property name="workflow" type="Workflow" required>
    [Объект экземпляра Workflow](/docs/reference/workflow-kit/workflow-instance).
  </Property>
  <Property name="trigger" type="object" required>
    Объект со свойством `name: string`, [представляющим имя события](/docs/reference/functions/create#trigger).
  </Property>
  <Property name="availableActions" type="PublicEngineAction[]">
    См. [справочник по `PublicEngineActionEngineAction[]`](/docs/reference/workflow-kit/actions#passing-actions-to-the-react-components-public-engine-action).
  </Property>
  <Property name="onChange" type="function" required>
    Функция обратного вызова, вызываемая после каждого изменения `workflow`.
  </Property>
  <Property name="{children}" type="React.ReactNode" required>
    Компонент `<Provider>` всегда должен получать в качестве дочерних элементов следующее дерево:
  </Property>
</Properties>

    ```tsx
    <Editor>
      <Sidebar position="right"></Sidebar>
    </Editor>
    ```
