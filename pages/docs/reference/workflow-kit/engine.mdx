# Использование движка рабочих процессов

Движок рабочих процессов `Engine` используется для запуска заданного [экземпляра рабочего процесса](/docs/reference/workflow-kit/workflow-instance) внутри функции Inngest:

```tsx {{ title: "src/inngest/workflow.ts" }}
import { Engine, type Workflow } from "@inngest/workflow-kit";

import { inngest } from "./client";
import { actions } from "./actions";
import { loadWorkflowInstanceFromEvent } from "./loaders";

const workflowEngine = new Engine({
  actions: actionsWithHandlers,
  loader: (event) => {
    return loadWorkflowInstanceFromEvent(event);
  },
});

export default inngest.createFunction(
  { id: "blog-post-workflow", triggers: { event: "blog-post.updated" } },
  async ({ event, step }) => {
    // При вызове `run`,
    // вызывается функция loader с доступом к событию
    await workflowEngine.run({ event, step });
  }
);

```

## Настройка


<Properties>
  <Property name="actions" type="EngineAction[]">
    См. [справочник по `EngineAction[]`](/docs/reference/workflow-kit/actions#passing-actions-to-the-workflow-engine-engine-action).
  </Property>
  <Property name="loader" type="function">
    Асинхронная функция, получающая [`event`](/docs/reference/functions/create#event) в качестве единственного аргумента и возвращающая валидный объект [экземпляра `Workflow`](/docs/reference/workflow-kit/workflow-instance).
  </Property>
  <Property name="disableBuiltinActions" type="boolean">
    Для выборочного добавления встроенных действий установите это значение в `true` и предоставьте нужные действия через проп `availableActions` компонента [`<Provider>`](/docs/reference/workflow-kit/components-api).
  </Property>
</Properties>
