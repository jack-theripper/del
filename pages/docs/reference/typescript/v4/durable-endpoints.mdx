# Durable Endpoints <VersionBadge version="Beta" />


Создавайте надёжные HTTP-эндпоинты с помощью `inngest.endpoint()`. Каждый шаг в обработчике создаёт чекпоинт, обеспечивая автоматическое восстановление после сбоев.

```ts
import { Inngest, step } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({ id: "my-app", endpointAdapter });

export const handler = inngest.endpoint(async (req: Request): Promise<Response> => {
  const data = await step.run("fetch-data", async () => {
    return await fetchExternalAPI();
  });

  return Response.json({ data });
});
```

---

## Настройка

### `endpointAdapter`

`endpointAdapter` должен быть передан в конструктор клиента Inngest для включения Durable Endpoints.

```ts
import { Inngest } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({
  id: "my-app",
  endpointAdapter,
});
```

<Callout>
  `endpointAdapter` импортируется из `inngest/edge` (или любой совместимой
  точки входа фреймворка) и обязателен для работы Durable Endpoints. Без него
  `inngest.endpoint()` будет недоступен.
</Callout>

## Доступные методы шагов

В обработчике Durable Endpoint доступны все методы шагов: смотрите [Доступные методы шагов](/docs/learn/inngest-steps#available-step-methods).

Например:

### `step.run(id, fn)`

Выполнение и чекпоинтинг функции. При повторной попытке эндпоинта завершённые шаги мгновенно возвращают кэшированный результат.

```ts
const user = await step.run("fetch-user", async () => {
  return await db.users.findOne({ id: userId });
});
```

Смотрите [справочник по step.run()](/docs/reference/typescript/v4/functions/step-run) для полной документации.

### `step.sleep(id, duration)`

Приостановка выполнения на заданную длительность. Эндпоинт будет возобновлён после завершения паузы.

```ts
await step.sleep("rate-limit-pause", "30s");
```

Смотрите [справочник по step.sleep()](/docs/reference/typescript/v4/functions/step-sleep) для полной документации.

### `step.waitForEvent(id, options)`

Ожидание внешнего события перед продолжением. Полезно для рабочих процессов с участием человека.

```ts
const approval = await step.waitForEvent("wait-for-approval", {
  event: "approval/received",
  match: "data.requestId",
  timeout: "24h",
});
```

Смотрите [справочник по step.waitForEvent()](/docs/reference/typescript/v4/functions/step-wait-for-event) для полной документации.

## Передача данных в эндпоинты

<Callout variant="warning">
  **Тело POST-запроса пока не поддерживается.** Используйте параметры строки запроса для передачи данных в Durable Endpoints. Поддержка тела POST будет добавлена в ближайшее время.
</Callout>

```ts
export const handler = inngest.endpoint(async (req: Request): Promise<Response> => {
  const url = new URL(req.url);

  // Читать данные из параметров запроса
  const userId = url.searchParams.get("userId");
  const action = url.searchParams.get("action");

  // Обработать с надёжными шагами
  const result = await step.run("process", async () => {
    return await processAction(userId, action);
  });

  return Response.json({ result });
});
```

## Возврат ответов

Любой способ возврата данных в вашем фреймворке совместим с Durable Endpoints.

Например, с использованием стандартного Web API:

```ts
// JSON-ответ
return Response.json({ success: true, data: result });

// Текстовый ответ
return new Response("OK", { status: 200 });

// Ответ с ошибкой
return new Response(JSON.stringify({ error: "Not found" }), {
  status: 404,
  headers: { "Content-Type": "application/json" },
});
```

## Обработка ошибок

Ошибки, выброшенные внутри `step.run()`, вызывают автоматические повторные попытки. Используйте стандартный try/catch для пользовательской обработки ошибок:

```ts
export const handler = inngest.endpoint(async (req: Request): Promise<Response> => {
  try {
    const result = await step.run("risky-operation", async () => {
      return await riskyAPICall();
    });

    return Response.json({ result });
  } catch (error) {
    // Все повторные попытки исчерпаны, обработать корректно
    return Response.json(
      { error: "Operation failed after retries" },
      { status: 500 }
    );
  }
});
```

## Запрос к Durable Endpoint

Durable Endpoints ведут себя как обычные API-эндпоинты при успешном выполнении. Их можно запрашивать из фронтенда (_или бэкенда_) с помощью `fetch()` или любой другой HTTP-библиотеки.

При сбое с повторными попытками или использовании длительных шагов вроде `step.waitForEvent()`
Durable Endpoint перенаправляет на отдельный эндпоинт, ожидающий завершения вызова.

По умолчанию это эндпоинт либо в Inngest Dev Server, либо в Inngest Cloud
(в зависимости от вашей среды), но `inngest.endpointProxy()`
можно использовать для создания собственного URL-адреса, удовлетворяющего ограничениям CORS при
использовании эндпоинта из браузеров.

```typescript
// При установке `endpointAdapter` используйте `.withOptions()` для дополнительных настроек
const inngest = new Inngest({
  id: "my-app",
  endpointAdapter: endpointAdapter.withOptions({
    asyncRedirectUrl: "/wait",
  }),
});

// Затем создайте маршрут с `inngest.endpointProxy()`
Bun.serve({
  port: 3000,
  routes: {
    "/process": ...,
    "/wait": inngest.endpointProxy(),
  },
});
```

Запросы теперь будут перенаправляться на `/wait`.
