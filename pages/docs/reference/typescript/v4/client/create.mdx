# Создание клиента Inngest


Объект клиента `Inngest` используется для настройки вашего приложения, позволяя создавать функции и отправлять события.

<CodeGroup>
```ts
import { Inngest } from "inngest";

const inngest = new Inngest({
  id: "my-application",
});

```
</CodeGroup>

---

## Конфигурация

<Properties>
  <Property name="id" type="string" required>
    Уникальный идентификатор вашего приложения. Рекомендуем использовать slug через дефис.
  </Property>
  <Property name="baseUrl" type="string">
    Переопределяет базовый URL по умолчанию (`https://inn.gs/`) для отправки событий. Смотрите также переменную окружения [`INNGEST_BASE_URL`](/docs/sdk/environment-variables#inngest-base-url).
  </Property>
    <Property name="env" type="string">
    Название среды. Требуется только при использовании [Branch Environments](/docs/platform/environments).
  </Property>
  <Property name="eventKey" type="string">
    [Ключ события](/docs/events/creating-an-event-key) Inngest. Альтернативно, установите переменную окружения [`INNGEST_EVENT_KEY`](/docs/sdk/environment-variables#inngest-event-key).
  </Property>
  <Property name="fetch" type="Fetch API compatible interface">
    Переопределяет стандартную реализацию
    [`fetch`](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API).
    По умолчанию используется нативный Fetch API среды выполнения.

    Если нужно указать это, убедитесь, что вы сохраняете [привязку](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_objects/Function/bind) функции, используя `.bind` или обернув её в анонимную функцию.
  </Property>
  <Property name="signingKey" type="string">
    Ключ подписи для аутентификации в Inngest Cloud. Рекомендуем устанавливать через переменную окружения [`INNGEST_SIGNING_KEY`](/docs/sdk/environment-variables#inngest-signing-key).
  </Property>
  <Property name="signingKeyFallback" type="string">
    Резервный ключ подписи, полезен при ротации ключей. Рекомендуем устанавливать через переменную окружения [`INNGEST_SIGNING_KEY_FALLBACK`](/docs/sdk/environment-variables#inngest-signing-key-fallback).
  </Property>
  <Property name="isDev" type="boolean">
    Установите `true` для принудительного включения Dev-режима — будут использоваться локальные URL по умолчанию и отключена проверка подписей; `false` для принудительного включения Cloud-режима. Альтернативно, установите [`INNGEST_DEV`](/docs/sdk/environment-variables#inngest-dev).
  </Property>
  <Property name="logger" type="Logger">
    Объект логгера с методами `.info()`, `.warn()`, `.error()` и `.debug()`. SDK использует логирование в стиле Pino (объект перед сообщением). Для логгеров, ставящих сообщение первым (например, Winston), используйте `wrapStringFirstLogger`. По умолчанию `new ConsoleLogger({ level: "info" })`. Подробнее в [справочнике по логированию](/docs/reference/typescript/v4/logging).

    ```ts
    import { ConsoleLogger, Inngest } from "inngest";

    export const inngest = new Inngest({
      id: "my-app",
      logger: new ConsoleLogger({ level: "debug" }),
    });
    ```
  </Property>
  <Property name="internalLogger" type="Logger">
    Отдельный логгер для внутренних сообщений SDK (регистрация, обработка запросов, ошибки промежуточного ПО). Если не указан, используется `logger`. Позволяет направлять внутренние логи SDK отдельно от логов функций. Подробнее в [справочнике по логированию](/docs/reference/typescript/v4/logging#internal-logger).
  </Property>
  <Property name="middleware" type="array">
    Стек [промежуточного ПО](/docs/features/middleware) для добавления к клиенту.
  </Property>
</Properties>

<Callout>
Рекомендуем устанавливать [`INNGEST_EVENT_KEY`](/docs/sdk/environment-variables#inngest-event-key) как переменную окружения, а не использовать опцию `eventKey`. Как и любой секрет, ключ события не стоит хранить в коде.
</Callout>

## Cloud Mode и Dev Mode

SDK может работать в двух режимах: **Cloud** или **Dev**.

- **Cloud Mode**
  - 🔒 Проверка подписей **ВКЛЮЧЕНА**
  - По умолчанию взаимодействует с Inngest Cloud (например, `https://api.inngest.com`)
- **Dev Mode**
  - ❌ Проверка подписей **ВЫКЛЮЧЕНА**
  - По умолчанию взаимодействует с Inngest Dev Server (например, `http://localhost:8288`)

Принудительно включить Dev или Cloud Mode можно через
[`INNGEST_DEV`](/docs/sdk/environment-variables#inngest-dev) или опцию
[`isDev`](#configuration).

Если режим не задан явно, SDK по умолчанию использует **cloud mode** для большей безопасности. Используйте `INNGEST_DEV` или `isDev=true`, чтобы указать SDK, что вы намеренно используете режим разработки.

## Лучшие практики

### Используйте один клиент для всего кода

Создание клиента `Inngest` в одном файле и его использование во всей кодовой базе — оптимальный подход: единственное место для настройки клиента и определения типов, доступных везде, где отправляются события или создаются функции.

<CodeGroup filename="./inngest/client.ts">
```ts
import { Inngest } from "inngest";

export const inngest = new Inngest({ id: "my-app" });

```

</CodeGroup>

```ts  {{ filename: './inngest/myFunction.ts' }}
import { inngest } from "./client";

export default inngest.createFunction(...);
```

### Работа с несколькими средами через промежуточное ПО

Если ваш клиент использует промежуточное ПО, оно может импортировать зависимости, не поддерживаемые сразу в нескольких средах, таких как "Edge" и "Serverless" (с разным доступом к WebAPI или Node).

В этом случае рекомендуем создавать отдельный клиент для каждой среды, гарантируя, что Node-совместимое промежуточное ПО используется только в Node-совместимых средах и наоборот.

Это распространено в случаях, когда при выполнении функций нужно более сложное промежуточное ПО, а отправка событий из edge часто требует значительно меньшего.

<CodeGroup filename="./inngest/client.ts">
```ts
// inngest/client.ts
import { Inngest } from "inngest";
import { nodeMiddleware } from "some-node-middleware";

export const inngest = new Inngest({
  id: "my-app",
  middleware: [nodeMiddleware],
});

// inngest/edgeClient.ts
import { Inngest } from "inngest";

export const inngest = new Inngest({
  id: "my-app-edge",
});

```

</CodeGroup>

Смотрите также [Ссылки на функции](/docs/functions/references), которые помогут вызывать функции в разных средах без лишних зависимостей.
