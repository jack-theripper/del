# Стриминг

В отдельных окружениях SDK позволяет передавать потоковые ответы обратно в Inngest, значительно увеличивая максимальные тайм-ауты на многих serverless-платформах до 15 минут.

Пока мы расширяем поддержку стриминга для других платформ, в настоящее время поддерживаются следующие:

- [Cloudflare Workers](/docs/learn/serving-inngest-functions#framework-cloudflare-workers)
- [Express](/docs/learn/serving-inngest-functions#framework-express)
- [Next.js на Vercel Fluid Compute или Edge Functions](/docs/learn/serving-inngest-functions#framework-next-js)
- [Remix на Vercel Edge Functions](/docs/learn/serving-inngest-functions#framework-remix)

## Включение стриминга

Выберите платформу выше и следуйте соответствующему разделу "Streaming" для включения стриминга в вашем приложении.

Каждый обработчик serve Inngest предоставляет опцию `streaming`, например:

<CodeGroup>
```ts
serve({
  client: inngest,
  functions: [...fns],
  streaming: true,
});
```
</CodeGroup>

Значение может быть одним из следующих:

- `false` — Стриминг никогда не будет использован. Это значение по умолчанию.
- `true` — Стриминг будет использован. Если обработчик serve не поддерживает стриминг, будет выброшена ошибка.

<Callout>
  В v3 стриминг принимал строковые значения `"allow"` и `"force"`. В v4 они упрощены до `true | false`. Смотрите [руководство по миграции](/docs/reference/typescript/v4/migrations/v3-to-v4#simplify-streaming-option) для подробностей.
</Callout>
