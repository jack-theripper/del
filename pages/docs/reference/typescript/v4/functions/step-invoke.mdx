# Invoke


Используйте `step.invoke()` для асинхронного вызова другой функции и обработки результата. Вызов других функций позволяет легко переиспользовать функциональность и компоновать её для создания более сложных рабочих процессов или заданий типа map-reduce. `step.invoke()` возвращает `Promise`, разрешающийся в возвращаемое значение вызванной функции.

```ts
// Некоторая функция, которую мы вызовем
const computeSquare = inngest.createFunction(
  {
    id: "compute-square",
    triggers: { event: "calculate/square" },
  },
  async ({ event }) => {
    return { result: event.data.number * event.data.number }; // Результат типизирован как { result: number }
  }
);

// В этой функции мы вызываем `computeSquare`
const mainFunction = inngest.createFunction(
  {
    id: "main-function",
    triggers: { event: "main/event" },
  },
  async ({ step }) => {
    const square = await step.invoke("compute-square-value", {
      function: computeSquare,
      data: { number: 4 }, // входные данные типизированы
    });

    return `Square of 4 is ${square.result}.`; // square.result типизирован как number
  }
);
```

## Как вызывать `step.invoke()`

Работа с `step.invoke()` аналогична работе с любым другим Promise в JavaScript:

```ts
// Используя ключевое слово "await"
const result = await step.invoke("invoke-function", {
  function: someInngestFn,
  data: { ... },
});

// Используя `then` для цепочки
step
  .invoke("invoke-function", { function: someInngestFn, data: { ... } })
  .then((result) => {
    // дальнейшая обработка
  });

// Параллельный запуск нескольких вызовов
Promise.all([
  step.invoke("invoke-first-function", {
    function: firstFunctionReference,
    data: { ... },
  }),
  step.invoke("invoke-second-function", {
    function: secondFn,
    data: { ... },
  }),
]);
```

## Использование ссылок на функции

Вместо прямого импорта локальной функции для вызова можно использовать [`referenceFunction()`](/docs/functions/references) для вызова функций Inngest из другого приложения или для избежания импорта зависимостей функции в рамках одного приложения.

```ts
import { referenceFunction } from "inngest";
import { type computePi } from "@/inngest/computePi";

// Создать локальную ссылку на функцию без импорта зависимостей
const computePi = referenceFunction<typeof computePi>({
  functionId: "compute-pi",
});

// Создать ссылку на функцию в другом приложении
const computeSquare = referenceFunction({
  appId: "my-python-app",
  functionId: "compute-square",
});

// square.result типизирован как number
const square = await step.invoke("compute-square-value", {
  function: computePi,
  data: { number: 4 }, // входные данные типизированы
});
```

Смотрите [Ссылки на функции](/docs/functions/references) для подробностей.

## Когда использовать `step.invoke()`

Использование `step.invoke()` для прямого вызова функции Inngest больше похоже на традиционный RPC, чем на обычный событийно-ориентированный подход Inngest. Хотя этот инструмент всё равно использует события за кулисами, его можно применять для разбиения кодовой базы на переиспользуемые рабочие процессы, вызываемые откуда угодно.

Используйте `step.invoke()` для задач, требующих специфических настроек, например лимитов параллелизма. Поскольку выполнение происходит с собственной конфигурацией, отдельной от вызывающей, можно настраивать конфигурацию для каждой функции индивидуально.

Если не нужна детальная настройка или функция не будет переиспользоваться между приложениями, используйте `step.run()` для простоты.

## Внутреннее поведение

При передаче объекта функции в качестве аргумента SDK автоматически получает ID функции. Для вызова между приложениями используйте [`referenceFunction()`](/docs/functions/references) для создания типизированной ссылки. Смотрите [Обработку ошибок](#error-handling) для подробностей.

При получении запроса на вызов функции Inngest выполняет её и ожидает события `inngest/function.finished`, которое используется для выполнения промиса с данными (или ошибкой) для шага.

## Возвращаемые значения и сериализация

Аналогично `step.run()`, все данные, возвращаемые из `step.invoke()`, сериализуются как JSON для обеспечения возможности SDK вернуть допустимый сериализованный ответ в сервис Inngest.

## Таймаут

Если явно не настроен, таймаут `step.invoke` по умолчанию составляет 1 год.

## Повторные попытки

Вызванная функция будет выполнена как обычная функция Inngest: у неё будет собственный набор повторных попыток, и она рассматривается как новый запуск.

Если `step.invoke()` завершается ошибкой по любой из перечисленных ниже причин, будет выброшен `NonRetriableError`. Это необходимо для предотвращения каскадных повторных попыток: например, если A вызывает B, которая вызывает C, которая вызывает D, при сбое D будет запущена 27 раз (`retryCount^n`).

В будущем это может измениться — [дайте нам знать](https://roadmap.inngest.com/roadmap?ref=docs), если хотите изменить это поведение.

## Обработка ошибок

### Функция не найдена

Если Inngest не удалось найти функцию для вызова по указанному ID (см. [Внутреннее поведение](#internal-behaviour)), будет отправлено событие `inngest/function.finished` с соответствующей ошибкой, и шаг завершится с `NonRetriableError`.

### Вызванная функция завершается ошибкой

Если функция исчерпывает все повторные попытки и завершается ошибкой, будет отправлено событие `inngest/function.finished` с соответствующей ошибкой, и шаг завершится с `NonRetriableError`.

### Истёк таймаут вызванной функции

Если достигнут `timeout` и вызванная функция всё ещё выполняется, шаг завершится с `NonRetriableError`.

### Вызванная функция ограничена по частоте

Если вызванная функция имеет конфигурацию ограничения частоты и была пропущена, шаг завершится с `NonRetriableError`. Рекомендуется оборачивать `step.invoke` в `try catch`, если вызванная функция может иногда пропускаться.

### Вызванная функция дебаунсирована

Если вызванная функция имеет конфигурацию дебаунса и была пропущена, шаг завершится с `NonRetriableError` после истечения таймаута. Предпочтительно всегда устанавливать значимый таймаут при вызове функции с конфигурацией дебаунса.
