# Эксклюзивное выполнение функции


Обеспечьте, чтобы только один запуск функции (_или набора конкретных функций на основе определённых свойств события_) выполнялся одновременно.

Смотрите [руководство по функциям-одиночкам](/docs/guides/singleton) для получения дополнительной информации о работе этой функции.

```ts
export default inngest.createFunction(
  {
    id: "data-sync",
    triggers: { event: "data-sync.start" },
    singleton: {
      key: "event.data.user_id",
      mode: "skip",
    },
  },
  async ({ event }) => {
    // Эта функция будет пропущена, если уже выполняется другой запуск той же функции для того же пользователя
  }
);
```

## Конфигурация

<Properties>
  <Property name="singleton" type="object">
    Параметры для настройки эксклюзивного выполнения функции.

    <Properties nested={true}>
      <Property name="key" type="string" required>
        Уникальное ключевое выражение для применения лимита. Выражение вычисляется для каждого триггерного события.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * Эксклюзивное выполнение по ID клиента: `'event.data.customer_id'`
        * Эксклюзивное выполнение по аккаунту и email: `'event.data.account_id + "-" + event.user.email'`
      </Property>
      <Property name="mode" type="string" required>
        Режим для функции-одиночки:

        * `"skip"`: Пропустить новый запуск.
        * `"cancel"`: Отменить существующий запуск и запустить новый.
      </Property>
    </Properties>
  </Property>
</Properties>

## Примеры

### Выполнение только при последнем событии

В этом примере активный запуск функции `data-sync` будет отменён, если получено другое событие с тем же `user_id`:

```ts
// Пример полезной нагрузки события:
// {
//   name: "data-sync.start",
//   data: {
//     user_id: "123456789",
//   }
// }
export default inngest.createFunction(
  {
    id: "data-sync",
    triggers: { event: "data-sync.start" },
    singleton: {
      key: "event.data.user_id",
      mode: "cancel",
    },
  },
  async ({ event, step }) => {
    const company = await step.run(
      "fetch-latest-data-from-source",
      async () => {
        return await client.fetchData(event.data.user_id);
      }
    );

    await step.run("update-data-in-database", async () => {
      return await database.upsert({ id: company.id }, company);
    });
  }
);
```

Хотя это похоже на [дебаунс](/docs/guides/debounce), функции-одиночки предназначены для обеспечения выполнения только одного запуска функции в данный момент, тогда как дебаунс гарантирует обработку только одного события в заданном временном окне.

Обратитесь к [руководству по функциям-одиночкам](/docs/guides/singleton) для получения дополнительной информации.
