# Run


Используйте `step.run()` для выполнения синхронного или асинхронного кода как повторяемого шага в вашей функции. `step.run()` возвращает [Promise](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise), разрешающийся в возвращаемое значение вашего обработчика.

```ts
export default inngest.createFunction(
  {
    id: "import-product-images",
    triggers: { event: "shop/product.imported" },
  },
  async ({ event, step }) => {
    const uploadedImageURLs = await step.run("copy-images-to-s3", async () => {
      return copyAllImagesToS3(event.data.imageURLs);
    });
  }
);
```

## Как вызывать `step.run()`

Поскольку `step.run()` возвращает [Promise](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise), его нужно обрабатывать как любой другой Promise в JavaScript. Вот несколько способов использования `step.run()`:

```ts
// Используйте ключевое слово "await" для ожидания выполнения промиса
await step.run("create-user", () => {/* ... */});
const user = await step.run("create-user", () => {/* ... */});

// Используйте `then` (или аналоги)
step.run("create-user", () => {/* ... */})
  .then((user) => {
    // сделать что-то ещё
  });

// Используйте с хелперами Promise для параллельного выполнения
Promise.all([
  step.run("create-subscription", () => {/* ... */}),
  step.run("add-to-crm", () => {/* ... */}),
  step.run("send-welcome-email", () => {/* ... */}),
]);
```

## Повторные попытки

Каждый вызов `step.run()` имеет собственный независимый счётчик попыток. При возникновении ошибки шаг будет повторён согласно конфигурации повторных попыток функции. Конфигурация повторных попыток применяется к каждому отдельному шагу, а не как общий пул для всех шагов функции.

Например, если функция настроена с `retries: 4`, каждый `step.run()` будет повторяться до 4 раз независимо (5 попыток всего включая первую). При наличии нескольких шагов каждый шаг получает полный набор повторных попыток.

Подробнее о [настройке повторных попыток](/docs/features/inngest-functions/error-retries/retries).

## Возвращаемые значения и сериализация

Все данные, возвращаемые из `step.run`, сериализуются как JSON для обеспечения возможности SDK вернуть допустимый сериализованный ответ в сервис Inngest.

```ts
const output = await step.run("create-user", () => {
  return { id: new ObjectId(), createdAt: new Date() };
});
/*
{
  "id": "647731d1759aa55be43b975d",
  "createdAt": "2023-05-31T11:39:18.097Z"
}
*/
```
