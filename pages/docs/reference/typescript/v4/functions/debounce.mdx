# Дебаунс функций


Дебаунс откладывает запуск функции на заданный `period` и перезапускает таймер при получении новых событий в период активного дебаунса. Функция запускается после истечения указанного `period`, в течение которого не было получено новых событий. Функции используют последнее полученное событие в качестве входных данных.

Смотрите [руководство по дебаунсу](/docs/guides/debounce) для получения дополнительной информации о работе этой функции.

```ts
export default inngest.createFunction(
  {
    id: "handle-webhook",
    debounce: {
      key: "event.data.account_id",
      period: "5m",
    },
    triggers: { event: "intercom/company.updated" },
  },
  async ({ event, step }) => {
    // Эта функция будет запланирована только через 5 минут после прекращения
    // получения событий с одинаковым полем `event.data.account_id`.
    //
    // `event` будет последним событием в полученной серии.
  }
);
```

<Properties>
  <Property name="debounce" type="object">
    Параметры для настройки дебаунса выполнения функции

    <Properties nested={true}>
      <Property name="period" type="string" required>
        Задержка выполнения. Период начинается при получении первого соответствующего события.

        Допустимые значения: от `1s` до `7d` (`168h`).
      </Property>
      <Property name="key" type="string">
        Опциональное уникальное ключевое выражение для применения лимита. Выражение вычисляется для каждого триггерного события и позволяет применять дебаунс к данным события.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * Дебаунс по ID клиента: `'event.data.customer_id'`
        * Дебаунс по аккаунту и email: `'event.data.account_id + "-" + event.user.email'`
      </Property>
      <Property name="timeout" type="string">
        Максимальное время, на которое можно продлить дебаунс перед запуском.
      </Property>
    </Properties>
  </Property>
</Properties>

<Callout>
  Функции будут запущены с использованием последнего полученного события в качестве входных данных.
</Callout>

<Callout variant="warning">
  Дебаунс нельзя комбинировать с [батчингом](/docs/guides/batching).
</Callout>
