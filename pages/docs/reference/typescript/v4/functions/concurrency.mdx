# Управление параллелизмом

Ограничьте количество параллельно выполняемых шагов функции с помощью параметров конфигурации [`concurrency`](/docs/reference/typescript/v4/functions/create#configuration). Установка опционального параметра `key` ограничивает параллелизм для каждого уникального значения выражения.

<Callout>
**Важно:** Лимиты параллелизма ограничивают выполнение _шагов_, а не общее количество запусков функций. Функции, которые спят, ожидают событий или приостановлены между шагами, **не** учитываются в лимите параллелизма. Это означает, что одновременно может выполняться значительно больше запусков функций, чем ваш лимит параллелизма. [Подробнее о том, как работает параллелизм →](/docs/guides/concurrency#how-concurrency-works)
</Callout>

[Читайте наше руководство по параллелизму для получения дополнительной информации о том, как он работает и каковы ограничения](/docs/guides/concurrency).

<CodeGroup>
```ts {{ title: "Simple" }}
export default inngest.createFunction(
  {
    id: "sync-contacts",
    triggers: { event: "app/user.created" },
    concurrency: {
      limit: 10,
    },
  },
  async ({ event, step }) => {
    // ...
  }
);
```
```ts {{ title: "Multiple keys" }}
inngest.createFunction(
  {
    id: "unique-function-id",
    triggers: { event: "ai/summary.requested" },
    concurrency: [
      {
         // Использовать лимит параллелизма на уровне аккаунта для этой функции,
         // используя ключ "openai" как виртуальную очередь. Любая другая функция,
         // выполняющаяся с тем же ключом "openai", учитывается в этом лимите.
         scope: "account",
         key: `"openai"`,
         // Если 10 шагов выполняются с ключом "openai", запуски этой функции
         // будут ждать доступной мощности.
         limit: 10,
      },
      {
         // Создать ещё одну виртуальную очередь параллелизма только для этой функции.
         // Ограничивает каждый аккаунт одним выполняющимся шагом на основе поля
         // event.data.account_id.
         // "fn" — область по умолчанию, поэтому это поле можно опустить.
         scope: "fn",
         key: "event.data.account_id",
         limit: 1,
      },
    ],
  },
  async ({ event, step }) => {
    // ...
  }
);
```
</CodeGroup>

Установка лимитов `concurrency` очень полезна для:

* Управления ограничениями API — ограничивайте параллелизм, чтобы не выходить за рамки квот стороннего API.
* Ограничения операций или соединений с базой данных
* Предотвращения избыточного потребления ресурсов одним аккаунтом пользователя (см. `key`)

Если вы хотите ограничить количество запусков функции за заданный период, [опция `rateLimit`](/docs/reference/typescript/v4/functions/rate-limit) может лучше подойти для вашего случая.


## Конфигурация

<Properties>
  <Property name="concurrency" type="number | object | [object, object]">
    Параметры для настройки параллелизма. Указание `number` является сокращением для установки свойства `limit`.

    <Properties nested={true}>
      <Property name="limit" type="number" required>
        Максимальное количество параллельно выполняемых шагов.
        Значение `0` или `undefined` эквивалентно отсутствию лимита.
        Максимальное значение определяется тарифным планом вашего аккаунта.
      </Property>
      <Property name="scope" type="'account' | 'env' | 'fn'">
        Область применения лимита параллелизма, влияющая на то, управляется ли параллелизм для отдельной функции, в пределах среды или всего аккаунта.

        * `fn` (по умолчанию): только запуски этой функции влияют на лимит параллелизма
        * `env`: все запуски в одной среде с одинаковым значением ключа влияют на лимит параллелизма. Требует установки `key`.
        * `account`: каждый запуск с одинаковым значением ключа влияет на лимит параллелизма во всех средах. Требует установки `key`.
      </Property>
      <Property name="key" type="string">
        Выражение, вычисляемое в строку на основе триггерного события. Возвращённая строка используется как имя очереди параллелизма. Ключ обязателен при установке области `env` или `account`.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * Ограничить параллелизм до `n` (через `limit`) по ID клиента: `'event.data.customer_id'`
        * Ограничить параллелизм до `n` по пользователю и ID импорта: `'event.data.user_id + "-" + event.data.import_id'`
        * Глобальное ограничение через конкретную строку: `'"global-quoted-key"'` (в кавычках, так как выражение вычисляется как код)
      </Property>
    </Properties>
  </Property>
</Properties>

<Callout>
**Параллелизм на уровне шагов:** Опция параллелизма управляет количеством параллельно _выполняемых шагов_, а не количеством запусков функций.

Поскольку запуски функций часто приостанавливаются (для сна, ожидания событий или между шагами), обычно одновременно выполняется значительно больше запусков, чем ваш лимит параллелизма. Однако только настроенное количество шагов будет _активно выполняться_ в любой момент времени.

Например, при `concurrency: 10` у вас может быть 500 запусков функций в процессе, но только 10 шагов выполняют код в данный момент.
</Callout>
