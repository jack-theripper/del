# Ограничение частоты выполнения функций


Установите _жёсткий лимит_ на количество запусков функций в течение периода времени. События, превышающие ограничение частоты, _пропускаются_ и не запускают функции.

Смотрите [руководство по ограничению частоты](/docs/guides/rate-limiting) для получения дополнительной информации о работе этой функции.

```ts
export default inngest.createFunction(
  {
    id: "synchronize-data",
    triggers: { event: "intercom/company.updated" },
    rateLimit: {
      key: "event.data.company_id",
      limit: 1,
      period: "4h",
    },
  },
  async ({ event, step }) => {
    // Эта функция будет ограничена по частоте
    // Она будет запускаться не более 1 раза в 4 часа для данного event.data.company_id
  }
);
```

## Конфигурация

<Properties>
  <Property name="rateLimit" type="object">
    Параметры для настройки ограничения частоты выполнения функций

    <Properties nested={true}>
      <Property name="limit" type="number" required>
        Максимальное количество функций, запускаемых за заданный период времени.
      </Property>
      <Property name="period" type="string" required>
        Период времени для применения лимита. Период начинается при получении первого соответствующего события.
        Допустимые значения: от `1s` до `24h`.
      </Property>
      <Property name="key" type="string">
        Уникальное ключевое выражение для применения лимита. Выражение вычисляется для каждого триггерного события.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * Ограничение по ID клиента: `'event.data.customer_id'`
        * Ограничение по аккаунту и email: `'event.data.account_id + "-" + event.user.email'`
      </Property>
    </Properties>
  </Property>
</Properties>

## Примеры

### Ограничение синхронизации, запущенной webhook-событиями

В этом примере мы используем события из webhook Intercom. Webhook может быть избыточно активным и отправлять несколько событий `intercom/company.updated` за короткое время. Кроме того, нам нужна синхронизация данных пользователя из Intercom не более 4 раз в день, поэтому устанавливаем лимит `6h`:

```ts
/** Пример полезной нагрузки события:
{
  name: "intercom/company.updated",
  data: {
    company_id: "123456789",
    company_name: "Acme, Inc."
  }
}
*/
export default inngest.createFunction(
  {
    id: "synchronize-data",
    triggers: { event: "intercom/company.updated" },
    rateLimit: {
      key: "event.data.company_id",
      limit: 1,
      period: "4h",
    },
  },
  async ({ event, step }) => {
    const company = await step.run(
      "fetch-latest-company-data-from-intercom",
      async () => {
        return await client.companies.find({
          companyId: event.data.company_id,
        });
      }
    );

    await step.run("update-company-data-in-database", async () => {
      return await database.companies.upsert({ id: company.id }, company);
    });
  }
);
```

### Отправка не более одного email при нескольких оповещениях за час

Когда в системе возникает проблема, вы можете захотеть отправить пользователю email-уведомление, но не перегружать его. Проблема может повторяться несколько раз в течение нескольких минут, но пользователю нужен только один email. Вы можете:

```ts
/** Пример полезной нагрузки события:
{
  name: "service/check.failed",
  data: {
    incident_id: "01HB9PWHZ4CZJYRAGEY60XEHCZ",
    issue: "HTTP uptime check failed at 2023-09-26T21:23:51.515631317Z",
    user_id: "user_aW5uZ2VzdF9pc19mdWNraW5nX2F3ZXNvbWU=",
    service_name: "api",
    service_id: "01HB9Q2EFBYG2B7X8VCD6JVRFH"
  },
  user: {
    external_id: "user_aW5uZ2VzdF9pc19mdWNraW5nX2F3ZXNvbWU=",
    email: "user@example.com"
  }
}
*/
export default inngest.createFunction(
  {
    id: "send-check-failed-notification",
    triggers: { event: "service/check.failed" },
    rateLimit: {
      // Не отправлять дублирующиеся email одному пользователю для одного сервиса в течение 1 часа
      key: `event.data.user_id + "-" + event.data.service_id`,
      limit: 1,
      period: "1h",
    },
  },
  async ({ event, step }) => {
    await step.run("send-alert-email", async () => {
      return await resend.emails.send({
        from: "notifications@myco.com",
        to: event.user.email,
        subject: `ALERT: ${event.data.issue}`,
        text: `Dear user, ...`,
      });
    });
  }
);
```
