# Приоритет запуска функций


Вы можете устанавливать приоритет для конкретных запусков функций выше других запусков **в пределах одной функции**.

Смотрите [руководство по приоритетам](/docs/guides/priority) для получения дополнительной информации о работе этой функции.

```ts
export default inngest.createFunction(
  {
    id: "ai-generate-summary",
    triggers: { event: "ai/summary.requested" },
    priority: {
      // Для корпоративных аккаунтов данный запуск функции будет приоритетным
      // по сравнению с функциями, поставленными в очередь до 120 секунд назад.
      // Для всех остальных аккаунтов функция будет выполнена без приоритета.
      run: "event.data.account_type == 'enterprise' ? 120 : 0",
    },
  },
  async ({ event, step }) => {
    // Эта функция будет приоритизирована на основе типа аккаунта
  }
);
```

## Конфигурация

<Properties>
  <Property name="priority" type="object">
    Параметры для настройки приоритета функций

    <Properties nested={true}>
      <Property name="run" type="string">
        Выражение, возвращающее целое число от -600 до 600 (по умолчанию); большие значения означают более высокий приоритет.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * Вернуть приоритет непосредственно из события: `event.data.priority` (где `event.data.priority` — целое число в допустимом диапазоне аккаунта)
        * Приоритизация по строковому полю: `event.data.plan == 'enterprise' ? 180 : 0`
      </Property>
    </Properties>
  </Property>
</Properties>

<Callout>
  Значения вне допустимого диапазона аккаунта (по умолчанию от -600 до 600) будут автоматически обрезаны до максимальных границ.

  Недопустимое выражение вычисляется как 0, то есть "без приоритета".
</Callout>
