# Вспомогательные функции триггеров

Вспомогательные функции триггеров обеспечивают типобезопасные способы определения триггеров и схем для функций Inngest. Они заменяют строковые триггеры на типизированные, валидированные альтернативы.

```ts
import { eventType, cron, invoke, staticSchema } from "inngest";
```

---

## `eventType`

Определение типизированного события с опциональной валидацией во время выполнения. Используйте возвращаемый объект как [триггер](/docs/reference/typescript/v4/functions/create#triggers), с [`step.waitForEvent()`](/docs/reference/typescript/v4/functions/step-wait-for-event), а также с [`inngest.send()`](/docs/reference/typescript/v4/events/send) или [`step.sendEvent()`](/docs/reference/typescript/v4/functions/step-send-event).

```ts
import { eventType } from "inngest";
import { z } from "zod";

const accountCreated = eventType("app/account.created", {
  schema: z.object({
    userId: z.string(),
    email: z.string(),
  }),
});
```

### Параметры

<Properties>
  <Property name="name" type="string" required>
    Имя события. Рекомендуем использовать нижний регистр с точечной нотацией и пространство имён с префиксом `/`, например `"app/account.created"`.
  </Property>
  <Property name="options" type="object">
    <Properties nested={true}>
      <Property name="schema" type="StandardSchemaV1 | StaticSchema">
        Схема для полезной нагрузки `data` события. Принимает любую библиотеку, совместимую со [Standard Schema](https://standardschema.dev/) (Zod, Valibot, ArkType и др.), или [`staticSchema()`](#staticschema) для проверки только на уровне типов.

        При указании схемы времени выполнения SDK валидирует полезные нагрузки событий во время выполнения.
      </Property>
      <Property name="version" type="string">
        Опциональный идентификатор версии для полезной нагрузки события, например `"2024-01-15.1"`. Устанавливает поле `v` в отправляемых событиях.
      </Property>
    </Properties>
  </Property>
</Properties>

### Возвращаемое значение

`eventType()` возвращает объект `EventType`. Единственный случай прямого взаимодействия с этим объектом — вызов метода `create` при отправке событий.

### Как триггер

Передайте результат `eventType()` непосредственно в массив `triggers` функции `createFunction`:

<Row>
  <Col>
    ```ts
    const orderPlaced = eventType("shop/order.placed", {
      schema: z.object({
        orderId: z.string(),
        total: z.number(),
      }),
    });

    inngest.createFunction(
      {
        id: "process-order",
        triggers: [orderPlaced],
      },
      async ({ event }) => {
        // event.data типизирован как { orderId: string; total: number }
        console.log(event.data.orderId);
      }
    );
    ```
  </Col>
  <Col>
    Можно также комбинировать с фильтрацией `if`, обернув в объект:

    ```ts
    inngest.createFunction(
      {
        id: "process-large-orders",
        triggers: [{
          ...orderPlaced,
          if: "event.data.total > 100",
        }],
      },
      async ({ event }) => {
        // Запускается только для заказов свыше 100
      }
    );
    ```
  </Col>
</Row>

### С `step.waitForEvent()`

Передайте результат `eventType()` как опцию `event` для получения типизированных возвращаемых значений:

```ts
const approvalReceived = eventType("app/approval.received", {
  schema: z.object({
    approved: z.boolean(),
    approvedBy: z.string(),
  }),
});

inngest.createFunction(
  {
    id: "approval-workflow",
    triggers: [orderPlaced],
  },
  async ({ event, step }) => {
    const approval = await step.waitForEvent("wait-for-approval", {
      event: approvalReceived,
      timeout: "7d",
      match: "data.orderId",
    });

    if (approval) {
      // approval.data типизирован как { approved: boolean; approvedBy: string }
    }
  }
);
```

### С `inngest.send()` и `step.sendEvent()`

Используйте метод `.create()` для создания полностью типизированных полезных нагрузок событий:

```ts
// С inngest.send()
await inngest.send(
  orderPlaced.create({
    orderId: "order_123",
    total: 99.99,
  })
);

// С step.sendEvent()
await step.sendEvent("notify-order",
  orderPlaced.create({
    orderId: "order_456",
    total: 149.99,
  })
);
```

### Несколько типов событий

Когда функция имеет несколько триггеров событий, используйте `event.name` для уточнения типа:

```ts
const userCreated = eventType("app/user.created", {
  schema: z.object({ userId: z.string() }),
});

const userUpdated = eventType("app/user.updated", {
  schema: z.object({ userId: z.string(), changes: z.record(z.unknown()) }),
});

inngest.createFunction(
  {
    id: "sync-user",
    triggers: [userCreated, userUpdated],
  },
  async ({ event }) => {
    if (event.name === "app/user.created") {
      // event.data типизирован как { userId: string }
    }

    if (event.name === "app/user.updated") {
      // event.data типизирован как { userId: string; changes: Record<string, unknown> }
    }
  }
);
```

### Символы подстановки

Можно использовать шаблоны с подстановочными символами в `eventType()` для сопоставления нескольких имён событий:

```ts
const anyUserEvent = eventType("user/*");

inngest.createFunction(
  {
    id: "audit-user-events",
    triggers: [anyUserEvent],
  },
  async ({ event }) => {
    // Запускается на "user/created", "user/updated" и т.д.
  }
);
```

<Callout>
  Типы событий с подстановочными символами не могут определять схему, так как соответствующие события могут иметь разные структуры полезной нагрузки.
</Callout>

## `cron`

Создание типизированного cron-триггера для плановых функций.

```ts
import { cron } from "inngest";

inngest.createFunction(
  {
    id: "daily-cleanup",
    triggers: [cron("0 0 * * *")],
  },
  async ({ step }) => {
    // Запускается каждый день в полночь UTC
  }
);
```

### Параметры

<Properties>
  <Property name="cronExpression" type="string" required>
    Строка расписания в формате [unix-cron](https://crontab.guru/). Поддерживает опциональный префикс часового пояса, например `"TZ=Europe/Paris 0 12 * * 5"`.
  </Property>
</Properties>

### Примеры

<CodeGroup>
```ts {{ title: "Basic" }}
inngest.createFunction(
  {
    id: "hourly-sync",
    triggers: [cron("0 * * * *")],
  },
  async ({ step }) => {
    // Запускается каждый час
  }
);
```

```ts {{ title: "With timezone" }}
inngest.createFunction(
  {
    id: "daily-report",
    triggers: [cron("TZ=America/New_York 0 9 * * 1-5")],
  },
  async ({ step }) => {
    // Запускается в 9:00 по времени ET в будни
  }
);
```

```ts {{ title: "Combined with eventType" }}
const manualSync = eventType("app/sync.requested");

inngest.createFunction(
  {
    id: "data-sync",
    triggers: [
      cron("0 */6 * * *"),
      manualSync,
    ],
  },
  async ({ event, step }) => {
    // Запускается каждые 6 часов ИЛИ при ручном запуске
  }
);
```
</CodeGroup>

## `invoke`

Определение типизированной схемы для входных данных [`step.invoke()`](/docs/reference/typescript/v4/functions/step-invoke). Все функции можно вызывать независимо от использования `invoke()`. Этот вспомогательный инструмент только определяет схему, чтобы вызывающий код получал проверку типов на передаваемые данные.

```ts
import { invoke } from "inngest";
import { z } from "zod";

const computeSquare = inngest.createFunction(
  {
    id: "compute-square",
    triggers: [
      { event: "calculate/square" },
      invoke({
        schema: z.object({ number: z.number() }),
      }),
    ],
  },
  async ({ event }) => {
    return { result: event.data.number * event.data.number };
  }
);
```

### Параметры

<Properties>
  <Property name="options" type="object" required>
    <Properties nested={true}>
      <Property name="schema" type="StandardSchemaV1 | StaticSchema" required>
        Схема, определяющая ожидаемую структуру `data` при вызове этой функции через `step.invoke()`. Принимает любую библиотеку, совместимую со [Standard Schema](https://standardschema.dev/), или [`staticSchema()`](#staticschema).
      </Property>
    </Properties>
  </Property>
</Properties>

### Пример

Определите схему invoke на целевой функции, затем вызовите её с типизированными данными:

<Row>
  <Col>
    ```ts {{ title: "Target function" }}
    import { invoke } from "inngest";
    import { z } from "zod";

    export const processImage = inngest.createFunction(
      {
        id: "process-image",
        triggers: [
          { event: "image/uploaded" },
          invoke({
            schema: z.object({
              imageUrl: z.string().url(),
              width: z.number(),
            }),
          }),
        ],
      },
      async ({ event }) => {
        // event.data.imageUrl и event.data.width типизированы
        return { processed: true };
      }
    );
    ```
  </Col>
  <Col>
    ```ts {{ title: "Calling function" }}
    const result = await step.invoke("resize-image", {
      function: processImage,
      data: {
        imageUrl: "https://example.com/photo.jpg",
        width: 800,
        // ^ Схема invoke типизирует и валидирует это
      },
    });
    ```
  </Col>
</Row>

## `staticSchema`

Утилита только для типов — когда нужна проверка типов TypeScript без валидации во время выполнения. Используйте как альтернативу библиотеке Standard Schema, например Zod.

```ts
import { eventType, staticSchema } from "inngest";

type OrderData = {
  orderId: string;
  total: number;
};

const orderPlaced = eventType("shop/order.placed", {
  schema: staticSchema<OrderData>(),
});
```

<Callout>
  Используйте `staticSchema()`, когда не нужна валидация во время выполнения. Если нужна валидация полезных нагрузок во время выполнения, используйте библиотеку Standard Schema, например [Zod](https://zod.dev/), [Valibot](https://valibot.dev/) или [ArkType](https://arktype.io/).
</Callout>

### Параметры типов

<Properties>
  <Property name="T" type="type" required>
    TypeScript-тип, представляющий структуру `data` события. Компилятор использует этот тип только для проверки на этапе компиляции — валидации во время выполнения не происходит.
  </Property>
</Properties>

### Примеры

<CodeGroup>
```ts {{ title: "With eventType" }}
import { eventType, staticSchema } from "inngest";

type UserPayload = {
  userId: string;
  email: string;
};

const userCreated = eventType("app/user.created", {
  schema: staticSchema<UserPayload>(),
});

inngest.createFunction(
  {
    id: "on-user-created",
    triggers: [userCreated],
  },
  async ({ event }) => {
    // event.data типизирован как { userId: string; email: string }
  }
);
```

```ts {{ title: "With invoke" }}
import { invoke, staticSchema } from "inngest";

type ResizeInput = {
  imageUrl: string;
  width: number;
};

const resizeImage = inngest.createFunction(
  {
    id: "resize-image",
    triggers: [
      invoke({ schema: staticSchema<ResizeInput>() }),
    ],
  },
  async ({ event }) => {
    // event.data типизирован как { imageUrl: string; width: number }
    return { resized: true };
  }
);
```
</CodeGroup>
