# Cancel on


Остановите выполнение запущенной функции при получении определённого события с помощью `cancelOn`.

```ts
inngest.createFunction(
  {
    id: "sync-contacts",
    triggers: { event: "app/user.created" },
    cancelOn: [
      {
        // Может быть строкой или EventType
        event: "app/user.deleted",
        // убедитесь, что userId в async (будущем) событии совпадает с userId в триггере
        if: "async.data.userId == event.data.userId",
      },
    ],
  }
  // ...
);
```

Использование `cancelOn` очень полезно для обработки сценариев, когда длительно выполняемая функция должна быть досрочно завершена из-за изменений в другом месте системы.

API похож на [`step.waitForEvent()`](/docs/reference/typescript/v4/functions/step-wait-for-event) и позволяет указать входящее событие и различные методы сопоставления данных.

---

## Как использовать `cancelOn`

Наиболее распространённый сценарий отмены — это отмена выполнения функции, если определённое поле входящего события совпадает с соответствующим полем в триггерном событии. Например, отмена синхронизации пользователя при его удалении. Для этого нужно указать выражение `match`. Рассмотрим пример функции и двух событий.

Эта функция указывает, что она будет `cancelOn` событии `"app/user.deleted"` только когда оно и исходное событие `"app/user.created"` имеют одинаковое значение `data.userId`:

```ts
inngest.createFunction(
  {
    id: "sync-contacts",
    triggers: { event: "app/user.created" },
    cancelOn: [
      {
        event: "app/user.deleted",
        // убедитесь, что userId в async (будущем) событии совпадает с userId в триггере
        if: "async.data.userId == event.data.userId",
      },
    ],
  },
  // ...
);
```

Для данной функции это пример события, которое запустит её:

```json
{
  "name": "app/user.created",
  "data": {
    "userId": "123",
    "name": "John Doe"
  }
}
```

А это пример события, которое отменит функцию, так как оно и исходное событие имеют одинаковое значение `data.userId` равное `"123"`:

```json
{
  "name": "app/user.deleted",
  "data": {
    "userId": "123"
  }
}
```

Выражения сопоставления могут быть простыми проверками равенства или более сложными. Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions).

<Callout Icon={InformationCircleIcon}>
  Функции отменяются _между шагами_, то есть если `step.run` выполняется в данный момент, он завершится до отмены функции.

  Inngest делает это для обеспечения атомарности шагов: каждый шаг либо завершается полностью, либо вообще не запускается.
</Callout>

## Конфигурация

<Properties>
  <Property name="cancelOn" type="array of objects">
    Определяет события, которые могут отменить выполняющуюся или спящую функцию
    <Properties nested={true}>
      <Property name="event" type="string | EventType" required>
        Событие для отмены. Может быть строковым именем (например, `"app/user.deleted"`) или объектом события, созданным с помощью `eventType()`
      </Property>
      <Property name="match" type="string">
        Свойство для сопоставления триггерного и отменяющего событий с использованием точечной нотации, например `data.userId`. Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions).
      </Property>
      <Property name="if" type="string">
        Выражение для условного сопоставления исходного триггерного события (`event`) и ожидаемого события (`async`). Нельзя комбинировать с `match`.

        Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

        * `event.data.userId == async.data.userId && async.data.billing_plan == 'pro'`
      </Property>
      <Property name="timeout" type="string">
        Время ожидания отменяющего события. Строка времени, совместимая с пакетом [ms](https://npm.im/ms), например `"30m"`, `"3 hours"` или `"2.5d"`
      </Property>
    </Properties>
  </Property>
</Properties>

## Примеры

### Использование объекта события

Свойство `event` также принимает объект события вместо строки. Это полезно при наличии типизированных определений событий или при ссылке на события из схемы.

```ts
import { userCreatedEvent, userDeletedEvent } from "./events";

inngest.createFunction(
  {
    id: "sync-contacts",
    triggers: userCreatedEvent,
    cancelOn: [
      {
        event: userDeletedEvent,
        if: "async.data.userId == event.data.userId",
      },
    ],
  }
  // ...
);
```

### С окном таймаута

Отмена выполнения функции, если в течение заданного времени с момента запуска получено соответствующее событие.

```ts
inngest.createFunction(
  {
    id: "sync-contacts",
    triggers: { event: "app/user.created" },
    cancelOn: [{ event: "app/user.deleted", match: "data.userId", timeout: "1h" }],
  }
  // ...
);
```

Это полезно для ограничения временного окна отмены, гарантируя, что функция продолжит выполнение, если соответствующее событие не получено в указанные сроки.
