# Wait for event


Используйте `step.waitForEvent()` для приостановки выполнения функции до получения подходящего события или истечения таймаута. Это полезно для построения [рабочих процессов с участием человека](/docs/ai-patterns/human-in-the-loop), ожидания подтверждений или координации между отдельными функциями.

## `step.waitForEvent(id, options): Promise<null | EventPayload>`

<Row>
  <Col>
    <Properties>
      <Property name="id" type="string" required>
        ID шага. Будет отображаться в логах функции и используется для мемоизации состояния шага между версиями функции.
      </Property>
      <Property name="options" type="object" required>
        Параметры для настройки ожидания события.
        <Properties nested={true}>
          <Property name="event" type="string | EventType" required>
            Имя ожидаемого события или результат [`eventType()`](/docs/reference/typescript/v4/functions/triggers#eventtype) для типизированных возвращаемых значений.
          </Property>
          <Property name="timeout" type="string" required>
            Время ожидания события. Строка времени, совместимая с пакетом [ms](https://npm.im/ms), например `"30m"`, `"3 hours"` или `"2.5d"`
          </Property>
          <Property name="match" type="string">
            Свойство для сопоставления триггерного события и ожидаемого события с использованием точечной нотации, например `data.userId`. Нельзя комбинировать с `if`.
          </Property>
          <Property name="if" type="string">
            Выражение для условного сопоставления исходного триггерного события (`event`) и ожидаемого события (`async`). Нельзя комбинировать с `match`.

            Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

            * `event.data.userId == async.data.userId && async.data.billing_plan == 'pro'`
          </Property>
        </Properties>
      </Property>
    </Properties>
  </Col>
  <Col>
  ```ts
  // Ждать 7 дней подтверждения и сопоставить ID счёта
  const approval = await step.waitForEvent("wait-for-approval", {
    event: "app/invoice.approved",
    timeout: "7d",
    match: "data.invoiceId",
  });

  // Ждать 30 дней начала подписки на тарифный план pro
  const subscription = await step.waitForEvent("wait-for-subscription", {
    event: "app/subscription.created",
    timeout: "30d",
    if: "event.data.userId == async.data.userId && async.data.billing_plan == 'pro'",
  });

  // Использовать eventType() для типизированных возвращаемых значений
  const approvalType = eventType("app/approval.received", {
    schema: z.object({ approved: z.boolean() }),
  });

  const approval = await step.waitForEvent("wait-for-approval", {
    event: approvalType,
    timeout: "7d",
  });
  // approval?.data типизирован как { approved: boolean }
  ```
  </Col>
</Row>

<Callout>
  `step.waitForEvent()` должен вызываться с `await` или другим обработчиком Promise для корректной работы функции.
</Callout>
