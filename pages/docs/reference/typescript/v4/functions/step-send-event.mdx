# Send Event


Используйте для надёжной отправки событий внутри функции. Применяйте вместо [`inngest.send()`](/docs/reference/typescript/v4/events/send), чтобы обеспечить гарантированную доставку событий из функций. Особенно полезно при [создании функций с разветвлением](/docs/guides/fan-out-jobs).

```ts
export default inngest.createFunction(
  {
    id: "user-onboarding",
    triggers: { event: "app/user.signup" },
  },
  async ({ event, step }) => {
    // Выполнить что-то
    await step.sendEvent("send-activation-event", {
      name: "app/user.activated",
      data: { userId: event.data.userId },
    });
    // Выполнить что-то ещё
  }
);
```

Для отправки событий вне контекста функции используйте [`inngest.send()`](/docs/reference/typescript/v4/events/send).

<Callout>
  Используйте [`eventType().create()`](/docs/reference/typescript/v4/functions/triggers#with-inngestsend-and-stepsendevent) для создания полностью типизированных полезных нагрузок событий для `step.sendEvent()`.
</Callout>

## `step.sendEvent(id, eventPayload | eventPayload[]): Promise<{ ids: string[] }>`

<Row>
  <Col>
    <Properties>
      <Property name="id" type="string" required>
        ID шага. Будет отображаться в логах функции и используется для мемоизации состояния шага между версиями функции.
      </Property>
      <Property name="eventPayload" type="object | object[]" required>
        Объект полезной нагрузки события или массив объектов полезной нагрузки событий.

        [Смотрите документацию по `inngest.send()`](/docs/reference/typescript/v4/events/send#inngest-send-event-payload-event-payload-promise) для формата полезной нагрузки события.
      </Property>
    </Properties>
  </Col>
  <Col>
  ```ts
  // Отправить одно событие
  await step.sendEvent("send-activation-event", {
    name: "app/user.activated",
    data: { userId: "01H08SEAXBJFJNGTTZ5TAWB0BD" },
  });

  // Отправить массив событий
  await step.sendEvent("send-invoice-events", [
    {
      name: "app/invoice.created",
      data: { invoiceId: "645e9e024befa68763f5b500" },
    },
    {
      name: "app/invoice.created",
      data: { invoiceId: "645e9e08f29fb563c972b1f7" },
    },
  ]);

  ```
  </Col>
</Row>

<Callout>
  `step.sendEvent()` должен вызываться с `await` или другим обработчиком Promise для корректной работы функции.
</Callout>

### Возвращаемые значения

Функция возвращает промис, разрешающийся в объект с массивом ID событий, которые были отправлены. Эти события можно использовать для поиска в панели управления Inngest или через [REST API](https://api-docs.inngest.com/docs/inngest-api/pswkqb7u3obet-get-an-event).

```ts
const { ids } = await step.sendEvent("send-invoices", [
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e024befa68763f5b500" }
  },
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e08f29fb563c972b1f7" }
  },
]);
/**
 * ids = [
 *   "01HQ8PTAESBZPBDS8JTRZZYY3S",
 *   "01HQ8PTFYYKDH1CP3C6PSTBZN5"
 * ]
 */
```
