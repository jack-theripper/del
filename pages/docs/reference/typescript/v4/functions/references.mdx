# Ссылки на функции

Используя [`step.invoke()`](/docs/reference/typescript/v4/functions/step-invoke), вы можете напрямую вызывать одну функцию Inngest из другой и обрабатывать результат. С `referenceFunction` можно вызывать функции Inngest из других приложений или избегать импорта зависимостей функций в рамках одного приложения.

```ts
// @/inngest/compute.ts
import { referenceFunction } from "inngest";
import { z } from "zod";
import { type computePi } from "@/inngest/computePi";

// Создать локальную ссылку на функцию без импорта зависимостей
export const computePi = referenceFunction<typeof computePi>({
  functionId: "compute-pi",
});

// Создать ссылку на функцию в другом приложении
export const computeSquare = referenceFunction({
  appId: "my-python-app",
  functionId: "compute-square",
  // Схемы опциональны, но обеспечивают типизацию при вызове
  schemas: {
    data: z.object({
      number: z.number(),
    }),
    return: z.object({
      result: z.number(),
    }),
  },
});
```

```ts
// @/inngest/someFn.ts
import { computeSquare } from "@/inngest/compute"; // импортировать ссылку

// square.result типизирован как number
const square = await step.invoke("compute-square-value", {
  function: computeSquare,
  data: { number: 4 }, // входные данные типизированы
});
```

## Как использовать `referenceFunction`

Самая простая ссылка содержит только `functionId`. При использовании будет вызвана функция с указанным ID в том же приложении, из которого выполняется вызов.

Типы входных и выходных данных будут `unknown`.

```ts
import { referenceFunction } from "inngest";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
  }),
});
```

Если ссылка на функцию в другом приложении, также укажите `appId`:

```ts
import { referenceFunction } from "inngest";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
    appId: "some-app",
  }),
});
```

Опционально можно указать `schemas` — набор [Standard Schemas](https://standardschema.dev/) для типизации входных и выходных данных ссылочной функции.

<Callout>
В будущем это также будет _валидировать_ входные и выходные данные.
</Callout>

```ts
import { referenceFunction } from "inngest";
import { z } from "zod";

await step.invoke("start-process", {
  function: referenceFunction({
    functionId: "some-fn",
    appId: "some-app",
    schemas: {
      data: z.object({
        foo: z.string(),
      }),
      return: z.object({
        success: z.boolean(),
      }),
    },
  }),
});
```

Даже если функции находятся в одном приложении, это можно использовать для избежания импорта зависимостей одной функции в другую, что полезно в фреймворках вроде Next.js, где edge и serverless-логика могут находиться рядом, но требуют разных зависимостей.

```ts
import { referenceFunction } from "inngest";
import { type someInngestFn } from "@/inngest/someFn"; // импортировать только тип

await step.invoke("start-process", {
  function: referenceFunction<typeof someInngestFn>({
    functionId: "some-fn",
  }),
});
```

## Конфигурация

<Properties>
     <Property name="functionId" type="string" required>
          ID функции для ссылки. Может быть локальным ID функции или ID функции в другом приложении.

          В последнем случае также нужно указать `appId`. Если `appId` не указан, ID функции предполагается локальным (будет использован ID приложения вызывающего).
     </Property>
     <Property name="appId" type="string">
          ID приложения, которому принадлежит функция. Обязателен, только если функция находится в другом приложении.
     </Property>
     <Property name="schemas" type="object">
          Схемы ссылочной функции, обеспечивающие типизацию входных `data` и возвращаемого значения при вызове.

          Если не указаны и тип локальной функции не передан как обобщённый параметр в `referenceFunction()`, схемы будут типизированы как `unknown`.

          <Properties nested>
               <Property name="data" type="zod">
                    [Standard Schema](https://standardschema.dev/) для типизации полезной нагрузки `data`, требуемой ссылочной функцией.
               </Property>
               <Property name="return" type="zod">
                    [Standard Schema](https://standardschema.dev/) для типизации возвращаемого значения ссылочной функции при вызове.
               </Property>
          </Properties>
     </Property>
</Properties>
