# Создание функции


Определяйте функции с помощью метода `createFunction` на [клиенте Inngest](/docs/reference/typescript/v4/client/create).

```ts
import { inngest } from "./client";

export default inngest.createFunction(
  {
    id: "import-product-images",
    triggers: { event: "shop/product.imported" },
  },
  async ({ event, step, runId }) => {
    // Код вашей функции
  }
);
```

---

## `inngest.createFunction(configuration, handler): InngestFunction`

Метод `createFunction` принимает ряд аргументов для определения вашей функции.

#### Триггеры

Вы можете использовать [вспомогательные функции триггеров](/docs/reference/typescript/v4/functions/triggers) для определения типизированных триггеров с опциональной валидацией во время выполнения:

```ts
import { eventType, cron } from "inngest";
import { z } from "zod";

const orderPlaced = eventType("shop/order.placed", {
  schema: z.object({ orderId: z.string(), total: z.number() }),
});

inngest.createFunction(
  {
    id: "process-order",
    triggers: [orderPlaced, cron("0 * * * *")],
  },
  async ({ event, step }) => {
    // event.data полностью типизирован
  }
);
```

Альтернативно можно задать массив до 10 триггеров для вызова функции несколькими событиями или cron-расписаниями. Смотрите [руководство по нескольким триггерам](/docs/guides/multiple-triggers).

<Callout>
Cron-триггеры с перекрывающимися расписаниями для одной функции будут дедуплицированы.
</Callout>

<Properties>
  <Property name="event" type="string | EventType">
    Событие, которое запустит эту функцию. Может быть строковым именем (например, `"app/user.created"`) или объектом события, созданным с помощью `eventType()`.
  </Property>
  <Property name="cron" type="string">
    Строка расписания в формате [unix-cron](https://crontab.guru/). <br/>Опциональный префикс часового пояса, например `TZ=Europe/Paris 0 12 * * 5`.
  </Property>
</Properties>

При использовании триггера `event` можно опционально комбинировать его с опцией `if` для фильтрации событий:

<Properties nested={true} name="Дополнительные параметры">
  <Property name="if" type="string">
    Выражение сравнения, возвращающее true или false — должна ли функция обработать или проигнорировать данное подходящее событие.

    Выражения определяются с использованием Common Expression Language (CEL). Читайте [наше руководство по написанию выражений](/docs/guides/writing-expressions). Примеры:

    * `'event.data.action == "published"'`
    * `'event.data.priority >= 4'`
  </Property>
</Properties>

### Обработчик

Обработчик — это ваш код, выполняемый при каждом срабатывании триггера. Каждый обработчик функции получает единственный объект-аргумент, который можно деструктурировать. Ключевые аргументы: `event` и `step`. Обратите внимание, что плановые функции с триггером `cron` не получают аргумент `event`.

```ts
function handler({ event, events, step, runId, logger, attempt }) {/* ... */}
```

#### `event`

Объект полезной нагрузки события, запустившего данный запуск функции. Соответствует тому, что вы отправили с помощью [`inngest.send()`](/docs/reference/typescript/v4/events/send). Пример:

```ts
{
  name: "app/account.created",
  data: {
    userId: "1234567890"
  },
  v: "2023-05-12.1",
  ts: 1683898268584
}
```

#### `events`

`events` — массив объектов полезной нагрузки событий, доступный когда в конфигурации функции задан `batchEvents`.
Если батчинг не настроен, массив содержит одно событие, совпадающее с аргументом `event`.

#### `step`

Объект `step` содержит методы для определения:

- [`step.run()`](/docs/reference/typescript/v4/functions/step-run) — Выполнение синхронного или асинхронного кода как повторяемого шага в функции
- [`step.sleep()`](/docs/reference/typescript/v4/functions/step-sleep) — Приостановить выполнение на заданное время
- [`step.sleepUntil()`](/docs/reference/typescript/v4/functions/step-sleep-until) — Приостановить выполнение до заданного времени
- [`step.invoke()`](/docs/reference/typescript/v4/functions/step-invoke) — Вызвать другую функцию Inngest как шаг, получая результат вызванной функции
- [`step.waitForEvent()`](/docs/reference/typescript/v4/functions/step-wait-for-event) — Приостановить выполнение функции до получения другого события
- [`step.sendEvent()`](/docs/reference/typescript/v4/functions/step-send-event) — Надёжно отправить события внутри функции. Используйте вместо `inngest.send()` для гарантированной доставки событий.

#### `runId`

Уникальный ID данного запуска функции. Полезен для логирования и поиска конкретных запусков в панели управления Inngest.

#### `logger`

Объект `logger` предоставляет следующие интерфейсы.

```ts
export interface Logger {
  info(...args: any[]): void;
  warn(...args: any[]): void;
  error(...args: any[]): void;
  debug(...args: any[]): void;
}
```

Это прокси-объект, работающий либо через `console`, либо через предоставленный вами логгер ([справочник](/docs/guides/logging)).

#### `attempt`

Текущий номер попытки (начиная с нуля) для данного выполнения функции. Первая попытка имеет номер 0, вторая — 1 и так далее. Номер попытки увеличивается при каждом сбое функции и повторном запуске.
