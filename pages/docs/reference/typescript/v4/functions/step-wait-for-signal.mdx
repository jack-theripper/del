# Wait for signal


Ожидание получения определённого сигнала перед продолжением с помощью step.waitForSignal(). Для возобновления сигналы нужно передавать через
`step.sendSignal` или `client.sendSignal` (как шаг или через клиент Inngest), передавая ту же строку сигнала
и любые данные для инъекции в запуск функции.

## `step.waitForSignal(id, options): Promise<null | EventPayload>`

<Row>
  <Col>
    <Properties>
      <Property name="id" type="string" required>
        ID шага. Будет отображаться в логах функции и используется для мемоизации состояния шага между версиями функции.
      </Property>
      <Property name="options" type="object" required>
        Параметры для настройки ожидания события.
        <Properties nested={true}>
          <Property name="signal" type="string" required>
            Уникальный идентификатор сигнала, используемый для возобновления данного запуска функции.
          </Property>
          <Property name="timeout" type="string" required>
            Время ожидания сигнала. Строка времени, совместимая с пакетом [ms](https://npm.im/ms), например `"30m"`, `"3 hours"` или `"2.5d"`.

            Если сигнал не получен до истечения таймаута, запуск возобновится с неопределённым возвращаемым значением.
          </Property>
          <Property name="onConflict" type="'fail' | 'replace'">
            Поведение при дублирующихся сигналах. По умолчанию `step.waitForSignal` требует уникального сигнала, и дублирующиеся сигналы приведут к ошибке функции.

            Установите `'replace'` для замены сигнала любого другого запуска. Обратите внимание, что предыдущие запуски НЕ возобновятся по этому сигналу при его замене и вместо этого истекут по таймауту.

            Для возобновления нескольких запусков по одному сигналу используйте `step.waitForEvent`.
          </Property>
        </Properties>
      </Property>
    </Properties>
  </Col>
  <Col>
  ```ts
  // Ждать 15 минут подтверждения
  const approval = await step.waitForSignal("wait-for-approval", {
    signal: "task/71651db4-9f27-466a-a6be-4759b9784b3c",
    timeout: "7d",
  });
  ```
  </Col>
</Row>

<Callout>
  `step.waitForSignal()` должен вызываться с `await` или другим обработчиком Promise для корректной паузы функции.
</Callout>

<Callout>
  `step.waitForSignal()` должен быть возобновлён через `inngest.sendSignal`, `step.sendSignal` или вызов API сигналов с соответствующей строкой сигнала.
</Callout>
