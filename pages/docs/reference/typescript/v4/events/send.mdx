# Отправка событий


Отправка событий в Inngest. Функции с соответствующими триггерами будут вызваны.

```ts
import { inngest } from "./client";

await inngest.send({
  name: "app/account.created",
  data: {
    accountId: "645e9f6794e10937e9bdc201",
    billingPlan: "pro",
  },
  user: {
    external_id: "645ea000129f1c40109ca7ad",
    email: "taylor@example.com",
  }
})
```

Для отправки событий внутри функции используйте [`step.sendEvent()`](/docs/reference/typescript/v4/functions/step-send-event).

<Callout>
  Используйте [`eventType().create()`](/docs/reference/typescript/v4/functions/triggers#with-inngestsend-and-stepsendevent) для создания полностью типизированных полезных нагрузок событий для `inngest.send()`.
</Callout>

### Возвращаемые значения

Функция возвращает промис, разрешающийся в объект с массивом ID событий, которые были отправлены. Эти события можно использовать для поиска в панели управления Inngest или через [REST API](https://api-docs.inngest.com/docs/inngest-api/pswkqb7u3obet-get-an-event).

```ts
const { ids } = await inngest.send([
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e024befa68763f5b500" }
  },
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e08f29fb563c972b1f7" }
  },
]);
/**
 * ids = [
 *   "01HQ8PTAESBZPBDS8JTRZZYY3S",
 *   "01HQ8PTFYYKDH1CP3C6PSTBZN5"
 * ]
 */
```

## Шифрование данных пользователя 🔐

Все данные, отправленные в объекте `user`, полностью шифруются при хранении.

<Callout variant="warning">
  <span className="text-yellow-300">⚠️</span> При [воспроизведении функции](/docs/platform/replay) поле `event.user` будет пустым. Это будет исправлено в будущем, но пока предполагается, что нельзя воспроизводить функции, зависящие от данных `event.user`.
</Callout>

В будущем этот объект будет использоваться для поддержки программного удаления через API для поддержки определённых процессов соблюдения права на забвение в вашей системе. Для поиска будет использоваться свойство `user.external_id`.
