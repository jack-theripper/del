# Промежуточное ПО шифрования

Промежуточное ПО шифрования обеспечивает сквозное шифрование для событий, вывода шагов и вывода функций. **На серверы Inngest отправляются только зашифрованные данные**: шифрование и расшифровка происходят в вашей инфраструктуре.

## Установка

Установите [пакет `@inngest/middleware-encryption`](https://www.npmjs.com/package/@inngest/middleware-encryption) ([GitHub](https://github.com/inngest/inngest-js/tree/main/packages/middleware-encryption#readme)) и настройте следующим образом:

```ts
import { encryptionMiddleware } from "@inngest/middleware-encryption";

const inngest = new Inngest({
  id: "my-app",
  middleware: [
    encryptionMiddleware({
      // строку ключа шифрования не следует хранить в коде
      key: process.env.MY_ENCRYPTION_KEY,
    }),
  ],
});
```

По умолчанию шифруется:
- Все данные шагов
- Все выходные данные функций
- Данные события в поле `data.encrypted`

## `encryptionMiddleware(options)`

Возвращает `Middleware.Class`, который можно передать в массив `middleware` клиента или функции.

### Параметры

<Properties>
  <Property name="key" type="string" required>
    Ключ шифрования для шифрования и расшифровки данных. Должен быть секретным значением, хранящимся в переменной окружения.
  </Property>
  <Property name="decryptOnly" type="boolean">
    Если `true`, отключает шифрование, но продолжает расшифровывать существующие данные. Полезно при переходе от шифрования.
  </Property>
  <Property name="eventEncryptionField" type="string">
    Поле в `event.data` для шифрования. По умолчанию `"encrypted"`.
  </Property>
  <Property name="fallbackDecryptionKeys" type="string[]">
    Дополнительные ключи для попытки расшифровки. Полезно при ротации ключей.
  </Property>
  <Property name="encryptionService" type="EncryptionService">
    Кастомный экземпляр сервиса шифрования. Если не указан, используется сервис на основе LibSodium по умолчанию.
  </Property>
  <Property name="legacyV0Service" type="object">
    Параметры для обратной совместимости с форматом шифрования AES v0.
  </Property>
</Properties>

## Изменение поля `event.data` для шифрования

Шифруются только отдельные части данных события. По умолчанию — только поле `data.encrypted`.

Это можно настроить через опцию `eventEncryptionField`:

```ts
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY,
  eventEncryptionField: "sensitive",
});
```

## Режим только расшифровки

Для отключения шифрования при сохранении расшифровки установите `decryptOnly: true`. Полезно при переходе от шифрования, когда старые события ещё нужно обрабатывать.

```ts
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY,
  decryptOnly: true,
});
```

## Ротация ключей

Для попытки расшифровки с несколькими ключами установите опцию `fallbackDecryptionKeys`. Полезно при ротации ключей, так как старые события могли быть зашифрованы другим ключом:

```ts
// 1. Начать с текущим ключом
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY,
});

// 2. Развернуть все сервисы с новым ключом в качестве резервного для расшифровки
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY,
  fallbackDecryptionKeys: ["new"],
});

// 3. Развернуть все сервисы с новым ключом для шифрования
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY_V2,
  fallbackDecryptionKeys: ["current"],
});

// 4. После истечения срока действия всех данных со старым ключом исключить его
encryptionMiddleware({
  key: process.env.MY_ENCRYPTION_KEY_V2,
});
```

## Поддержка между языками

Это промежуточное ПО совместимо с промежуточным ПО шифрования в Python SDK. Зашифрованные события можно отправлять из Python и расшифровывать в TypeScript, и наоборот.
