# Интеграция почтовых событий через вебхуки Resend

<ImageTheme light={ResendLogo} alt="Логотип Resend"/>

[Вебхуки Resend](https://resend.com/docs/dashboard/webhooks/introduction) можно использовать для реализации функционала в приложении на основе изменений статуса электронных писем. В этом руководстве вы узнаете:

- Какие события вебхуков предлагает Resend.
- Как настроить Inngest для получения событий вебхуков Resend.
- Как определять функции Inngest в приложении, используя события Resend.
- Как создать динамическую маркетинговую цепочку (drip campaign), реагирующую на поведение пользователя.

Для прохождения этого руководства вам понадобятся аккаунты [Resend](https://resend.com/) и [Inngest](/), а также настроенный Inngest в вашей кодовой базе ([инструкция](/docs/getting-started/nextjs-quick-start)).

## Вебхуки Resend

Resend использует вебхуки для отправки в ваше приложение уведомлений в реальном времени об отправляемых вами письмах. Доступны следующие типы событий:

- `email.sent` — API-запрос прошел успешно, и Resend попытается доставить сообщение на почтовый сервер получателя.
- `email.bounced` — почтовый сервер получателя окончательно отклонил письмо.
- `email.delivery_delayed` — письмо не удалось доставить сразу (например, если почтовый ящик получателя переполнен или на принимающем сервере возникла временная проблема).
- `email.delivered` — Resend успешно доставил письмо на почтовый сервер получателя.
- `email.complained` — получатель пометил доставленное письмо как спам.
- `email.opened` — получатель открыл письмо.
- `email.clicked` — получатель кликнул по ссылке в письме.

Эти события позволяют реализовать ответную логику на основе изменения статуса письма. Например:

- Если письмо вернулось (bounced), удалить адрес из списка рассылки или пометить его в базе данных.
- Создать динамическую цепочку писем на основе поведения получателя.
- Создать отчеты об инцидентах или удержании (retention) для ваших кампаний.

### Работа с вебхуками Resend

Вы можете обрабатывать события вебхуков напрямую в бэкенде через эндпоинт или использовать такой инструмент, как [Inngest](/docs), который гарантирует надежное выполнение функций. Inngest предоставляет такие возможности, как:

- Встроенная очередь для надежного выполнения длительных функций.
- [Управление конкурентностью](/docs/guides/concurrency) для обработки всплесков нагрузки без перегрузки API или базы данных.
- Запуск нескольких функций от одного события ([fan-out jobs](/docs/guides/fan-out-jobs)).
- Реализация [отложенного выполнения кода](/docs/guides/multi-step-functions) через заданный период.
- [Устранение дребезга событий (debouncing)](/docs/reference/functions/debounce) для минимизации дублирующей обработки.

Проще говоря, использование Inngest делает ваше приложение более отказоустойчивым, масштабируемым и упрощает восстановление после инцидентов.

## Получение событий вебхуков Resend в Inngest

Давайте подключим Resend к Inngest.

1. Настройте вебхук в Inngest. Для этого в Inngest Cloud перейдите на страницу **[Manage → Webhooks](https://app.inngest.com/env/production/manage/webhooks)** и нажмите кнопку **Create Webhook** справа.

![Сайт Inngest Cloud на пустой странице Webhooks.](/assets/docs/guides/resend-webhook-events/inngest-1-no-webhooks.png)

1. В модальном окне укажите имя вебхука (например, “Resend”):

![Модальное окно с заголовком: "Create a New Webhook". Выбрано имя "Resend" для нового вебхука.](/assets/docs/guides/resend-webhook-events/inngest-2-new-webhook.png)

Теперь вы увидите страницу вашего вебхука с URL в верхней части:

![Страница вебхука в Inngest Cloud.](/assets/docs/guides/resend-webhook-events/inngest-3-webhook-page.png)

1. Вставьте следующую [функцию трансформации](/docs/platform/webhooks#defining-a-transform-function) в область **Transform Event**:

```jsx
function transform(evt, headers = {}, queryParams = {}) {
  return {
    // Добавляем префикс к имени события
    name: `resend/${evt.type}`,
    data: evt.data,
   };
};
```

Функция трансформации преобразует входящие данные в формат полезной нагрузки Inngest и добавит префикс `resend/` ко всем событиям.

Затем нажмите кнопку **Save Transformed Changes**, чтобы сохранить функцию.

![Страница вебхука в Inngest Cloud с видом Transform Event.](/assets/docs/guides/resend-webhook-events/inngest-4-transform-function.png)

Ваш вебхук в Inngest настроен!

## Написание вашей первой функции Inngest

*Обратите внимание: этот пример предполагает использование TypeScript и Next.js, и что Inngest уже добавлен в проект. Если вы используете Inngest впервые, следуйте [руководству по быстрому старту](/docs/getting-started/nextjs-quick-start).*

После настройки Inngest в коде вы можете написать функцию, которая запускается при каждом поступлении определенного события. Например, если отправленное пользователю письмо вернулось (bounce), вы можете пометить email пользователя как невалидный в базе данных:

```tsx
import db from "./database";

const invalidateUserEmail = inngest.createFunction(
  { id: 'invalidate-user-email', triggers: { event: 'resend/email.bounced' } },
  async ({ event }) => {
    const email = event.data.to[0];
    const user = await db.users.byEmail(email);
    if (user) {
      user.email_status = "invalid";
      await db.users.update(user);
    }
  }
)
```

Теперь, когда вы увидели, как обрабатывать события Resend, давайте рассмотрим более сложный пример.

## Создание функции для отправки письма

Предположим, вы хотите отправлять пользователю приветственное письмо при регистрации в приложении.

Сначала создайте вспомогательную функцию `sendEmail`, адаптировав пример из [Resend Next.js Quickstart](https://resend.com/docs/send-with-nextjs):

```tsx
// resend.ts

import { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendEmail(
  to: string,
  subject: string,
  content: React.ReactElement
) {
  const { data, error } = await resend.emails.send({
    from: 'Acme <noreply@acme.dev>',
    to: [to],
    subject,
    react: content
  });

  if (error) {
    throw error;
  }

	return data;
};

```

Теперь вы можете отправлять письма! Давайте применим эту функцию.

Пример ниже предполагает, что приложение получает событие `app/signup.completed` при регистрации нового пользователя:

```tsx

import { sendEmail } from "./resend";

const sendWelcomeEmail = inngest.createFunction(
  { id: 'send-welcome-email', triggers: { event: 'app/signup.completed' } },
  async ({ event }) => {
	  const { user } = event.data;
    await sendEmail(user.email, "Welcome to Acme", (
      <div>
        <h1>Добро пожаловать в ACME, {user.firstName}</h1>
      </div>
    ));
  }
)
```

Освоив основы отправки писем через Resend из функции Inngest, вы можете создавать еще более сложные сценарии.

## Отправка отложенного последующего письма

Каждый обработчик функции Inngest получает дополнительный [объект `step`](/docs/reference/functions/step-sleep-until), который предоставляет инструменты для создания детальных рабочих процессов. Использование `step.run` позволяет инкапсулировать определенный код, который будет автоматически перезапускаться при ошибках, гарантируя, что проблемы в одной части функции не заставят всю функцию выполняться заново. Также доступны [другие инструменты, такие как `step.sleep`](/docs/reference/functions/step-sleep).

Код ниже отправляет приветственное письмо, затем использует `step.sleep`, чтобы подождать три дня перед отправкой письма с предложением пробного периода:

```tsx
const sendOnboardingEmails = inngest.createFunction(
  { id: 'onboarding-emails', triggers: { event: 'app/signup.completed' } },
  async ({ event, step }) => { // ← step доступен в аргументах обработчика
    const { user } = event.data
	  const { email, first_name } = user

    await step.run('welcome-email', async () => {
      await sendEmail(email, "Welcome to ACME", (
        <div>
          <h1>Добро пожаловать в ACME, {firstName}</h1>
        </div>
      ));
    })

   // ждем 3 дня перед вторым письмом
   await step.sleep('wait-3-days', '3 days')

   await step.run('trial-offer-email', async () => {
     await sendEmail(email, "Бесплатный пробный период ACME Pro", (
        <div>
          <h1>Привет, {firstName}, попробуйте наши Pro-функции бесплатно в течение 30 дней</h1>
        </div>
      ));
    })
  }
)
```

Это удобно, но можно сделать еще лучше. Так как Resend отправляет события о доставке, открытии и кликах, вы можете создавать динамические кампании, адаптированные под каждого пользователя.

## Создание динамической цепочки писем (drip campaign)

Допустим, вы хотите создать следующую кампанию:

- Отправить каждому пользователю приветственное письмо при регистрации.
- Если получено событие `resend/email.clicked` (пользователь кликнул по ссылке в письме), подождать день и отправить советы для продвинутых пользователей.
- В противном случае подождать до 3 дней и отправить стандартное предложение пробного периода, но только если пользователь еще не перешел на платный тариф.

```tsx
const signupDripCampaign = inngest.createFunction(
  { id: "signup-drip-campaign", triggers: { event: "app/signup.completed" } },
  async ({ event, step }) => {
    const { user } = event.data;
    const { email, first_name } = user
    const welcome = "Welcome to ACME";

    const { id: emailId } = await step.run("welcome-email", async () => {
      return await sendEmail(
        email,
        welcome,
        <div>
          <h1>Добро пожаловать в ACME, {user.firstName}</h1>
        </div>
      );
    });

    // Ждем до 3 дней, пока пользователь откроет письмо и кликнет по любой ссылке в нем
    const clickEvent = await step.waitForEvent("wait-for-engagement", {
      event: "resend/email.clicked",
			if: `async.data.email_id == ${emailId}`,
      timeout: "3 days",
    });

    // если пользователь кликнул в письме, отправляем советы для продвинутых пользователей
    if (clickEvent) {
      await step.sleep("delay-power-tips-email", "1 day");
      await step.run("send-power-user-tips", async () => {
        await sendEmail(
          email,
          "Раскройте все возможности ACME",
          <h1>
            Привет, {firstName}, вот советы, как получить максимум от ACME
          </h1>
        );
      });

      // ждем еще один день перед отправкой предложения пробного периода
      await step.sleep("delay-trial-email", "1 day");
    }

    // проверяем, не перешел ли пользователь уже на тариф pro
    const dbUser = db.users.byEmail(email);

    if (dbUser.plan !== "pro") {
      // отправляем предложение пробного периода
      await step.run("trial-offer-email", async () => {
        await sendEmail(
          email,
          "Бесплатный пробный период ACME Pro",
          <h1>
            Привет, {firstName}, попробуйте наши Pro-функции бесплатно в течение 30 дней
          </h1>
        );
      });
    }
  }
);
```

Вуаля! Вы создали динамическую маркетинговую кампанию, в которой последующие письма зависят от поведения пользователя.

## Тестирование событий вебхуков с помощью Inngest Dev Server

При локальной разработке вы можете использовать Inngest Dev Server для запуска и тестирования функций на своей машине. Чтобы запустить сервер, выполните в директории проекта следующую команду:

```bash
npx --ignore-scripts=false inngest-cli@latest dev
```

Откройте в браузере [http://localhost:8288](http://localhost:8288/), чтобы увидеть интерфейс разработки.

Чтобы переслать и быстро протестировать события из Inngest Cloud на локальный Dev Server, перейдите в [Inngest Cloud](https://app.inngest.com/env/production/events). Выберите вкладку **Events**. Выберите любое событие, перейдите в **Logs** в боковой панели и выберите **Send to Dev Server**.

![Вкладка Events в Inngest Cloud. Область кода содержит кнопку: "Send to the Dev Server".](/assets/docs/guides/resend-webhook-events/inngest-6-cloud-dev-server.png)

Теперь вы увидите событие во вкладке **Stream** локального Dev Server вместе с функциями, которые оно запустило.

![Вкладка Stream в Inngest Dev Server с событием.](/assets/docs/guides/resend-webhook-events/inngest-7-event-stream-1.png)

Здесь вы можете выбрать событие, повторить его для перезапуска функций или отредактировать и повторить, чтобы протестировать различные типы данных в событии.

![Детали события Resend.](/assets/docs/guides/resend-webhook-events/inngest-8-event-details.png)
