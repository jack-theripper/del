# Несколько триггеров и маски (wildcards)

Функции Inngest можно настроить на запуск по нескольким событиям или расписаниям.

Использование нескольких триггеров полезно для выполнения одной и той же логики для широкого спектра событий или для того, чтобы гарантировать, что задача также запускается по расписанию. Например, выполнение проверки целостности каждое утро или по запросу через событие.

Несколько триггеров можно настроить с помощью [списка триггеров](#multiple-triggers) или [масок в именах событий (wildcard triggers)](#wildcard-event-triggers).

## Несколько триггеров

Функции поддерживают до 10 уникальных триггеров. Это позволяет явно сопоставлять несколько событий или расписаний. Перекрывающиеся расписания будут дедуплицированы — подробнее об этом в разделе [«Перекрывающиеся cron-расписания»](#overlapping-crons).

<CodeGroup>
```ts {{ title: "TypeScript" }}
inngest.createFunction(
  {
    id: "resync-user-data",
    triggers: [
      { event: "user.created" },
      { event: "user.updated" },
      { cron: "0 5 * * *" }, // Каждое утро в 5:00
    ],
  },
  async ({ event, step }) => {
    // ...
  },
);
```
```go {{ title: "Go" }}
inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{Name: "resync-user-data"},
	inngestgo.MultipleTriggers{
		inngestgo.EventTrigger("user.created", nil),
		inngestgo.EventTrigger("user.updated", nil),
		inngestgo.CronTrigger("0 5 * * *"),
	},
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		// ...
		return nil, nil
	},
)
```
```py {{ title: "Python" }}
@inngest_client.create_function(
    fn_id="resync-user-data",
    trigger=[
        inngest.TriggerEvent(event="user.created"),
        inngest.TriggerEvent(event="user.updated"),
        inngest.TriggerCron(cron="0 5 * * *")
    ],
)
def my_handler(ctx: inngest.Context) -> None:
     # ...
```
</CodeGroup>

## Триггеры событий с использованием масок (wildcards)

Триггеры событий можно настраивать с использованием масок для сопоставления нескольких событий. Это полезно для обработки целых групп событий, например, при пересылке событий в другую систему или в ETL-процессах реального времени.

Маски можно использовать после любого символа `/` или `.` для сопоставления групп событий. Вот несколько примеров:

* `app/*` сопоставляется с любым событием с префиксом `app`, например `app/user.created` и `app/blog.post.published`.
* `app/user.*` сопоставляется с любым событием с префиксом `app/user.`, например `app/user.created` и `app/user.updated`.
* `app/blog.post.*` сопоставляется с любым событием с префиксом `app/blog.post.`, например `app/blog.post.published`.

<Info>
Маски нельзя использовать после любых символов, отличных от `/` и `.`, а также в середине шаблона. Таким образом, маски внутри слова, такие как `app/user.update*`, или в середине пути, например `app/blog.*.published`, не поддерживаются.
</Info>

### Определение типов для триггеров с масками

Чтобы определить типы для триггеров с масками, вам необходимо явно их указать:

```ts
import { eventType, Inngest } from "inngest";

const blogPostCreated = eventType({
  name: "app/blog.post.created",
  data: {} as {
    postId: string;
    authorId: string;
    createdAt: string;
  },
});

const blogPostPublished = eventType({
  name: "app/blog.post.published",
  data: {} as {
    postId: string;
    authorId: string;
    publishedAt: string;
  },
});

const inngest = new Inngest({
  id: "my-app",
});

inngest.createFunction(
  { id: "blog-updates-to-slack", triggers: [{ event: "app/blog.post.*" }] },
  async ({ event, step }) => {
    // ...
  },
);
```

## Определение типа события

В обработчике функции с несколькими триггерами событие, запустившее функцию, можно определить с помощью свойства `event.name`.

<CodeGroup>
```ts {{ title: "TypeScript" }}
async ({ event }) => {
  //      ^? type event: EventA | EventB | InngestScheduledEvent | InngestFnInvoked
  if (event.name === "a") {
    // Тип `event` сужается только до события `a`
  } else if (event.name === "b") {
    // Тип `event` сужается только до события `b`
  } else {
    // Тип `event` сужается только до события `inngest/function.invoked`
  }
}
```
```go {{ title: "Go" }}
func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
	switch input.Event.Name {
	case "a":
		// Обработка события A
	case "b":
		// Обработка события B
	case "inngest/function.invoked":
		// Обработка события вызова функции
	}
	return nil, nil
},
```
</CodeGroup>

Обратите внимание, что пакеты `events` могут содержать множество различных событий; вам потребуется проверять структуру каждого события в пакете индивидуально.

## Перекрывающиеся cron-расписания

Если ваша функция определяет несколько cron-триггеров, расписания иногда могут перекрываться. Например, cron `0 * * * *`, который запускается каждый час, и cron `*/30 * * * *`, который запускается каждые полчаса, будут перекрываться в начале каждого часа.

Для каждой конкретной секунды будет запущен только один cron-запуск, поэтому в приведенном выше случае одна задача будет запускаться каждые полчаса.
