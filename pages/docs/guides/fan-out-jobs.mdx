# Fan-out (один ко многим)

Паттерн fan-out («разветвление») позволяет отправить одно событие и запустить несколько функций параллельно (один ко многим). Ключевыми преимуществами этого подхода являются:

* **Надежность**: логика каждой функции выполняется независимо, что означает, что проблема в одной функции не повлияет на другие.
* **Производительность**: поскольку функции запускаются параллельно, вся работа будет выполнена быстрее, чем при последовательном запуске.

Примером использования fan-out может служить ситуация, когда пользователь регистрируется в вашем продукте. В этом сценарии вы можете захотеть:
1. Отправить приветственное письмо.
2. Запустить пробный период в Stripe.
3. Добавить пользователя в CRM.
4. Добавить email пользователя в список рассылки.

Паттерн fan-out также полезен в распределенных системах, где одно событие потребляется функциями, работающими в разных приложениях.

## Как реализовать fan-out на несколько функций

<LanguageSelector>

<LanguageSection show="typescript">

Поскольку Inngest работает на основе событий, реализация fan-out так же проста, как определение нескольких функций, использующих один и тот же триггер события. Давайте возьмем приведенный выше пример регистрации пользователя и реализуем его в Inngest.

Сначала настройте обработчик маршрута `/signup`, чтобы отправлять событие в Inngest при регистрации пользователя:

```ts {{ filename: "app/routes/signup/route.ts" }}
import { inngest } from '../inngest/client';

export async function POST(request: Request) {
  // ПРИМЕЧАНИЕ — этот код упрощен для примера:
  const { email, password } = await request.json();
  const user = await createUser({ email, password });
  await createSession(user.id);

  // Отправляем событие в Inngest
  await inngest.send({
    name: 'app/user.signup',
    data: {
      user: {
        id: user.id,
        email: user.email,
      },
    },
  });

  redirect('https://myapp.com/dashboard');
}
```

Теперь, благодаря этому событию, любая функция, использующая `"app/user.signup"` в качестве триггера события, будет вызвана автоматически.

Затем определите две функции: `sendWelcomeEmail` и `startStripeTrial`. Как вы видите ниже, обе функции используют один и тот же триггер события, но выполняют разную работу.

```ts {{ filename: "inngest/functions.ts" }}
const sendWelcomeEmail = inngest.createFunction(
  { id: 'send-welcome-email', triggers: { event: 'app/user.signup' } },
  async ({ event, step }) => {
    await step.run('send-email', async () => {
      await sendEmail({ email: event.data.user.email, template: 'welcome' });
    });
  }
)

const startStripeTrial = inngest.createFunction(
  { id: 'start-stripe-trial', triggers: { event: 'app/user.signup' } },
  async ({ event }) => {
    const customer = await step.run('create-customer', async () => {
      return await stripe.customers.create({ email: event.data.user.email });
    });
    await step.run('create-subscription', async () => {
      return await stripe.subscriptions.create({
        customer: customer.id,
        items: [{ price: 'price_1MowQULkdIwHu7ixraBm864M' }],
        trial_period_days: 14,
      });
    });
  }
)
```

Теперь вы успешно реализовали fan-out в нашем приложении. Каждая функция будет выполняться независимо и параллельно. Если одна функция даст сбой, это не помешает работе других.

Другие преимущества fan-out включают:

* **Массовый повтор (Bulk Replay)**: если сторонний API на время выйдет из строя (например, ваш почтовый сервис), вы можете использовать [Replay (Повтор)](/docs/platform/replay), чтобы выборочно перезапустить все функции, которые завершились ошибкой, без необходимости перезапускать весь поток регистрации.
* **Тестирование**: каждую функцию можно тестировать изолированно, без необходимости запускать весь поток регистрации.
* **Новые функции или рефакторинг**: поскольку каждая функция независима, вы можете добавлять новые функции или рефакторить существующие, не затрагивая несвязанный код.
* **Запуск функций в разных кодовых базах**: если у вас несколько кодовых баз, даже на разных языках программирования (например, [Python](/docs/reference/python) или [Go](https://pkg.go.dev/github.com/inngest/inngestgo)), вы можете запускать функции в обеих кодовых базах от одного события.
