# Отправка событий из функций

В некоторых рабочих процессах или пайплайн-функциях вам может потребоваться транслировать события изнутри функции для запуска _других_ функций. Этот паттерн полезен, когда:

* Вы хотите разделить логику на отдельные функции, которые можно повторно использовать в вашей системе
* Вы хотите отправить событие для [fan-out](/docs/guides/fan-out-jobs) к нескольким другим функциям
* Ваша функция обрабатывает множество элементов, которые вы хотите обрабатывать в параллельных функциях
* Вы хотите [делегировать задачи подагентам асинхронно](/docs/ai-patterns/sub-agent-delegation#delegate-asynchronously-with-stepsendevent) в системе AI-агентов
* Вы хотите [отменить](/docs/guides/cancel-running-functions) другую функцию
* Вы хотите отправить данные в другую функцию, [ожидающую событие](/docs/reference/functions/step-wait-for-event)

Если вашей функции нужно обработать результат другой функции или дождаться её завершения, вместо этого следует использовать [прямой вызов функции](/docs/guides/invoking-functions-directly).

## Как отправлять события из функций

<LanguageSelector>

<LanguageSection show="typescript">

Для отправки событий из функций используйте [`step.sendEvent()`](/docs/reference/functions/step-send-event). Этот метод принимает одно событие или массив событий. Пример ниже использует массив событий.

Это пример [запланированной функции](/docs/guides/scheduled-functions), которая отправляет еженедельное письмо об активности всем пользователям.

Сначала функция получает всех пользователей, затем для каждого пользователя создаёт событие `"app/weekly-email-activity.send"`, и наконец отправляет все события в Inngest.

```ts
import { GetEvents, Inngest } from "inngest";

const inngest = new Inngest({ id: "signup-flow" });
type Events = GetEvents<typeof inngest>;

export const loadCron = inngest.createFunction(
  { id: "weekly-activity-load-users", triggers: { cron: "0 12 * * 5" } },
  async ({ event, step }) => {
    // Получаем всех пользователей
    const users = await step.run("fetch-users", async () => {
      return fetchUsers();
    });

    // Для каждого пользователя отправляем событие. Inngest поддерживает пакеты событий,
    // если весь payload занимает менее 512 КБ.
    const events = users.map<Events["app/weekly-email-activity.send"]>(
      (user) => {
        return {
          name: "app/weekly-email-activity.send",
          data: {
            ...user,
          },
          user,
        };
      }
    );

    // Отправляем все события в Inngest, что запускает все функции,
    // слушающие указанные имена событий.
    await step.sendEvent("fan-out-weekly-emails", events);

    // Возвращаем количество запущенных пользователей.
    return { count: users.length };
  }
);
```

Далее создайте функцию, которая слушает событие `"app/weekly-email-activity.send"`. Эта функция будет запускаться для каждого пользователя, которому было отправлено событие в предыдущей функции.

```ts
export const sendReminder = inngest.createFunction(
  { id: "weekly-activity-send-email", triggers: { event: "app/weekly-email-activity.send" } },
  async ({ event, step }) => {
    const data = await step.run("load-user-data", async () => {
      return loadUserData(event.data.user.id);
    });

    await step.run("email-user", async () => {
      return sendEmail(event.data.user, data);
    });
  }
);
```

Каждая из этих функций будет выполняться параллельно и отдельно повторяться при ошибке, что даёт более быструю и надёжную систему.

<Callout>
  💡 **Совет**: При запуске большого количества параллельных функций рекомендуется настроить ограничения `concurrency`, чтобы не перегрузить вашу систему. Подробнее см. в [руководстве по параллелизму](/docs/guides/concurrency).
</Callout>

### Почему `step.sendEvent()`, а не `inngest.send()`?

Используя [`step.sendEvent()`](/docs/reference/functions/step-send-event), SDK Inngest может автоматически добавлять контекст и трассировку, связывающие события с текущим запуском функции. При использовании [`inngest.send()`](/docs/reference/events/send) контекст запуска функции отсутствует.

Для отправки событий из функций используйте [`step.send_event()`](/docs/reference/python/steps/send-event). Этот метод принимает одно событие или массив событий. Пример ниже использует массив событий.

Это пример [запланированной функции](/docs/guides/scheduled-functions), которая отправляет еженедельное письмо об активности всем пользователям.

Сначала функция получает всех пользователей, затем для каждого пользователя создаёт событие `"app/weekly-email-activity.send"`, и наконец отправляет все события в Inngest.

```py
import inngest
from src.inngest.client import inngest_client

@inngest_client.create_function(
    fn_id="weekly-activity-load-users",
    trigger=inngest.TriggerCron(cron="0 12 * * 5")
)
async def load_cron(ctx: inngest.Context):
    # Получаем всех пользователей
    async def fetch():
        return await fetch_users()

    users = await ctx.step.run("fetch-users", fetch)

    # Для каждого пользователя отправляем событие. Inngest поддерживает пакеты событий,
    # если весь payload занимает менее 512 КБ.
    events = []
    for user in users:
        events.append(
            inngest.Event(
                name="app/weekly-email-activity.send",
                data={
                    **user,
                    "user": user
                }
            )
        )

    # Отправляем все события в Inngest, что запускает все функции,
    # слушающие указанные имена событий.
    await ctx.step.send_event("fan-out-weekly-emails", events)

    # Возвращаем количество запущенных пользователей.
    return {"count": len(users)}
```

Далее создайте функцию, которая слушает событие `"app/weekly-email-activity.send"`. Эта функция будет запускаться для каждого пользователя, которому было отправлено событие в предыдущей функции.

```py
@inngest_client.create_function(
    fn_id="weekly-activity-send-email",
    trigger=inngest.TriggerEvent(event="app/weekly-email-activity.send")
)
async def send_reminder(ctx: inngest.Context):
    async def load_data():
        return await load_user_data(ctx.event.data["user"]["id"])

    data = await ctx.step.run("load-user-data", load_data)

    async def send():
        return await send_email(ctx.event.data["user"], data)

    await ctx.step.run("email-user", send)
```

Каждая из этих функций будет выполняться параллельно и отдельно повторяться при ошибке, что даёт более быструю и надёжную систему.

## Параллельные функции vs. параллельные шаги

Похожая техника — запуск нескольких шагов параллельно (см. [руководство по параллелизму шагов](/docs/guides/step-parallelism)). Ключевые отличия:

* Оба паттерна выполняют код параллельно
* При параллельных шагах вы можете получить доступ к результату каждого шага, тогда как в примере выше это невозможно
* Параллельные шаги ограничены 1000 шагами, однако с помощью паттерна отправки событий можно запустить столько функций, сколько нужно
* Разделённые функции можно тестировать и [воспроизводить](/docs/platform/replay) по отдельности, тогда как параллельные шаги можно тестировать только как единое целое
* При необходимости можно легко повторить отдельные функции после постоянной ошибки, тогда как при постоянном сбое шага (после повторных попыток) сама функция завершится с ошибкой и остановится

## Отправка событий vs. вызов функций

Связанный паттерн — прямой вызов внешних функций вместо их запуска через событие. См. руководство [Прямой вызов функций](/docs/guides/invoking-functions-directly). Ключевые отличия:

* Отправка событий из функций лучше подходит для параллельной обработки независимых задач, а вызов — для координации взаимозависимых функций
* Отправка событий может выполняться пакетами, тогда как вызов позволяет вызывать только одну функцию за раз
* Отправку событий можно комбинировать с [fan-out](/docs/guides/fan-out-jobs) для запуска нескольких функций из одного события
* В отличие от вызова, отправка событий не возвращает результат вызванной функции
