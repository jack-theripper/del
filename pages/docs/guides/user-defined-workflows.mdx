# Создание рабочих процессов, настраиваемых пользователями

Современные пользователи требуют кастомизации и интеграций от каждого продукта. Ваши пользователи могут захотеть, чтобы ваш продукт поддерживал пользовательские рабочие процессы для автоматизации ключевых действий.
Используйте наш [Workflow Kit](/docs/reference/workflow-kit) для добавления мощных функций пользовательских рабочих процессов в ваш продукт.

Workflow Kit от Inngest поставляется как full-stack пакет ([`@inngest/workflow-kit`](https://npmjs.com/package/@inngest/workflow-kit)), призванный упростить разработку пользовательских рабочих процессов как на фронтенде, так и на бэкенде:

Мы хотим предоставить нашим пользователям следующие задачи AI-автоматизации:

**Задачи проверки**
- Добавить оглавление: _задача, использующая OpenAI для вставки оглавления в запись блога_
- Выполнить проверку грамматики: _задача, использующая OpenAI для исправления грамматических ошибок_

**Задачи создания контента для соцсетей**
- Сгенерировать посты для LinkedIn: _задача, использующая OpenAI для генерации твитов_
- Сгенерировать посты для Twitter: _задача, использующая OpenAI для генерации поста в LinkedIn_

Наши пользователи смогут комбинировать эти задачи для создания собственных рабочих процессов.

### 1. Добавление определений задач в приложение

После [установки и настройки Inngest](/docs/getting-started/nextjs-quick-start?ref=docs-guide-user-defined-workflows) в нашем приложении Next.js мы создадим следующий файл [определений Workflow Actions](/docs/reference/workflow-kit/actions):

```ts {{ title: "lib/inngest/workflowActions.ts" }}
import { type PublicEngineAction } from "@inngest/workflow-kit";

export const actions: PublicEngineAction[] = [
  {
    kind: "add_ToC",
    name: "Add a Table of Contents",
    description: "Add an AI-generated ToC",
  },
  {
    kind: "grammar_review",
    name: "Perform a grammar review",
    description: "Use OpenAI for grammar fixes",
  },
  {
    kind: "wait_for_approval",
    name: "Apply changes after approval",
    description: "Request approval for changes",
  },
  {
    kind: "apply_changes",
    name: "Apply changes",
    description: "Save the AI revisions",
  },
  {
    kind: "generate_linkedin_posts",
    name: "Generate LinkedIn posts",
    description: "Generate LinkedIn posts",
  },
  {
    kind: "generate_tweet_posts",
    name: "Generate Twitter posts",
    description: "Generate Twitter posts",
  },
];

```

### 2. Обновление схемы базы данных

Чтобы пользователи могли настраивать рабочие процессы, мы создадим следующую таблицу `workflows`.

Таблица `workflows` хранит [объект экземпляра рабочего процесса](/docs/reference/workflow-kit/workflow-instance), содержащий информацию о том, как пользователь упорядочил различные выбранные [действия рабочего процесса](/docs/reference/workflow-kit/actions). Дополнительные столбцы добавлены для хранения свойств, специфичных для нашего приложения: названия и описания автоматизации, события, запускающего автоматизацию, и её статуса (`enabled`).


|Название столбца|Тип столбца|Описание|
|----------------|-----------|--------|
id| `bigint`|
name | `text`| _Название автоматизации_
description | `text`| _Краткое описание автоматизации_
workflow | `jsonb`| _[Объект экземпляра рабочего процесса](/docs/reference/workflow-kit/workflow-instance)_
enabled | `boolean`|
trigger | `text`| _Название [события Inngest](/docs/features/events-triggers), запускающего рабочий процесс_
created_at | `timestamp`|

После создания таблицы `workflows` добавим две записи [экземпляров рабочего процесса](/docs/reference/workflow-kit/workflow-instance):
- _«Когда запись блога публикуется»_: Получение проверки от AI
- _«Когда запись блога переходит на проверку»_: Действия для оптимизации распространения записей блога

с помощью следующего SQL-запроса на вставку:

```sql
INSERT INTO "public"."workflows" ("id", "created_at", "workflow", "enabled", "trigger", "description", "name") VALUES
	(2, '2024-09-14 20:19:41.892865+00', NULL, true, 'blog-post.published', 'Actions performed to optimize the distribution of blog posts', 'When a blog post is published'),
	(1, '2024-09-14 15:46:53.822922+00', NULL, true, 'blog-post.updated', 'Getting a review from AI', 'When a blog post is moved to review');
```

### 3. Добавление страницы редактора рабочего процесса

Имея готовые определения действий рабочего процесса и таблицу `workflows`, создадим новую страницу Next.js с редактором рабочего процесса.

Сначала добавим новую [страницу Next.js](https://nextjs.org/docs/app/building-your-application/routing/pages) для загрузки рабочего процесса и отображения редактора:

```tsx {{ title: "app/automation/[id]/page.tsx" }}
import { AutomationEditor } from "@/components/automation-editor";
import { createClient } from "@/lib/supabase/server";
import { notFound } from "next/navigation";

export const runtime = "edge";

export default async function Automation({
  params,
}: {
  params: { id: string };
}) {
  const supabase = createClient();
  const { data: workflow } = await supabase
    .from("workflows")
    .select("*")
    .eq("id", params.id!)
    .single();
  if (workflow) {
    return <AutomationEditor workflow={workflow} />;
  } else {
    notFound();
  }
}

```

Компонент `<AutomationEditor>` затем рендерится со следующими обязательными свойствами:
- `workflow={}`: [экземпляр рабочего процесса](/docs/reference/workflow-kit/workflow-instance), загруженный из базы данных вместе с
- `event={}`: имя события, запускающего рабочий процесс
- `availableActions={}`: [действия](/docs/reference/workflow-kit/actions#passing-actions-to-the-react-components-public-engine-action), которые пользователь может выбрать для создания автоматизации


```tsx {{ title: "src/components/automation-editor.ts" }}
import { useState } from "react";
import { Editor, Provider, Sidebar } from "@inngest/workflow-kit/ui";

import { actions } from "@/inngest/workflowActions";
import { type Workflow } from "@/lib/supabase/types";

import "@inngest/workflow-kit/ui/ui.css";
import "@xyflow/react/dist/style.css";

export const AutomationEditor = ({ workflow }: { workflow: Workflow }) => {
  const [workflowDraft, updateWorkflowDraft] =
    useState<typeof workflow>(workflow);

  return (
    <Provider
      key={workflowDraft?.id}
      workflow={workflowDraft?.workflow}
      trigger={{
        event: {
          name: workflowDraft.trigger,
        },
      }}
      availableActions={actions}
      onChange={(updated) => {
        updateWorkflowDraft({
          ...workflowDraft,
          workflow: updated,
        });
      }}
    >
      <Editor>
        <Sidebar position="right"/>
      </Editor>
    </Provider>
  );
};
```

[`<Provider>`](/docs/reference/workflow-kit/components-api) является [управляемым компонентом (Controlled Component)](https://react.dev/learn/sharing-state-between-components#controlled-and-uncontrolled-components), использующим объект `workflow={}` для обновления своего UI.

Каждое изменение, выполненное пользователем, будет вызывать коллбэк `onChange={}`. Этот коллбэк должен обновлять объект, передаваемый в пропс `workflow={}`, и может использоваться также для реализации механизма автосохранения.

<Callout>

Полная версия `<AutomationEditor>` [доступна на GitHub](https://github.com/inngest/workflow-kit/blob/main/examples/nextjs-blog-cms/components/automation-editor.tsx).

</Callout>

### 4. Реализация обработчиков действий рабочего процесса

Теперь реализуем логику наших задач автоматизации, создав новый файл в `lib/inngest` и начав с действия рабочего процесса «Добавить оглавление»:

```tsx {{ title: "lib/inngest/workflowActionHandlers.ts" }}
import OpenAI from "openai";
import { type EngineAction, type WorkflowAction } from "@inngest/workflow-kit";

import { type BlogPost } from "../supabase/types";

import { loadBlogPost } from "../loaders/blog-post";
import { createClient } from "../supabase/server";
import { actions } from "./workflowActions";


export const actions: EngineAction[] = [
  {
    // Добавить оглавление
    ...actionsDefinition[0],
    handler: async ({ event, step, workflowAction }) => {
      const supabase = createClient();

      const blogPost = await step.run("load-blog-post", async () =>
        loadBlogPost(event.data.id)
      );

      const aiRevision = await step.run("add-toc-to-article", async () => {
        const openai = new OpenAI({
          apiKey: process.env["OPENAI_API_KEY"], // This is the default and can be omitted
        });

        const prompt = `
        Please update the below markdown article by adding a Table of Content under the h1 title. Return only the complete updated article in markdown without the wrapping "\`\`\`".

        Here is the text wrapped with "\`\`\`":
        \`\`\`
        ${getAIworkingCopy(workflowAction, blogPost)}
        \`\`\`
        `;

        const response = await openai.chat.completions.create({
          model: process.env["OPENAI_MODEL"] || "gpt-3.5-turbo",
          messages: [
            {
              role: "system",
              content: "You are an AI that make text editing changes.",
            },
            {
              role: "user",
              content: prompt,
            },
          ],
        });

        return response.choices[0]?.message?.content || "";
      });

      await step.run("save-ai-revision", async () => {
        await supabase
          .from("blog_posts")
          .update({
            markdown_ai_revision: aiRevision,
            status: "under review",
          })
          .eq("id", event.data.id)
          .select("*");
      });
    },
    }
  },
];
```

Этот новый файл добавляет свойство `handler` к существующему действию _«Добавить оглавление»_.

[`handler()`](/docs/reference/workflow-kit/actions#handler-function-argument-properties) действия рабочего процесса имеет сигнатуру, аналогичную обработчикам функций Inngest, получая два ключевых аргумента: `event` и [`step`](/docs/reference/functions/create#step).

Наше действие _«Добавить оглавление»_ использует [step API](/docs/reference/functions/step-run) от Inngest для создания надёжных и повторяемых шагов, генерирующих и вставляющих оглавление.


<Callout>

Полная реализация всех действий рабочего процесса [доступна на GitHub](https://github.com/inngest/workflow-kit/blob/main/examples/nextjs-blog-cms/lib/inngest/workflowActionHandlers.ts).

</Callout>



### 5. Создание функции Inngest

После [реализации](https://github.com/inngest/workflow-kit/blob/main/examples/nextjs-blog-cms/lib/inngest/workflowActionHandlers.ts) всех обработчиков действий рабочего процесса мы можем создать экземпляр [`Engine`](/docs/reference/workflow-kit/engine) и передать его в выделенную [функцию Inngest](/docs/features/inngest-functions), которая будет запускать автоматизацию при срабатывании событий `"blog-post.updated"` и `"blog-post.published"`:

```tsx {{ title: "lib/inngest/workflow.ts" }}
import { Engine } from "@inngest/workflow-kit";

import { loadWorkflow } from "../loaders/workflow";
import { inngest } from "./client";
import { actionsWithHandlers } from "./workflowActionHandlers";

const workflowEngine = new Engine({
  actions: actionsWithHandlers,
  loader: loadWorkflow,
});

export default inngest.createFunction(
  {
    id: "blog-post-workflow",
    // Триггеры
    // - Когда запись блога переводится в статус «review»
    // - Когда запись блога публикуется
    triggers: [{ event: "blog-post.updated" }, { event: "blog-post.published" }],
  },
  async ({ event, step }) => {
    // При вызове `run` вызывается функция загрузчика с доступом к событию
    await workflowEngine.run({ event, step });
  }
);

```
