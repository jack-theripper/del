# Приоритет

Приоритет позволяет динамически выполнять одни запуски функций раньше или позже других на основе любых данных. Это позволяет приоритизировать одни задачи перед другими без необходимости создания отдельной очереди. Некоторые сценарии использования приоритета включают:

* Повышение приоритета на основе уровня подписки пользователя (например, бесплатные пользователи против платных).
* Гарантия того, что критически важные задачи выполняются раньше других задач в очереди.
* Приоритизация определенных задач во время онбординга для улучшения первого опыта пользователя.

## Как настроить приоритет

<CodeGroup forceTabs>
```ts
export default inngest.createFunction(
  {
    id: "ai-generate-summary",
    priority: {
      // Для корпоративных аккаунтов (enterprise) запуск функции будет приоритизирован
      // перед задачами, которые были поставлены в очередь до 120 секунд назад.
      // Для всех остальных аккаунтов функция будет запускаться без приоритета.
      run: "event.data.account_type == 'enterprise' ? 120 : 0",
    },
    triggers: { event: "ai/summary.requested" },
  },
  async ({ event, step }) => {
    // Эта функция будет приоритизирована на основе типа аккаунта
  }
);
```

### Справочник по конфигурации

* `run` — [выражение](/docs/guides/writing-expressions) динамического фактора, вычисляемое в секундах, по которому приоритизируется функция. Возврат положительного числа повысит этот приоритет перед другими задачами, уже находящимися в очереди. Возврат отрицательного числа отложит выполнение задачи на указанное количество секунд.

## Как работает приоритет

Функции планируются в очереди с приоритетами на основе времени, в которое они должны запуститься. По умолчанию все функции ставятся в очередь в текущее время (фактор `0`). Если для функции настроен `priority`, Inngest вычисляет выражение `run` для каждого нового запуска функции на основе данных входного события. Выражение `run` должно возвращать фактор в секундах (положительный или отрицательный) для корректировки приоритета запуска функции.

Выражения, возвращающие **положительное** число, **повышают приоритет** запуска функции перед другими задачами, уже находящимися в очереди, на указанное значение в секундах. Функция будет запущена раньше других задач, поставленных в очередь до этого количества секунд назад. Например, если запуск функции запланирован с фактором `120`, он будет выполнен раньше любых задач, поставленных в очередь в последние 120 секунд, при условии, что они всё еще находятся в очереди и не завершены.

Выражения, возвращающие **отрицательное** число, **откладывают** запуск функции на указанное значение в секундах.

### Практический пример

Предположим, у нас есть три задачи в очереди, каждая из которых была поставлена в очередь в следующее время:

```plaintext
Задачи:         [A,        B,        C       ]
Приоритет/Время: [12:00:10, 12:00:40, 12:02:10]
```

Если текущее время — `12:02:30`, и ставятся две новые задачи со следующими факторами `run`:

```plaintext
- Задача X: фактор 0
- Задача Y: фактор 120
```

Тогда Задача Y будет запущена раньше Задачи X. Задача Y также будет запущена раньше любых задач, запланированных на 120 секунд ранее. Очередь будет выглядеть так:

```plaintext
Задачи:         [A,        Y,        B,        C,        X     ]
Приоритет/Время: [12:00:10, 12:00:30, 12:00:40, 12:02:10, 12:02:30]
                          │                             │
                          └ 12:02:30 - 120s = 12:00:30  └ 12:02:30 - 0s = 12:02:30
```

Задача Y была успешно приоритизирована на фактор `120` секунд перед другими задачами в очереди.

## Сочетание с конкурентностью

Приоритизация наиболее полезна в сочетании с опцией управления потоком, ограничивающей пропускную способность, такой как [конкурентность (concurrency)](/docs/guides/concurrency). При ограничении пропускной способности задачи часто ждут в очереди, и приоритизация позволяет вам контролировать порядок выполнения задач в этом бэклоге.

## Ограничения

* Максимальный приоритет составляет `600` (секунд).
* Минимальный приоритет составляет `-600` (секунд).
* Несовместимо с [пакетной обработкой (batching)](/docs/guides/batching).
