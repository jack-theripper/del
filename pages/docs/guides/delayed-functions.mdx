# Отложенные функции

С помощью Inngest вы можете легко запускать функции в будущем. Существует три способа отложить выполнение функции:

- [**Пауза на определенную длительность**](#sleep-for-a-duration) — приостановка выполнения внутри функции на заданное время с помощью `step.sleep()`.
- [**Пауза до определенного времени**](#sleep-until-a-specific-time) — приостановка выполнения внутри функции до указанной даты/времени с помощью `step.sleepUntil()`.
- [**Планирование запуска функции на потом**](#schedule-a-function-for-later) — задержка первого вызова функции путем установки поля `ts` в событии.

Задержки могут составлять до одного года (до семи дней на бесплатном тарифе). Использование Inngest для откладывания функций дает ряд преимуществ:

- Работает с любым провайдером или платформой.
- Задержки надежны и сохраняются при перезагрузке сервера, использовании серверлесс-функций и повторных развертываниях.
- Вы можете планировать функции на далекое будущее.
- Серверлесс-функции полностью поддерживаются на всех платформах.
- Наш SDK обходит ограничения по времени выполнения (timeouts) серверлесс-функций на всех платформах.
- Вам никогда не нужно управлять очередями или списками ожидания вручную.

### Поддержка платформ

**Это работает на всех провайдерах и платформах**, независимо от того, используете ли вы серверлесс-функции или серверы (например, Express). **Это также позволяет обходить тайм-ауты серверлесс-функций** на всех платформах, так как вы можете установить паузу на более длительный срок, чем поддерживает ваш провайдер.

import { Inngest } from "inngest";

const inngest = new Inngest({ id: "signup-flow" });

    await step.run("do-some-work-in-the-future", async () => {

    });
  }
);

## Пауза до определенного времени

Вы можете приостановить выполнение функции до определенного момента времени, используя метод [`step.sleepUntil()`](/docs/reference/functions/step-sleep-until):


```ts
import { Inngest } from "inngest";

const inngest = new Inngest({ id: "signup-flow" });

export const fn = inngest.createFunction(
  { id: "send-signup-email", triggers: { event: "app/user.created" } },
  async ({ event, step }) => {
    await step.sleepUntil("wait-for-iso-string", "2023-04-01T12:30:00");

    // Вы также можете ждать до времени, указанного в данных события. Это позволяет
    // передавать время запуска задачи извне:
    await step.sleepUntil("wait-for-timestamp", event.data.run_at); // Предполагаем, что event.data.run_at — это метка времени.

    await step.run("do-some-work-in-the-future", async () => {
      // Это выполнится в указанное время.
    });
  }
);
```



Для получения дополнительной информации о `step.sleepUntil()` [прочтите справочник](/docs/reference/functions/step-sleep-until).

## Пауза до определенного времени

Вы можете приостановить выполнение функции до определенного момента времени, используя метод [`step.sleep_until()`](/docs/reference/python/steps/sleep-until):


```python
import inngest
from src.inngest.client import inngest_client

inngest_client = inngest.Inngest(
    app_id="my-app",
)

@inngest_client.create_function(
    fn_id="send-signup-email", 
    trigger=inngest.TriggerEvent(event="app/user.created")
)
async def send_signup_email(ctx: inngest.Context):
    async def send_email():
        await sesclient.send_email(
            to=ctx.event.data["user_email"],
            subject="Welcome to Inngest!",
            message="..."
        )
    
    await ctx.step.run("send-the-user-a-signup-email", send_email)
    
    await ctx.step.sleep_until("wait-for-the-future", "2023-02-01T16:30:00")

    async def future_work():
        # Этот код выполнится в будущем автоматически
        pass

    await ctx.step.run("do-some-work-in-the-future", future_work)
```



Для получения дополнительной информации о `step.sleep_until()` [прочтите справочник](/docs/reference/python/steps/sleep-until).

## Планирование запуска функции на потом

Вы также можете отложить время запуска функции, установив поле `ts` в [полезной нагрузке события](/docs/events#event-payload-format) на метку времени в будущем (миллисекунды с начала эпохи Unix). В отличие от `step.sleep()` или `step.sleepUntil()`, которые добавляют задержки _внутри_ функции, поле `ts` задерживает сам _начало_ выполнения функции.

Например, чтобы запланировать запуск функции через 5 минут:

```typescript
await inngest.send({
  name: "notifications/reminder.scheduled",
  data: {
    user: { email: "johnny.utah@fbi.gov" },
    message: "Не забудь поймать волну в 15:00",
  },
  // Указываем метку времени для времени через 5 минут в будущем:
  ts: Date.now() + 5 * 60 * 1000,
});
```

Полный пример см. в руководстве [Планирование разовой функции](/docs/examples/scheduling-one-off-function).

### `step.sleep()` и `step.sleepUntil()`

С помощью `step.sleep()` и `step.sleepUntil()` **сама функция управляет временем своего запуска**. Вы контролируете поток выполнения кода, вызывая `sleep` или `sleepUntil` непосредственно внутри функции, вместо того чтобы использовать очередь для управления временем выполнения. Это позволяет держать логику вместе и упрощает изменение кода.

Inngest *останавливает выполнение функции* на указанное время. Когда вы вызываете `step.sleep` или `step.sleepUntil`, функция автоматически прекращает выполнение любой дальнейшей работы. Затем функция сообщает исполнителю Inngest, что она должна быть вызвана снова в будущем. Мы повторно вызываем функцию на следующем шаге, пропуская всю предыдущую работу. Именно так мы обходим ограничения по времени выполнения серверлесс-функций и обеспечиваем работу при перезагрузках или повторных развертываниях сервера.

### Поле `ts` в событии

Когда вы устанавливаете поле `ts` в событии на метку времени Unix в будущем, Inngest откладывает вызов функции до этого времени. Сама функция затем выполняется в обычном режиме — задержка происходит перед ее началом, а не внутри нее. Это полезно, когда задержка известна в момент отправки события и вам не нужно выполнять какую-либо логику перед ожиданием.
