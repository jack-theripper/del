# Пакетная обработка событий (Batching)

Пакетная обработка позволяет функции обрабатывать несколько событий за один запуск. Это полезно для систем с высокой нагрузкой, где эффективнее обработать группу событий вместе, чем каждое событие по отдельности. Некоторые варианты использования пакетной обработки включают:

* Уменьшение количества запросов к внешнему API, поддерживающему пакетные операции.
* Создание пакета записей в базе данных для уменьшения количества транзакций.
* Уменьшение количества запросов к вашему [приложению Inngest](/docs/apps) для повышения производительности или снижения затрат на серверлесс-инфраструктуру.

```ts {{ title: "TypeScript"}}
inngest.createFunction(
  {
    id: "record-api-calls",
    batchEvents: {
      maxSize: 100,
      timeout: "5s",
      key: "event.data.user_id", // Необязательно: группировать события в пакеты по ID пользователя
      if: "event.data.account_type == \"free\"", // Необязательно: пакетировать только события от бесплатных аккаунтов
    },
    triggers: { event: "log/api.call" },
  },
  async ({ events, step }) => {
    // ПРИМЕЧАНИЕ: Используйте аргумент `events`, который является массивом данных событий
    const attrs = events.map((evt) => {
      return {
        user_id: evt.data.user_id,
        endpoint: evt.data.endpoint,
        timestamp: toDateTime(evt.ts),
        account_type: evt.data.account_type,
      };
    });

    const result = await step.run("record-data-to-db", async () => {
      return db.bulkWrite(attrs);
    });

    return { success: true, recorded: result.length };
  }
);
```

### Справочник по конфигурации

* `maxSize` — максимальное количество событий, добавляемых в один пакет.
* `timeout` — время ожидания для наполнения пакета событиями. Если пакет не заполнится по истечении этого времени, функция будет вызвана с теми событиями, которые уже есть в текущем пакете, независимо от его размера.
* `key` — необязательное [выражение](/docs/guides/writing-expressions), использующее данные события для группировки в пакеты. Каждое уникальное значение `key` получит свой собственный пакет, что позволяет группировать события по любому конкретному ключу, например по ID пользователя.
* `if` — необязательное [логическое выражение](/docs/guides/writing-expressions), использующее данные события для условного включения в пакет (событие включается, если выражение истинно).


<Callout>
  Рекомендуется учитывать общий размер пакета, который вам придется обрабатывать, включая типичный размер полезной нагрузки события. Обработка больших пакетов может привести к проблемам с памятью или производительностью вашего приложения.

  В целях безопасности системы мы также применяем ограничение размера пакета в 10 МиБ. Это означает, что если общий размер всех событий превысит 10 МиБ, пакет начнет выполнение, даже если он не полон или не достигнут тайм-аут.
  На данный момент этот лимит изменить нельзя.
</Callout>

## Как работает пакетная обработка

Когда пакетная обработка включена, Inngest создает новый пакет при получении первого события. Пакет наполняется событиями до тех пор, пока не будет достигнут `maxSize` _или_ не истечет `timeout`. Затем функция вызывается с полным списком событий в пакете. Если установлен `key`, Inngest будет поддерживать отдельный пакет для каждого уникального ключа, что позволяет группировать события, относящиеся к одной сущности, например к клиенту.

В зависимости от вашего SDK, аргумент `events` будет содержать полный список событий внутри пакета. Это позволяет вам оперировать всеми ними в рамках одной функции.

### Условная пакетная обработка (Conditional Batching)
Условную пакетную обработку можно включить, предоставив логическое выражение в `if`. Если выражение не может быть вычислено в логическое значение или если оно вычисляется в `false`, пакетная обработка для этого события будет пропущена, и событие будет запланировано к немедленному выполнению.

## Сочетание с другими методами управления потоком

Пакетная обработка работает не со всеми функциями управления потоком.

### Пакетная обработка с лимитами конкурентности

Вы _можете_ сочетать пакетную обработку с простыми лимитами [конкурентности](/docs/guides/concurrency). Например, установка `concurrency: { limit: 1 }` приведет к обработке одного пакета за раз.

Однако **параметр `key` в настройках конкурентности игнорируется, когда включена пакетная обработка**. Это означает:

- Если вы настроите `concurrency: { limit: 1, key: "event.data.user_id" }` вместе с пакетной обработкой, выражение `key` не возымеет эффекта.
- Пакеты будут обрабатываться по одному глобально (на основе `limit`), а не для каждого уникального значения ключа.
- Лимит конкурентности применяется ко всем пакетам, независимо от того, какие значения ключей содержат события.

Если вам нужна конкурентность по ключам при пакетной обработке событий, рассмотрите возможность индивидуальной обработки событий с лимитом конкурентности по ключам вместо использования пакетной обработки.

### Несовместимые функции управления потоком

Вы _не можете_ использовать пакетную обработку вместе с [идемпотентностью](/docs/guides/handling-idempotency), [ограничением частоты (rate limiting)](/docs/guides/rate-limiting), [событиями отмены](/docs/guides/cancel-running-functions#cancel-with-events) или [приоритетами (priority)](/docs/guides/priority).
