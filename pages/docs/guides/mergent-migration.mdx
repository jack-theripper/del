# Руководство по миграции с Mergent

Поскольку Mergent меняет вектор своего развития, мы рекомендуем перенести ваши фоновые задачи, рабочие процессы (workflows) и событийно-ориентированные системы на Inngest — современную платформу для разработчиков, созданную специально для надежных и масштабируемых фоновых функций и рабочих процессов.

При миграции на Inngest выполнение кода по-прежнему происходит на ваших собственных серверах (будь то serverless или обычные серверы). Мы берем на себя оркестрацию, очереди и хранение состояния функций, что делает миграцию быстрой и простой.

Это руководство охватывает:

- [Миграция задач (Tasks) Mergent в Inngest](#migrating-tasks-to-inngest)
- [Миграция запланированных задач (Schedules) Mergent в Inngest](#migrating-schedules-to-inngest)
- [Развертывание в продакшн](#deploying-to-production)

## Почему стоит перейти на Inngest

Inngest — это больше, чем просто альтернатива Mergent… Это платформа рабочих процессов нового поколения для современных разработчиков, и она работает везде — как в serverless-средах, так и на выделенных серверах. Используя Inngest, вы получаете мощный набор инструментов «из коробки» с минимальной настройкой:

- **Устойчивые функции и возобновляемые рабочие процессы:** определяйте длительные возобновляемые процессы с помощью простого синтаксиса async/await. Никаких стейт-машин, ручного управления контекстом или очередями.
- **Встроенные повторы, воспроизведение (replay), идемпотентность и управление потоком:** автоматические повторные попытки, изоляция шагов и сохранение состояния гарантируют, что рабочие процессы не прервутся даже во время деплоя или сбоев.
- **Инструменты для локальной разработки и отладки:** создавайте и тестируйте локально с помощью Inngest Dev Server. Логируйте, отслеживайте (trace) и воспроизводите события для полной прозрачности системы.
- **Масштабируемость по умолчанию:** Inngest спроектирован для обработки сотен миллионов запусков без необходимости управления новой инфраструктурой.

## Миграция задач (Tasks) в Inngest

Миграция существующих задач Mergent в Inngest требует лишь нескольких изменений.

Сначала вы измените HTTP-обработчик задач, затем тело задачи и, наконец, вызовы API Mergent Tasks.

**1. Создайте клиент Inngest**

Сначала создадим клиент Inngest, который будет использоваться для создания и запуска функций Inngest:

```tsx
//src/inngest/client.ts
import { Inngest } from "inngest";

export const inngest = new Inngest({ id: "my-app" });
```

**2. Обновление обработчиков задач**

Как и Mergent, Inngest будет взаимодействовать с вашим приложением (где выполняются ваши задачи) через HTTP.

Для этого перенесите следующий обработчик задач Mergent:

```tsx
// pages/api/tasks.ts
import Mergent from "mergent";
import { sendEmail } from "@/mergent/tasks/sendEmail"

export default async function handler(req, res) {
  try {
    await sendEmail(req);
    // задача выполнена успешно, отвечаем 200
    res.status(200).send("");
  } catch (err) {
    // задача не удалась, отвечаем 500, чтобы Mergent повторил попытку
    res.status(500).send({ error: err });
  }
}
```

В эквивалент на Inngest:

```tsx
// src/api/inngest/route.ts
import { serve } from "inngest/next";
import { inngest } from "@/inngest/client";

// Создаем API для ваших функций Inngest
export const { GET, POST, PUT } = serve({
  client: inngest,
  functions: [
    /* сюда позже будут добавлены ваши функции Inngest! */
  ],
});

```

**3. Обновите вашу задачу**

Теперь перенесем существующую задачу Mergent в функцию Inngest:

```tsx
// src/mergent/tasks/sendEmail.ts

async function sendEmail(req) {
  await sendEmail(req);
}
```

Оберните (*или вынесите в новый файл, если код находился в обработчике задачи*) существующую задачу с помощью `inngest.createFunction()` следующим образом:

```tsx
// src/inngest/functions/sendEmail.ts
import { inngest } from "@/inngest/client";

export const helloWorld = inngest.createFunction(
  { id: "send-email", triggers: { event: "tasks/sendEmail" } },
  async ({ event, step }) => {
    await step.run("Отправить ежедневное письмо", async () => {
      await sendEmail(event.data.email);
    });
  },
);

```

Вы заметите здесь несколько новых концепций:

- Функции Inngest **идентифицируются уникальным ID.**
- Функции Inngest **запускаются событиями** (см. следующий раздел).
- Функции Inngest получают доступ к `step` API, что позволяет вам **создавать устойчивые рабочие процессы, состоящие из [атомарных шагов с автоматическими повторами](/docs/features/inngest-functions/steps-workflows).**

В нашей только что перенесенной задаче `sendEmail` любая ошибка при отправке письма приведет к повторной попытке до 4 раз ([настраивается в параметрах функции Inngest](/docs/reference/functions/create)).

Зарегистрируем эту функцию, добавив её в обработчик Inngest serve:

```tsx
// src/api/inngest/route.ts
import { serve } from "inngest/next";
import { inngest } from "@/inngest/client";
import { myTask } from "@/inngest/functions/myTask";

// Создаем API для ваших функций Inngest
export const { GET, POST, PUT } = serve({
  client: inngest,
  functions: [
    myTask
  ],
});

```

**4. Перенесите вызовы API задач**

 В то время как задачи Mergent запускаются через [эндпоинт Create Tasks API](https://docs.mergent.co/api-reference/operations/create-task), функции Inngest запускаются с помощью событий через SDK.

Раньше задача `sendEmail` в Mergent запускалась так:

```tsx
const Mergent = require("mergent");

// устанавливаем API ключ Mergent
const mergent = new Mergent("...");

// создаем задачу, которая запустится через 5 минут
// URL должен указывать на ваш обработчик задач
// при использовании JSON не забудьте установить заголовок Content-Type
mergent.tasks
  .create({
    request: {
      url: "...",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ hello: "world" }),
    },
    delay: { minutes: 5 },
  })
  .then((task) => console.log(task))
  .catch((error) => console.error(error));
```

Теперь функция `sendEmail` в Inngest запускается через `inngest.send()`:

```tsx
import { inngest } from "@/inngest/client";

await inngest.send({
  name: "tasks/sendEmail",
  data: {
    email: "testUser@example.com",
  },
});
```

Использование событий вместо прямых HTTP-вызовов дает множество преимуществ:

- **Fan-out:** одно событие может запускать сразу несколько функций.
- **Массовый повтор (Replay):** возможность массово перезапускать события в случае сбоя.
- **Ожидание событий:** возможность дождаться подходящего события прямо посреди выполнения функции ([документация](/docs/features/inngest-functions/steps-workflows/wait-for-event)).

**Запуск функций Inngest через интерфейс**

Функции Inngest также можно запускать из интерфейса: либо локально через [Inngest Dev Server](/docs/local-development), либо [после деплоя](https://www.notion.so/Mergent-co-migration-guide-1ddb64753bbd805990b2e03bcc6f82f0?pvs=21) через Inngest Cloud, отправив событие вручную.

## Миграция расписаний (Schedules) в Inngest

Расписания Mergent — это задачи, которые запускаются с определенным интервалом, настроенным в панели управления Mergent.

Чтобы перенести существующие расписания Mergent в запланированную функцию Inngest (CRON), выполните шаги 1 (*«1. Создайте клиент Inngest»*) и 2 (*«2. Обновите обработчики задач»*) из раздела миграции задач.

После того как клиент Inngest создан и HTTP-обработчик задачи перенесен, преобразуйте вашу запланированную задачу Mergent следующим образом:

```tsx
export const dailyReminder = async (req) => {
    await sendEmail(event.data.email)
};
```

В приведенном выше примере задача настроена в Mergent Schedules на выполнение в `0 0 * * *` (каждый день в полночь).

Inngest позволяет настраивать интервал CRON прямо в коде:

```tsx
export const dailyReminder = inngest.createFunction(
  { id: "daily-reminder", triggers: { cron: "0 0 * * *" } }, // Запуск ежедневно в полночь UTC
  async ({ event, step }) => {
    await step.run("Отправить ежедневное письмо", async () => {
      await sendEmail(event.data.email);
    });
  }
);
```

Обратите внимание, что, как и любую другую функцию Inngest, запланированные функции можно запускать вручную из вкладок Functions в Inngest Cloud или Inngest Dev Server.

## **Развертывание в продакшн**

Когда всё будет готово, разверните ваши функции Inngest на выбранной вами платформе (Vercel, Netlify или собственный сервер Node.js). Inngest бесшовно работает с serverless-платформами, а также может запускаться внутри любого приложения Express/Next.js. [Читайте документацию по деплою здесь.](/docs/platform/deployment)
