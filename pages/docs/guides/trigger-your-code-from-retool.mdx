# Запуск кода из Retool

Создание и поддержка внутренних инструментов — непростая задача. К счастью, [Retool](https://retool.com/) помог многим компаниям снизить эту нагрузку. Retool ориентирован на создание дашбордов и форм и хорошо интегрируется с несколькими базами данных и облачными API.

Однако зачастую существуют действия, которые вашей команде поддержки или отдела по работе с клиентами необходимо выполнять, но которые слишком сложны для встроенных возможностей Retool. Возможно, вы даже написали часть необходимого кода в своём приложении, но не можете легко запустить его из Retool.

## Проблема

Допустим, у вас есть интеграция в вашем приложении, и ваш бэкенд-код импортирует данные из стороннего сервиса. Сторонний API или ваш бэкенд мог быть недоступен несколько часов, и у определённых пользователей могут отсутствовать данные. Когда кто-то обращается в поддержку, вам может потребоваться повторно запустить скрипт импорта для восстановления пропущенных данных.

Мы расскажем, как это сделать, чтобы ваша команда могла запускать важные скрипты прямо из приложения Retool. Это руководство предполагает базовый опыт создания форм в Retool (*Если у вас его нет, ознакомьтесь с [этим отличным руководством](https://docs.retool.com/docs/create-forms-using-form-component)*).

## План

Цель — дать вашей команде возможность запускать скрипт каждый раз, когда они нажимают кнопку в Retool. Для этого мы:

1. Создадим кнопку Retool, которая отправляет событие в Inngest
2. Напишем функцию Inngest, использующую наш существующий скрипт
3. Настроим функцию для запуска при получении нашего события
4. Посмотрим, как всё работает от начала до конца

## Отправка события из Retool

Для отправки данных из Retool нам сначала нужно настроить «[Ресурс (Resource)](https://docs.retool.com/docs/resources)». На вкладке Resources в Retool нажмите «Create New», затем выберите «Resource» и «Rest API». Перейдите в дашборд Inngest Cloud и [создайте новый Event Key в дашборде Inngest](/docs/events/creating-an-event-key). Скопируйте новый ключ и в дашборде Retool добавьте перед ним URL и путь Inngest Event API: `https://inn.gs/e/`

```shell
https://inn.gs/e/<INNGEST-EVENT-KEY>
```

Ваш новый ресурс будет выглядеть так. После этого нажмите «Create resource.»

![Скриншот ресурса Inngest в Retool](/assets/guides/trigger-your-code-from-retool/retool-resource.png)

Теперь перейдём в приложение Retool, в которое хотим добавить форму с кнопкой. Допустим, вы уже создали следующую форму `runBackfillForm` с одним полем `userId` и кнопкой отправки:

![Скриншот формы Retool](/assets/guides/trigger-your-code-from-retool/retool-form.png)

Далее создайте новый «Resource query» на панели «Code» внизу слева (используйте кнопку +). Назовём новый запрос `sendBackfillRequested` и выберем наш новый ресурс «Inngest» из выпадающего списка. Обновите «Action type» до запроса `POST`. В разделе «Body» добавьте данные, которые хотите отправить в Inngest. События Inngest требуют имени и данных в формате JSON. Полезно использовать префикс в именах событий для группировки; здесь мы назовём наше событие `"retool/backfill.requested"` и передадим id пользователя из формы и email текущего пользователя Retool в вашей команде для будущего аудита:

```json
{ "user_id": "{{runBackfillForm.data.userId}}", "agent_id": "{{current_user.email}}" }
```

В конечном счёте ваш resource query будет выглядеть так. Сохраним его и нажмём «Run» для тестирования.

![Скриншот resource query в Retool](/assets/guides/trigger-your-code-from-retool/retool-resource-query.png)

На вкладке «Events» в дашборде Inngest Cloud вы должны увидеть новое событие `retool/backfill.requested`. Нажмите на событие, и вы сможете выбрать payload, который мы только что отправили.

После того как мы убедились, что данные отправляются в Inngest, можно привязать resource query как обработчик события к кнопке отправки. Выберите тип взаимодействия «Default» и нажмите «+ Add», чтобы выбрать наш resource query `sendBackfillRequested`. При желании можно добавить `isFetching` для отображения индикатора загрузки.

## Создание функции Inngest

Используя [Inngest SDK](/features/sdk?ref=retool-guide), вы можете определить функцию Inngest и её триггер события в одном файле. Создадим директорию `inngest` в корне нашего проекта:

```
mkdir -p inngest
```

Теперь создадим файл в этой директории для нашей функции — `runBackfillForUser.js`. Это будет наша функция Inngest, которая импортирует существующий код восстановления, использует `user_id` из payload события для запуска этого кода и возвращает HTTP-статус код в ответе, чтобы сообщить Inngest [нужно ли повторить попытку](/docs/functions/retries?ref=retool-guide).


<CodeGroup>

```ts {{ title: "runBackfillForUser.ts" }}
import { runBackfillForUser } from "../lib/backfill-scripts";
import { inngest } from "./client";

export default inngest.createFunction(
  {
    id: "run-backfill-for-user", // Имя, отображаемое в дашборде Inngest
    triggers: { event: "retool/backfill.requested" }, // Триггер события
  },
  async ({ event }) => {
    const result = await runBackfillForUser(event.data.user_id);

    return {
      status: result.ok ? 200 : 500,
      message: `Ran backfill for user ${event.data.user_id}`,
    };
  }
);
```

```ts  {{ title: "client.ts" }}
import { Inngest } from "inngest";

export const inngest = new Inngest({ id: "my-app" })
```

</CodeGroup>

Вот наша функция — теперь нам просто нужно подключить её.

### Подключение функции

Вам нужно подключить функцию, чтобы Inngest мог удалённо и безопасно вызывать её через HTTP.

Для этого руководства объясним, как сделать это с существующим приложением [Express.js](https://expressjs.com/). Стандартный обработчик [`serve()`](/docs/reference/serve) от Inngest можно импортировать и передать в `app.use` или `router.use` Express.js. Ключ подписи Inngest можно получить в [дашборде Inngest](https://app.inngest.com/env/production/manage/signing-key).

```js
import { serve } from "inngest/express"
import runBackfillForUser from "../inngest/runBackfillForUser"

app.use("/api/inngest", serve("My API", process.env.INNGEST_SIGNING_KEY, [
  runBackfillForUser,
]))
// ваши существующие маршруты...
app.get("/api/whatever", ...)
app.post("/api/something_else", ...)
```

## Развёртывание функции

Подключая функции через HTTP, вам не нужно разворачивать код в Inngest Cloud или настраивать новый процесс деплоя. После развёртывания кода вам нужно посетить дашборд Inngest для синхронизации приложения. Это позволяет Inngest обнаруживать и удалённо выполнять ваши функции.

## Ваша очередь

Теперь вы знаете, как взять существующий код из приложения, подключить его к Inngest и запускать прямо из Retool с полным журналом аудита — кто запустил, для какого пользователя и с полными логами. Не нужно было настраивать сложную инфраструктуру с очередью или новым эндпоинтом на вашем production API — просто разверните код в Inngest и отправьте событие из Retool.
