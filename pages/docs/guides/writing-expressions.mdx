# Написание выражений

Выражения используются несколькими способами для настройки ваших функций. Они применяются для:

* Определения ключей на основе свойств событий для [параллелизма (concurrency)](/docs/functions/concurrency), [ограничения частоты (rate limiting)](/docs/reference/functions/rate-limit), [debounce](/docs/reference/functions/debounce) или [идемпотентности (idempotency)](/docs/guides/handling-idempotency)
* Условного сопоставления событий для [ожидания события (wait for event)](/docs/reference/functions/step-wait-for-event), [отмены (cancellation)](/docs/guides/cancel-running-functions) или [параметра `if` триггера функции](/docs/reference/functions/create#trigger)
* Возврата значений для [приоритета запуска](/docs/reference/functions/run-priority) функции

Все выражения определяются с использованием [Common Expression Language (CEL)](https://github.com/google/cel-go). CEL предлагает простые, быстрые, нетьюринг-полные выражения. Это позволяет Inngest оценивать миллионы выражений для всех пользователей в масштабе.

## Типы выражений

В контексте Inngest выражения должны вычисляться либо в булево значение, либо в значение:

* **Булевы значения** — любое выражение, используемое для условного сопоставления, должно возвращать булево значение. Они используются в ожидании события, отмене и параметре `if` триггера функции.
* **Значения** — другие выражения могут возвращать любое значение, которое может использоваться как ключи (например, для параллелизма, ограничения частоты, debounce или [ключей идемпотентности](/docs/guides/handling-idempotency)) или динамическое значение (например, приоритет запуска).

## Переменные

- `event` — ссылается на событие, запустившее выполнение функции, во всех случаях.
- `async` — ссылается на новое событие в `step.waitForEvent` и [отмене](/docs/guides/cancel-running-functions). Это входящее событие, которое сопоставляется асинхронно. Присутствует только при сопоставлении новых событий в запуске функции.

## Примеры

В большинстве выражений в качестве входных данных используется объект payload `event`. Выражения, сопоставляющие дополнительные события (например, ожидание события, отмена), также будут иметь объект `async` для payload сопоставляемого события. Подробнее см. в [справочнике всех операторов, доступных в CEL](https://github.com/google/cel-spec/blob/master/doc/langdef.md#list-of-standard-definitions).

### Булевы выражения

```js
// Сопоставить поле со строкой
"event.data.billingPlan == 'enterprise'"

// Числовое сравнение
"event.data.amount > 1000"

// Объединение нескольких условий
"event.data.billingPlan == 'enterprise' && event.data.amount > 1000"
"event.data.billingPlan != 'pro' || event.data.amount < 300"

// Сравнить триггер функции с входящим событием (для ожидания события или отмены)
"event.data.userId == async.data.userId"

// Альтернативно, можно использовать интерполяцию строк JavaScript для ожидания события
`${userId} == async.data.userId` // => "user_1234 == async.data.userId"
```

## Советы

* Используйте `+` для конкатенации строк
* Используйте `==` для проверки равенства
* Можно использовать одинарные `'` или двойные кавычки `"` для строк, но рекомендуем придерживаться одного варианта для единообразия кода
* При работе с TypeScript SDK пишите выражения в обратных кавычках `` ` ``, чтобы использовать кавычки внутри выражения, или используйте интерполяцию строк JavaScript
* Используйте тернарные операторы для возврата значений по умолчанию
* При использовании оператора or (`||`) CEL всегда возвращает булево значение. Это отличается от JavaScript, где оператор or возвращает значение выражения слева от оператора, если оно истинно. Используйте тернарный оператор (`?`) вместо `||` для условных возвратов.

<Info>
  Обратите внимание, что хотя CEL поддерживает широкий спектр вспомогательных функций и макросов, Inngest поддерживает только их подмножество для обеспечения высокого уровня производительности и надёжности.
</Info>

## Тестирование выражений

Вы можете тестировать выражения на [CEL Playground от Undistro](https://playcel.undistro.io/). Это отличный способ быстро проверить более сложные выражения, особенно с условными возвратами.
