# Работа с циклами в Inngest

В Inngest каждый шаг вашей функции выполняется как отдельный HTTP-запрос. Это означает, что для каждого шага функция вызывается повторно, начиная с самого начала, вплоть до точки выполнения следующего шага. Эта [модель выполнения](/docs/learn/how-functions-are-executed) помогает управлять повторными попытками, тайм-аутами и обеспечивает надёжность в распределённых системах.

На этой странице описано, как реализовать циклы в функциях Inngest и избежать типичных ошибок.

```javascript
inngest.createFunction(
  { id: "simple-function", triggers: { event: "test/simple.function" } },
  async ({ step }) => {
    console.log("hello");

    await step.run("a", async () => { console.log("a") });
    await step.run("b", async () => { console.log("b") });
    await step.run("c", async () => { console.log("c") });
  }
);
```

Любая недетерминированная логика (например, обращения к базе данных или API) должна быть помещена внутрь вызова `step.run`, чтобы гарантировать корректное выполнение в рамках каждого шага.

С учётом этого, вот как можно исправить предыдущий пример:

```ts
inngest.createFunction(
  { id: "simple-function", triggers: { event: "test/simple.function" } },
  async ({ step }) => {
    await step.run("hello", () => { console.log("hello") });

    await step.run("a", async () => { console.log("a") });
    await step.run("b", async () => { console.log("b") });
    await step.run("c", async () => { console.log("c") });
  }
);

// hello
// a
// b
// c
```

Теперь «hello» печатается только один раз, как и ожидалось.

Вот [пример](/blog/import-ecommerce-api-data-in-seconds) функции Inngest, которая импортирует все товары из магазина Shopify в локальную систему. Эта функция перебирает все страницы, объединяя все товары в единый массив.

```typescript
export default inngest.createFunction(
  { id: "shopify-product-import", triggers: { event: "shopify/import.requested" } },
  async ({ event, step }) => {
    const allProducts = []
    let cursor = null
    let hasMore = true

    // Используем «data» события для передачи ключевой информации, например ID
    // Примечание: в данном примере это детерминировано между несколькими запросами
    // Если возвращаемые результаты должны сохранять одинаковый порядок, оберните вызов db в step.run()
    const session = await database.getShopifySession(event.data.storeId)

    while (hasMore) {
      const page = await step.run(`fetch-products-${pageNumber}`, async () => {
        return await shopify.rest.Product.all({
          session,
          since_id: cursor,
        })
      })
      // Объединяем все данные в единый список
      allProducts.push(...page.products)
      if (page.products.length === 50) {
        cursor = page.products[49].id
      } else {
        hasMore = false
      }
    }

    // Теперь у нас есть полный список товаров в allProducts!
  }
)
```

В примере выше каждая итерация цикла управляется с помощью `step.run()`, гарантируя, что **вся недетерминированная логика (например, получение товаров из Shopify) инкапсулирована внутри шага**. Такой подход гарантирует, что при сбое запроса он будет автоматически повторён в правильном порядке. Эта структура соответствует модели выполнения Inngest, где каждый шаг — отдельный HTTP-запрос, обеспечивая надёжное и согласованное поведение цикла.

Обратите внимание, что в примере выше `getShopifySession` детерминирована между несколькими запросами (и добавляется ко всем API-вызовам для авторизации). Если возвращаемые результаты должны сохранять одинаковый порядок, оберните вызов базы данных в `step.run()`.

## Лучшие практики: реализация циклов в Inngest

Чтобы циклы корректно работали в рамках [модели выполнения Inngest](/docs/learn/how-functions-are-executed):

### 1. Рассматривайте каждую итерацию цикла как отдельный шаг

В обычной среде программирования циклы сохраняют своё состояние между итерациями. В Inngest каждый шаг повторно выполняет функцию с самого начала, чтобы гарантировать повтор только неудавшихся шагов. Для корректной работы рассматривайте каждую итерацию цикла как отдельный шаг. Таким образом цикл будет продвигаться правильно, и каждая итерация будет строиться на предыдущей.

### 2. Помещайте недетерминированную логику внутрь шагов

Помещайте недетерминированную логику (например, API-вызовы, запросы к базе данных или генерацию случайных чисел) внутрь вызовов `step.run`. Это гарантирует корректное и согласованное выполнение таких операций в рамках каждого шага, предотвращая повторное выполнение при каждом повторном входе в функцию.

### 3. Эффективно используйте sleep

При использовании `step.sleep` внутри цикла убедитесь, что он сочетается со структурированием цикла таким образом, чтобы каждая итерация обрабатывалась как отдельный шаг. Это предотвращает кажущийся перезапуск функции и обеспечивает контролируемые интервалы между итерациями.
