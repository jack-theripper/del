# Инструментирование GraphQL

При разработке с использованием GraphQL вы можете дать мощный толчок вашему событийно-ориентированному приложению, инструментируя каждый запрос (query) и мутацию (mutation), отправляя события при их успешном выполнении.

Для этого можно использовать плагин [Envelop](https://envelop.dev/), `useInngest`, предназначенный для [GraphQL Yoga](https://the-guild.dev/graphql/yoga-server) и серверов или фреймворков на его основе, таких как [RedwoodJS](https://www.redwoodjs.com/).

Использование плагина `useInngest` дает следующие преимущества:

- Получение готового набора событий для реагирования, который автоматически растет вместе с вашим GraphQL API.
- Не требуется никаких изменений в существующих резолверах.
- Тонкая настройка отправляемых событий: операции (запросы, мутации или подписки), события интроспекции, возникновение ошибок GraphQL, включение результирующих данных, черные списки типов и схем и многое другое.
- Автоматический захват контекста, например, данных пользователя.

## Начало работы

```sh
npm install envelop-plugin-inngest # или yarn add
```

### Пример использования

Для использования `useInngest` требуется наличие клиента Inngest (см. [Быстрый старт](/docs/getting-started/nextjs-quick-start)), настроенного с соответствующим ключом событий (см. [Создание ключа событий](https://www.inngest.com/docs/events/creating-an-event-key)).

Вот пример добавления плагина в одном файле:

<CodeGroup>
```ts
import { useInngest } from "envelop-plugin-inngest";
import { createSchema, createYoga } from "graphql-yoga";
import { Inngest } from "inngest";

const inngest = new Inngest({ id: "my-app" });

// Опишите вашу схему
const yoga = createYoga({
  schema: createSchema({
    typeDefs: /* GraphQL */ `
      type Query {
        greetings: String!
      }
    `,
    resolvers: {
      Query: {
        greetings: () => "Hello World!",
      },
    },
  }),

  // Добавьте плагин на сервер. Пользователи RedwoodJS могут использовать
  // опцию `extraPlugins`.
  plugins: [useInngest({ inngestClient: inngest })],
});

// Запустите сервер и перейдите на http://localhost:4000/graphql
const server = createServer(yoga);

server.listen(4000, () => {
  console.info("Server is running on http://localhost:4000/graphql");
});
```
</CodeGroup>


### Выходные события

После установки плагина события будут отправляться для всех успешных операций GraphQL, формируя готовый к использованию набор событий, на которые можно реагировать немедленно.

Вот пример события, отправленного мутацией создания нового товара в корзине пользователя:

```json
{
  "name": "graphql/create-cart-item.mutation",
  "data": {
    "identifiers": [
      {
        "id": 27,
        "typename": "CartItem"
      }
    ],
    "operation": {
      "id": "create-cart-item",
      "name": "CreateCartItem",
      "type": "mutation"
    },
    "result": {
      "data": {
        "createCartItem": {
          "id": 27,
          "productId": "123"
        }
      }
    },
    "types": [
      "CartItem"
    ],
    "variables": {}
  },
  "id": "01GXXAQ1M0A1SFVGEHACRF4K1C"
}
```

### Реагирование на события

Мы можем реагировать на это событие, создав новую функцию Inngest с этим событием в качестве триггера.

```ts
inngest.createFunction(
  { id: "send-cart-alert", triggers: { event: "graphql/create-cart-item.mutation" } },
  async ({ event }) => {
    await sendSlackMessage(
      "#marketing",
      `Кто-то добавил товар #${event.data.identifiers[0].id} в корзину!`
    );
  }
);
```

Для получения дополнительной информации о настройке отправляемых событий посетите репозиторий [envelop-plugin-inngest](https://github.com/inngest/envelop-plugin-inngest) или раздел [Написание функций](https://www.inngest.com/docs/functions), чтобы узнать, как реагировать на эти события различными способами.
