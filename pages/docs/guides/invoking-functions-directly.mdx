# Прямой вызов функций

Функция Inngest `step.invoke()` предоставляет мощный инструмент для прямого вызова функций внутри вашей событийно-ориентированной системы. Она отличается от традиционных триггеров событий, предлагая более прямой подход, подобный RPC (удаленному вызову процедур). Это дает несколько ключевых преимуществ:

- Позволяет функциям вызывать другие функции и получать их результат.
- Естественным образом разделяет вашу систему на переиспользуемые функции, которые могут быть распределены между различными процессами.
- Позволяет использовать синхронное взаимодействие между функциями в архитектуре, которая в остальном является асинхронной и событийно-ориентированной, что значительно упрощает управление функциями, требующими немедленных результатов.
- Включает [синхронное делегирование субагентам](/docs/ai-patterns/sub-agent-delegation#delegate-synchronously-with-stepinvoke) в системах ИИ-агентов.

### Когда следует использовать вызов?

Используйте `step.invoke()` для задач, требующих специфических настроек, таких как лимиты конкурентности. Поскольку такая функция запускается со своей собственной конфигурацией, отличной от вызывающей стороны, вы можете предоставить индивидуальные настройки для каждой функции.

Если вам не нужно определять детализированную конфигурацию или если ваша функция не будет переиспользоваться за пределами приложения, используйте `step.run()` для простоты.
</Callout>

```ts
// Какая-то функция, которую мы будем вызывать
const computeSquare = inngest.createFunction(
  { id: "compute-square", triggers: { event: "calculate/square" } },
  async ({ event }) => {
    return { result: event.data.number * event.data.number }; // Результат типизирован как { result: number }
  }
);

// В этой функции мы вызовем `computeSquare`
const mainFunction = inngest.createFunction(
  { id: "main-function", triggers: { event: "main/event" } },
  async ({ step }) => {
    const square = await step.invoke("compute-square-value", {
      function: computeSquare,
      data: { number: 4 }, // входные данные типизированы
    });

    return `Квадрат 4 равен ${square.result}.`; // square.result типизирован как число
  }
);
```

В приведенном выше примере наша `mainFunction` вызывает `computeSquare` для получения вычисленного значения. Теперь `computeSquare` можно вызывать отсюда или из любого другого процесса, подключенного к Inngest.

## Ссылка на другую функцию Inngest

Если функция существует в другом приложении, вы можете создать ссылку (reference), которую можно будет вызвать таким же образом, как и локальную функцию `computeSquare` выше.

```ts
// @/inngest/computeSquare.ts
import { referenceFunction } from "inngest";
import { z } from "zod";

// Создаем ссылку на функцию в другом приложении.
export const computeSquare = referenceFunction({
  appId: "my-python-app",
  functionId: "compute-square",
  // Схемы необязательны, но обеспечивают типизацию вашего вызова
  schemas: {
    data: z.object({
      number: z.number(),
    }),
    return: z.object({
      result: z.number(),
    }),
  },
});
```

```ts
import { computeSquare } from "@/inngest/computeSquare";

// square.result типизирован как число
const square = await step.invoke("compute-square-value", {
  function: computeSquare,
  data: { number: 4 },
});
```

Ссылки также можно использовать для вызова локальных функций без необходимости импортировать их (и их зависимости) напрямую. Это может быть полезно для таких фреймворков, как Next.js, где обработчики edge и serverless могут быть смешаны и требовать разных наборов зависимостей.

```ts
import { inngest, referenceFunction } from "inngest";
import { type computeSquare } from "@/inngest/computeSquare"; // Импортируем только тип

const mainFunction = inngest.createFunction(
  { id: "main-function", triggers: { event: "main/event" } },
  async ({ step }) => {
    const square = await step.invoke("compute-square-value", {
      function: referenceFunction<typeof computeSquare>({
        functionId: "compute-square",
      }),
      data: { number: 4 }, // входные данные все еще типизированы
    });

    return `Квадрат 4 равен ${square.result}.`;
  }
);
```

Для получения дополнительной информации о ссылках на функции см. [TypeScript -> Referencing Functions](/docs/functions/references).

### Когда следует использовать вызов?

Используйте `step.Invoke()` для задач, требующих специфических настроек, таких как лимиты конкурентности. Поскольку такая функция запускается со своей собственной конфигурацией, отличной от вызывающей стороны, вы можете предоставить индивидуальные настройки для каждой функции.

Если вам не нужно определять детализированную конфигурацию или если ваша функция не будет переиспользоваться за пределами приложения, используйте `step.Run()` для простоты.
</Callout>

```go
import (
	"context"
	"fmt"

	"github.com/inngest/inngestgo"
	"github.com/inngest/inngestgo/step"
)

// Какая-то функция, которую мы будем вызывать
inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{Name: "compute-square"},
	inngestgo.EventTrigger("calculate/square", nil),
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		number, ok := input.Event.Data["number"].(float64)
		if !ok {
			return nil, fmt.Errorf("invalid number")
		}

		return map[string]any{
			"result": int(number * number),
		}, nil
	},
)

// В этой функции мы вызовем функцию compute-square
inngestgo.CreateFunction(
	client,
	inngestgo.FunctionOpts{Name: "main-function"},
	inngestgo.EventTrigger("main/event", nil),
	func(ctx context.Context, input inngestgo.Input[map[string]any]) (any, error) {
		square, err := step.Invoke(ctx, "compute-square-value", &inngestgo.InvokeOpts{
			Function: "compute-square",
			Data: map[string]any{
				"number": 4,
			},
		})
		if err != nil {
			return nil, err
		}

		result := square.Data["result"].(int)
		return fmt.Sprintf("Квадрат 4 равен %d.", result), nil
	},
)

```

В приведенном выше примере наша `mainFunction` вызывает `computeSquare` для получения вычисленного значения. Теперь `computeSquare` можно вызывать отсюда или из любого другого процесса, подключенного к Inngest.

```py
import inngest
from src.inngest.client import inngest_client

# Какая-то функция, которую мы будем вызывать
@inngest_client.create_function(
    fn_id="compute-square",
    trigger=inngest.TriggerEvent(event="calculate/square")
)
async def compute_square(ctx: inngest.Context):
    return {"result": ctx.event.data["number"] * ctx.event.data["number"]}  # Результат типизирован как { result: number }

# В этой функции мы вызовем compute_square
@inngest_client.create_function(
    fn_id="main-function",
    trigger=inngest.TriggerEvent(event="main/event")
)
async def main_function(ctx: inngest.Context):
    square = await ctx.step.invoke(
        "compute-square-value",
        function=compute_square,
        data={"number": 4}  # входные данные типизированы
    )

    return f"Квадрат 4 равен {square['result']}."
```

## Создание распределенной системы

Вы можете вызывать функции Inngest, написанные на любом языке и размещенные в разных облаках. Например, функция на TypeScript на Vercel может вызвать функцию на Python, размещенную в AWS.

Начиная определять такие блоки функциональности, вы создаете умную распределенную систему со всеми преимуществами событийно-ориентированной архитектуры и без лишних сложностей.

## Похожий паттерн: Fan-Out

Похожим на прямой вызов функций является паттерн fan-out (разветвление) — [ознакомьтесь с руководством здесь](/docs/guides/fan-out-jobs). Вот некоторые ключевые отличия:

- Fan-out запускает несколько функций одновременно, в то время как вызов запускает только одну.
- В отличие от вызова, fan-out не получает результат выполнения вызванной функции.
- Выбирайте fan-out для параллельной обработки независимых задач и вызов (invocation) для скоординированных, взаимозависимых функций.
