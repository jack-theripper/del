# Фоновые задачи
Это руководство поможет вам за несколько минут создать фоновые задачи с повторными попытками.

Запуская фоновые задачи в Inngest:

- Вам не нужно создавать очереди, воркеры или подписки.
- Вы можете запускать фоновые задачи в серверлесс-функциях без настройки инфраструктуры.
- Вы можете ставить задачи в очередь для выполнения в будущем, аналогично очереди задач, без какой-либо конфигурации.

## Как создавать фоновые задачи

<LanguageSelector>

Фоновые задачи в Inngest выполняются в ответ на триггер (событие или cron).

Пример ниже показывает фоновую задачу, которая использует событие (здесь оно называется `app/user.created`) для отправки электронного письма новым зарегистрированным пользователям. Она состоит из двух частей: создания функции, которая выполняется в фоновом режиме, и запуска этой функции.

<aside>
👉 Чтобы иметь возможность использовать код ниже, не забудьте сначала [развернуть ваши функции в Inngest API](/docs/learn/serving-inngest-functions), чтобы они были доступны для Inngest.
</aside>

### 1. Создайте функцию, которая выполняется в фоновом режиме

<LanguageSection show="typescript">

Давайте разберем код пошагово:
1. Мы [создаем новую функцию Inngest](/docs/reference/functions/create), которая будет запускаться в фоновом режиме каждый раз, когда событие `app/user.created` отправляется в Inngest.
2. Мы надежно отправляем электронное письмо, используя метод [`step.run()`](/docs/reference/functions/step-run). Каждый [шаг Inngest](/docs/steps) автоматически повторяется в случае сбоя.
3. Мы приостанавливаем выполнение функции до определенной даты, используя [`step.sleepUntil()`](/docs/reference/functions/step-sleep-until). Выполнение функции будет возобновлено автоматически, несмотря на перезапуски сервера или особенности серверлесс-функций. Вам не нужно беспокоиться о масштабировании, утечках памяти, соединениях или перезапусках.
4. Мы возобновляем выполнение и выполняем другие задачи.

<CodeGroup>
```ts
import { Inngest } from "inngest";
const inngest = new Inngest({ id: "signup-flow" });

export const sendSignUpEmail = inngest.createFunction(
  { id: "send-signup-email", triggers: { event: "app/user.created" } },
  ({ event, step }) => {
    await step.run("send-the-user-a-signup-email", async () => {
      await sesclient.clientsendEmail({
        to: event.data.user_email,
        subject: "Welcome to Inngest!"
        message: "...",
      });
    });
    await step.sleepUntil("wait-for-the-future", "2023-02-01T16:30:00");

    await step.run("do-some-work-in-the-future", async () => {
      // Этот код выполнится в будущем автоматически.
    });
  }
);
```

Давайте разберем код пошагово:
1. Мы [создаем новую функцию Inngest](/docs/reference/python/functions/create), которая будет запускаться в фоновом режиме каждый раз, когда событие `app/user.created` отправляется в Inngest.
2. Мы надежно отправляем электронное письмо, используя метод [`step.run()`](/docs/reference/python/steps/run). Каждый [шаг Inngest](/docs/steps) автоматически повторяется в случае сбоя.
3. Мы приостанавливаем выполнение функции до определенной даты, используя [`step.sleep_until()`](/docs/reference/python/steps/sleep-until). Выполнение функции будет возобновлено автоматически, несмотря на перезапуски сервера или особенности серверлесс-функций. Вам не нужно беспокоиться о масштабировании, утечках памяти, соединениях или перезапусках.
4. Мы возобновляем выполнение и выполняем другие задачи.

```python
import inngest

inngest_client = inngest.Inngest(
    app_id="my-app",
)

@inngest_client.create_function(
    fn_id="send-signup-email",
    trigger=inngest.TriggerEvent(event="app/user.created")
)
async def send_signup_email(ctx: inngest.Context):
    async def send_email():
        await sesclient.send_email(
            to=ctx.event.data["user_email"],
            subject="Welcome to Inngest!",
            message="..."
        )

    await ctx.step.run("send-the-user-a-signup-email", send_email)

    await ctx.step.sleep_until("wait-for-the-future", "2023-02-01T16:30:00")

    async def future_work():
        # Этот код выполнится в будущем автоматически
        pass

    await ctx.step.run("do-some-work-in-the-future", future_work)
```


### 2. Запустите функцию

Ваша функция `sendSignUpEmail` будет запускаться всякий раз, когда Inngest получает событие с именем `app/user.created`. Вы отправляете это событие в Inngest следующим образом:

```python
from src.inngest.client import inngest_client

await inngest_client.send(
    name="app/user.created",  # Это совпадает с событием, используемым в `create_function`
    data={
        "email": "test@example.com",
        # любые данные, которые вы хотите отправить
    }
)
```

</LanguageSection>

Когда вы отправляете событие в Inngest, он автоматически находит любые функции, которые запускаются по ID события, и автоматически запускает эти функции в фоновом режиме. Весь JSON-объект, который вы передаете в `inngest.send()`, будет доступен вашим функциям.

<Callout>
💡 Подсказка: Вы можете создать множество функций, которые слушают одно и то же событие, и все они будут работать в фоновом режиме. Узнайте больше об этом паттерне в нашем [руководстве по "Fan out"](/docs/guides/fan-out-jobs).
</Callout>
