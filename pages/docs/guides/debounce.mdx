# Удаление дребезга (Debounce)

Debounce (удаление дребезга) задерживает выполнение функции до тех пор, пока не перестанет поступать серия событий. Это полезно для предотвращения избыточной работы, когда функция может инициироваться несколько раз подряд в течение короткого периода. Сценарии использования debounce включают:

* Предотвращение лишней работы при обработке событий пользовательского ввода, которые могут меняться несколько раз за короткий промежуток времени.
* Задержка обработки зашумленных событий вебхуков до тех пор, пока они не перестанут поступать.
* Обеспечение того, чтобы функции использовали самое последнее событие из серии обновлений (например, при синхронизации).

### Справочник по конфигурации

* `period` — временная задержка для откладывания выполнения. Период начинается при получении первого подходящего события.
* `key` — необязательное [выражение](/docs/guides/writing-expressions), использующее данные события для применения лимита. Каждое уникальное значение `key` имеет свой собственный лимит, что позволяет ограничивать частоту запусков функций по любому конкретному ключу, например по ID пользователя.
* `timeout` — необязательно. Максимальное время, на которое может быть продлен debounce перед запуском.

## Как это работает

При срабатывании функции начинается период `period` (удаление дребезга). Если получено другое событие, соответствующее триггеру функции, период `period` сбрасывается. Это продолжается до тех пор, пока в течение периода `period` не перестанут поступать новые события. Как только период пройдет без новых событий, функция выполняется с использованием последнего полученного события.

Если указан `timeout`, функция обязательно запустится по истечении этого времени, даже если продолжают поступать новые события. Это гарантирует, что выполнение функции не будет откладываться бесконечно, если события продолжают поступать.

### Использование ключа (`key`)

При добавлении ключа `key` для каждого уникального значения выражения `key` применяется отдельный период debounce. Например, если ваш ключ `key` установлен в `event.data.customer_id`, для каждого клиента будет применяться свой индивидуальный период debounce при запуске функций. Для получения дополнительной информации прочтите [наше руководство по написанию выражений](/docs/guides/writing-expressions).

## Сравнение с ограничением частоты (Rate Limiting)

Если вы предпочитаете выполнять функцию для *первого* полученного события, рассмотрите возможность использования [ограничения частоты (rate limiting)](/docs/guides/rate-limiting). Ограничение частоты гарантирует, что функция запустится один раз для каждого `key` в тот момент, когда событие получено *впервые*, в то время как debounce использует *последнее* событие в течение указанного периода.

## Сочетание с идемпотентностью

Debounce можно сочетать с [идемпотентностью](/docs/guides/handling-idempotency#at-the-function-level-the-consumer), чтобы гарантировать, что после того, как функция с удалением дребезга была запущена, она не запустится снова.

## Ограничения

* Максимальный период debounce `period` составляет 7 дней (168 часов).
* Минимальный период debounce `period` составляет 1 секунду.
* Debounce не работает с [функциями с пакетной обработкой (batched functions)](/docs/guides/batching).
