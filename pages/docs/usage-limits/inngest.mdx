# Лимиты использования

Мы установили ряд ограничений на сервис, чтобы обеспечить хорошие настройки по умолчанию для начала работы, а также приятный опыт для всех пользователей Inngest.

Некоторые из этих ограничений можно изменить — если вам нужно больше, чем предусмотрено текущими лимитами, пожалуйста, [свяжитесь с нами][contact], и мы обновим их для вас.

## Функции

Следующее применяется к использованию `step`.

### Длительность сна

Режим сна (с `step.sleep()` и `step.sleepUntil()`) поддерживается до одного года, а для бесплатного плана — до семи дней. Подробнее на [странице тарифов](/pricing).

### Таймаут

Каждый шаг имеет таймаут, зависящий от выбранного вами хостинг-провайдера ([подробнее][provider-docs]), однако Inngest поддерживает максимум `2 часа`.

### Конкурентность <Tag>Upgradable</Tag>

Проверьте свои лимиты конкурентности на [странице биллинга](https://app.inngest.com/billing). Подробнее о лимитах конкурентности для всех планов на [странице тарифов](https://www.inngest.com/pricing).

### Размер полезной нагрузки

Лимит данных, возвращаемых шагом, составляет `4 МБ`.

### Размер состояния запуска функции

Состояние запуска функции не может превышать `32 МБ`. Состояние включает:
- Данные событий (несколько событий при использовании батчинга)
- Данные, возвращённые шагами
- Данные, возвращённые функцией
- Внутренние метаданные (_небольшой размер — несколько байт_)

### Количество шагов на функцию

Максимальное количество шагов для одной функции — `1000`.

<Callout>
  ⚠️
  Этот лимит легко достигается, если вы используете `step` для каждого элемента в цикле.
  Вместо этого мы рекомендуем одно или оба из следующих решений:
  - Обрабатывайте цикл внутри одного `step` и возвращайте его данные
  - Используйте функцию [fan out][fanout-guide] для обработки каждого элемента в отдельной функции
</Callout>

## События

### Длина имени

Максимальная длина имени события — `256` символов.

### Размер тела запроса <Tag>Upgradable</Tag>

Максимальный размер полезной нагрузки события зависит от вашего тарифного плана. По умолчанию для бесплатного тарифа — `256 КБ`, с возможностью увеличения до `3 МБ`. Подробнее на [странице тарифов](/pricing?ref=docs-usage-limits).

### Количество событий в одном запросе <Tag>Customizable</Tag>

Максимальное количество событий в одном запросе — `5000`.
При использовании fan out необходимо учитывать это ограничение при вызове `step.sendEvent(events)`.

<CodeGroup>
```ts {{ title: "TypeScript" }}
// этот список `events` должен содержать <= 5000 элементов
const events = [{name: "<event-name>", data: {}}, ...];

await step.sendEvent("send-example-events", events);
// или
await inngest.send(events);
```

```go {{ title: "Go" }}
// этот список `events` должен содержать <= 5000 элементов
events := []inngestgo.Event{{Name: "<event-name>", Data: {}}}

ids, err := client.SendMany(ctx, events)
```

```python {{ title: "Python" }}
# этот список `events` должен содержать <= 5000 элементов
events = [{'name': '<event-name>', 'data': {}}, ...]

await step.send_event('send-example-events', events)
# или
await inngest.send(events)
```
</CodeGroup>

### Размер батча

Жёсткий лимит размера батча составляет 10 МиБ независимо от ограничений `timeout` или `maxSize`.
Это означает, что батч будет запущен при превышении этого лимита, даже если батч не заполнен или не достигнут настроенный таймаут.
