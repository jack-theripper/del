# Получение статуса и результата запуска

Inngest предоставляет способ получить статус и результат запуска функции с помощью [REST API](https://api-docs.inngest.com/docs/inngest-api/1j9i5603g5768-introduction). Это полезно когда:

* Вы хотите проверить статус или результат конкретного запуска.
* Вы хотите отобразить статус запуска функции в своём приложении, например, в пользовательском дашборде.

На этой странице приведён быстрый пример того, как получить статус и результат запуска функции с помощью Inngest API.

## Быстрый пример

Вот базовая функция, которая обрабатывает CSV-файл и возвращает количество обработанных элементов:

```typescript
const processCSV = inngest.createFunction(
  { id: "process-csv-upload", triggers: { event: "imports/csv.uploaded" } },
  async ({ event, step }) => {
    // CSV processing logic omitted for the sake of the example
    return {
      status: "success",
      processedItems: results.length,
      failedItems: failures.length,
    }
  }
);
```

### Запуск функции

Для запуска этой функции отправьте событие `"imports/csv.uploaded"` с помощью `inngest.send()` с необходимыми данными полезной нагрузки. Функция `inngest.send()` возвращает массив ID событий, которые вы будете использовать для получения статуса и результата запуска функции.

```typescript
const { ids } = await inngest.send({
  name: "imports/csv.uploaded",
  data: {
    file: "http://s3.amazonaws.com/acme-uploads/user_0xp3wqz7vumcvajt/JVLO6YWS42IXEIGO.csv",
    userId: "user_0xp3wqz7vumcvajt",
  },
});
// ids = ["01HWAVEB858VPPX47Z65GR6P6R"]
```

### Получение статуса и результата запущенной функции

С помощью REST API можно использовать ID события для получения всех запусков, инициированных этим событием, через [эндпоинт запусков события](https://api-docs.inngest.com/docs/inngest-api/yoyeen3mu7wj0-list-event-function-runs):

```bash
https://api.inngest.com/v1/events/01HWAVEB858VPPX47Z65GR6P6R/runs
```

Для запроса можно использовать простой `fetch`-запрос с ключом подписи для аутентификации в API. Обернём это в многократно используемую функцию:

```typescript
async function getRuns(eventId) {
  const response = await fetch(`https://api.inngest.com/v1/events/${eventId}/runs`, {
    headers: {
      Authorization: `Bearer ${process.env.INNGEST_SIGNING_KEY}`,
    },
  });
  const json = await response.json();
  return json.data;
}
```

Теперь можно использовать ID события для получения статуса и результата запуска функции. Функция `getRuns` вернёт массив запусков, так как события могут инициировать несколько запусков через [fan-out](/docs/guides/fan-out-jobs). Будем считать, что это событие запускает только одну функцию:

```typescript
const runs = await getRuns("01HWAVEB858VPPX47Z65GR6P6R");
console.log(runs[0]);
/*
{
  run_id: '01HWAVJ8ASQ5C3FXV32JS9DV9Q',
  run_started_at: '2024-04-25T14:46:45.337Z',
  function_id: '6219fa64-9f58-41b6-95ec-a45c7172fa1e',
  function_version: 12,
  environment_id: '6219fa64-9f58-41b6-95ec-a45c7172fa1e',
  event_id: '01HWAVEB858VPPX47Z65GR6P6R',
  status: 'Completed',
  ended_at: '2024-04-25T14:46:46.896Z',
  output: {
    status: "success",
    processedItems: 98,
    failedItems: 2,
  }
}
*/
```

Если нужно запустить функцию и сразу дождаться её результата в том же коде, можно обернуть `getRuns` для опроса до достижения статуса `Completed`:

```typescript
async function getRunOutput(eventId) {
  let runs = await getRuns(eventId);
  while (runs[0].status !== "Completed") {
    await new Promise((resolve) => setTimeout(resolve, 1000));
    runs = await getRuns(eventId);
    if (runs[0].status === "Failed" || runs[0].status === "Cancelled") {
      throw new Error(`Function run ${runs[0].status}`);
    }
  }
  return runs[0];
}
```

### Всё вместе

Объединяя всё это, мы можем запустить функцию и дождаться её результата:

```typescript
const { ids } = await inngest.send({
  name: "imports/csv.uploaded",
  data: {
    file: "http://s3.amazonaws.com/acme-uploads/user_0xp3wqz7vumcvajt/JVLO6YWS42IXEIGO.csv",
    userId: "user_0xp3wqz7vumcvajt",
  },
});

const run = await getRunOutput(ids[0]);
console.log(run.output);
/*
{
  status: "success",
  processedItems: 98,
  failedItems: 2,
}
*/
```
