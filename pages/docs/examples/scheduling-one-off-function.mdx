# Планирование разовой функции

Inngest предоставляет способ отложить запуск функции до определённого момента в будущем. Это полезно когда:

* Вы хотите запланировать работу в будущем на основе пользовательского ввода.
* Вы хотите немного задержать выполнение несрочной функции на несколько секунд или минут.

На этой странице приведён быстрый пример того, как отложить запуск функции до определённого момента в будущем, используя поле `ts` [полезной нагрузки события](/docs/events#event-payload-format).

## Быстрый пример

Вот базовая функция, которая отправляет напоминание пользователю на указанный email.

```typescript
const sendReminder = inngest.createFunction(
  { id: "send-reminder", triggers: { event: "notifications/reminder.scheduled" } },
  async ({ event, step }) => {
    const { user, message } = event.data;

    const { id } = await emailApi.send({
      to: user.email,
      subject: "Reminder for your upcoming event",
      body: message,
    });

    return { id }
  }
);
```

### Запуск функции с временной меткой

Для запуска этой функции отправьте событие `"notifications/reminder.scheduled"` с помощью `inngest.send()` с необходимыми данными. Поле `ts` в [полезной нагрузке события](/docs/events#event-payload-format) должно быть установлено на время, в которое вы хотите запустить функцию, в миллисекундах с начала эпохи Unix. Например, для планирования напоминания через 5 минут:

```typescript
await inngest.send({
  name: "notifications/reminder.scheduled",
  data: {
    user: { email: "johnny.utah@fbi.gov" }
    message: "Don't forget to catch the wave at 3pm",
  },
  // Include the timestamp for 5 minutes in the future:
  ts: Date.now() + 5 * 60 * 1000,
});
```

<Callout variant="warning">
  ⚠️ Указание временной метки в событии применяется только для запуска новых запусков функций. Функции, ожидающие подходящего события, немедленно возобновятся независимо от временной метки.
</Callout>

### Альтернативы

В зависимости от вашего случая использования вы можете рассмотреть [запланированные функции (cron-задачи)](/docs/guides/scheduled-functions) для периодических задач или использовать [`step.sleepUntil()`](/docs/reference/functions/step-sleep-until) для добавления задержек внутри функции.
