# Устойчивые эндпоинты (Durable Endpoints)

[Устойчивые эндпоинты](/docs/learn/durable-endpoints) позволяют добавить устойчивость к обычным HTTP-обработчикам без отдельных определений функций или триггеров событий. Вы оборачиваете существующий маршрут API с помощью `inngest.endpoint()` и используете `step.run()` внутри для получения автоматических повторных попыток и мемоизации для каждого шага.

Это полезно, когда вы хотите, чтобы ваш эндпоинт оркестрировал несколько операций (например, бронирование рейса, обработку платежа и отправку подтверждения) и гарантировал, что каждый шаг выполнится ровно один раз, даже если обработчик аварийно завершится или перезапустится в процессе.


## Отличие от традиционного подхода Inngest

При традиционном подходе вы определяете отдельные функции, запускаемые событиями. Устойчивые эндпоинты меняют это: вы сохраняете HTTP-обработчики и делаете их устойчивыми изнутри.

| Традиционный Inngest              | Устойчивые эндпоинты        |
| --------------------------------- | --------------------------- |
| Отдельные функции                 | Внутри HTTP-обработчиков    |
| Запуск через события              | Прямые HTTP-вызовы          |
| `inngest.createFunction()`        | `inngest.endpoint()`        |
| `{ event, step }` из контекста    | Прямой импорт `step`        |
| Отдельный эндпоинт `/api/inngest` | Отдельный эндпоинт не нужен |

## Настройка клиента

Сначала настройте клиент Inngest с адаптером эндпоинта для вашего фреймворка:

<CodeGroup>
```typescript {{ title: "Next.js" }}
import { Inngest } from "inngest";
import { endpointAdapter } from "inngest/next";

const inngest = new Inngest({
  id: "my-app",
  endpointAdapter,
});
```
```typescript {{ title: "Bun" }}
import { Inngest } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({
  id: "my-app",
  endpointAdapter,
});
```
</CodeGroup>

`endpointAdapter` сообщает Inngest, как взаимодействовать с моделью запрос/ответ вашего фреймворка. Импортируйте из `"inngest/next"` для Next.js или `"inngest/edge"` для Bun. Это единственная необходимая настройка — отдельный serve-эндпоинт `/api/inngest` не нужен.

## Простой устойчивый эндпоинт

После настройки клиента вы можете превратить любой обработчик маршрута в устойчивый эндпоинт. Каждый вызов `step.run()` мемоизируется: если обработчик выполняется повторно (из-за повторной попытки или восстановления после сбоя), завершённые шаги пропускаются и их предыдущие результаты используются повторно.

<CodeGroup>
```typescript {{ title: "Next.js" }}
import { step } from "inngest";
import { inngest } from "@/inngest/client";
import { NextRequest } from "next/server";

export const POST = inngest.endpoint(async (req: NextRequest) => {
  const { userId, data } = await req.json();

  // Step 1: Validate and enrich the data
  const enriched = await step.run("enrich-data", async () => {
    const user = await db.users.find(userId);
    return { ...data, account: user.accountId };
  });

  // Step 2: Process the enriched data
  const result = await step.run("process", async () => {
    return await processData(enriched);
  });

  // Step 3: Send notification
  await step.run("notify", async () => {
    await sendNotification(userId, result);
  });

  return Response.json({ success: true, result });
});
```
```typescript {{ title: "Bun" }}
import { Inngest, step } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({ id: "my-app", endpointAdapter });

Bun.serve({
  port: 3000,
  routes: {
    "/process": inngest.endpoint(async (req) => {
      const { userId, data } = await req.json();

      // Step 1: Validate and enrich the data
      const enriched = await step.run("enrich-data", async () => {
        const user = await db.users.find(userId);
        return { ...data, account: user.accountId };
      });

      // Step 2: Process the enriched data
      const result = await step.run("process", async () => {
        return await processData(enriched);
      });

      // Step 3: Send notification
      await step.run("notify", async () => {
        await sendNotification(userId, result);
      });

      return new Response(JSON.stringify({ success: true, result }));
    }),
  },
});
```
</CodeGroup>

Если обработчик аварийно завершается после выполнения "enrich-data", но до завершения "process", Inngest повторно вызывает обработчик. Первый шаг мгновенно возвращает кешированный результат, и выполнение продолжается со второго шага. Без дублирования работы, без потери прогресса.

## Оркестрация нескольких шагов с повторными попытками

Реальные рабочие процессы часто включают несколько внешних систем, которые могут отказывать независимо. Устойчивые эндпоинты справляются с этим естественным образом, поскольку каждый шаг повторяется самостоятельно без повторного выполнения предыдущих шагов.

Рассмотрим следующий демо-пример бронирования поездки, который ищет рейсы, резервирует место, обрабатывает платёж и подтверждает бронирование:

Реализация устойчивого эндпоинта будет выглядеть следующим образом:


<CodeGroup>
```typescript {{ title: "Next.js" }}
import { step } from "inngest";
import { inngest } from "@/inngest/client";
import { NextRequest } from "next/server";

export const GET = inngest.endpoint(async (req: NextRequest) => {
  const url = new URL(req.url);
  const origin = url.searchParams.get("origin") || "NYC";
  const destination = url.searchParams.get("destination") || "LAX";
  const date = url.searchParams.get("date") || new Date().toISOString().split("T")[0];

  // Step 1: Search for available flights
  const availability = await step.run("search-availability", async () => {
    const flights = await searchFlights(origin, destination, date);
    return { flights };
  });

  // Step 2: Reserve the best flight
  // If the seat lock times out, Inngest automatically retries this step
  // without re-running the search
  const reservation = await step.run("reserve-flight", async () => {
    const best = availability.flights[0];
    return await reserveSeat(best.flightNumber, "14A");
  });

  // Step 3: Process payment
  const payment = await step.run("process-payment", async () => {
    return await chargeCard(reservation.price, reservation.pnr);
  });

  // Step 4: Confirm the booking
  const confirmation = await step.run("confirm-booking", async () => {
    return await confirmReservation(reservation.reservationId);
  });

  return Response.json({
    success: true,
    trip: { origin, destination, date },
    reservation,
    payment,
    confirmation,
  });
});
```
```typescript {{ title: "Bun" }}
import { Inngest, step } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({ id: "trip-booker", endpointAdapter });

Bun.serve({
  port: 3000,
  routes: {
    "/api/booking": inngest.endpoint(async (req) => {
      const url = new URL(req.url);
      const origin = url.searchParams.get("origin") || "NYC";
      const destination = url.searchParams.get("destination") || "LAX";
      const date = url.searchParams.get("date") || new Date().toISOString().split("T")[0];

      // Step 1: Search for available flights
      const availability = await step.run("search-availability", async () => {
        const flights = await searchFlights(origin, destination, date);
        return { flights };
      });

      // Step 2: Reserve the best flight
      // If the seat lock times out, Inngest automatically retries this step
      // without re-running the search
      const reservation = await step.run("reserve-flight", async () => {
        const best = availability.flights[0];
        return await reserveSeat(best.flightNumber, "14A");
      });

      // Step 3: Process payment
      const payment = await step.run("process-payment", async () => {
        return await chargeCard(reservation.price, reservation.pnr);
      });

      // Step 4: Confirm the booking
      const confirmation = await step.run("confirm-booking", async () => {
        return await confirmReservation(reservation.reservationId);
      });

      return new Response(JSON.stringify({
        success: true,
        trip: { origin, destination, date },
        reservation,
        payment,
        confirmation,
      }));
    }),
  },
});
```
</CodeGroup>

Каждый шаг в этом рабочем процессе независимо устойчив:
- Если резервирование места завершается с таймаутом, повторяется только этот шаг. Результаты поиска рейсов сохраняются.
- Если шаг оплаты завершается с ошибкой, бронирование остаётся активным и оплата повторяется без повторного резервирования.
- Если весь сервер аварийно завершается после успешной оплаты, обработчик возобновляется на шаге подтверждения.

## Параллельное выполнение шагов

Если шаги не зависят друг от друга, их можно запускать параллельно с помощью `Promise.all()`. Каждый параллельный шаг по-прежнему независимо устойчив и повторяется при необходимости.

<CodeGroup>
```typescript {{ title: "Next.js" }}
import { step } from "inngest";
import { inngest } from "@/inngest/client";
import { NextRequest } from "next/server";

export const POST = inngest.endpoint(async (req: NextRequest) => {
  const { flightId, hotelId, carId } = await req.json();

  // Book flight, hotel, and car rental in parallel
  // Each step retries independently if it fails
  const [flight, hotel, car] = await Promise.all([
    step.run("book-flight", async () => {
      return await bookFlight(flightId);
    }),
    step.run("book-hotel", async () => {
      return await bookHotel(hotelId);
    }),
    step.run("book-car", async () => {
      return await bookCarRental(carId);
    }),
  ]);

  // Send a single confirmation after all bookings succeed
  await step.run("send-confirmation", async () => {
    await sendTripConfirmation({
      flight,
      hotel,
      car,
    });
  });

  return Response.json({
    success: true,
    bookings: { flight, hotel, car },
  });
});
```
```typescript {{ title: "Bun" }}
import { Inngest, step } from "inngest";
import { endpointAdapter } from "inngest/edge";

const inngest = new Inngest({ id: "my-app", endpointAdapter });

Bun.serve({
  port: 3000,
  routes: {
    "/api/book-trip": inngest.endpoint(async (req) => {
      const { flightId, hotelId, carId } = await req.json();

      // Book flight, hotel, and car rental in parallel
      // Each step retries independently if it fails
      const [flight, hotel, car] = await Promise.all([
        step.run("book-flight", async () => {
          return await bookFlight(flightId);
        }),
        step.run("book-hotel", async () => {
          return await bookHotel(hotelId);
        }),
        step.run("book-car", async () => {
          return await bookCarRental(carId);
        }),
      ]);

      // Send a single confirmation after all bookings succeed
      await step.run("send-confirmation", async () => {
        await sendTripConfirmation({
          flight,
          hotel,
          car,
        });
      });

      return new Response(JSON.stringify({
        success: true,
        bookings: { flight, hotel, car },
      }));
    }),
  },
});
```
</CodeGroup>

Если бронирование отеля завершается с ошибкой, оно повторяется независимо, пока результаты бронирования рейса и аренды автомобиля сохраняются. Как только все три завершатся успешно, выполняется шаг подтверждения.
