# Fetch: выполнение API-запросов и получение данных <VersionBadge version="TypeScript only" />

TypeScript SDK Inngest предоставляет API `step.fetch()` и утилиту `fetch()`, позволяя выполнять запросы к сторонним API или получать данные устойчивым способом, передавая их на платформу Inngest.

Для получения дополнительной информации о работе Fetch смотрите [документацию по Fetch](/docs/features/inngest-functions/steps-workflows/fetch).

## Начало работы с `step.fetch()`

API `step.fetch()` позволяет выполнять устойчивые HTTP-запросы, передавая их на платформу Inngest, что экономит вычислительные ресурсы и повышает надёжность:

```ts {{ title: "src/inngest/functions.ts" }}
import { inngest } from "./client";

export const retrieveTextFile = inngest.createFunction(
  { id: "retrieveTextFile", triggers: { event: "textFile/retrieve" } },
  async ({ step }) => {
    // The fetching of the text file is offloaded to the Inngest Platform
    const response = await step.fetch(
      "https://example-files.online-convert.com/document/txt/example.txt"
    );

    // The Inngest function run is resumed when the HTTP request is complete
    await step.run("extract-text", async () => {
      const text = await response.text();
      const exampleOccurences = text.match(/example/g);
      return exampleOccurences?.length;
    });
  }
);
```

`step.fetch()` принимает те же аргументы, что и [нативный API `fetch`](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch).

## Параллельные HTTP-запросы с `step.fetch()`

`step.fetch()` разделяет все преимущества `step.run()`, включая возможность параллельного выполнения запросов с помощью `Promise.all()`:

```typescript
const processFiles = inngest.createFunction(
  { id: "process-files", concurrency: 10, triggers: { event: "files/process" } },
  async ({ step, event }) => {
    // All requests will be offloaded and processed in parallel while matching the concurrency limit
    const responses = await Promise.all(event.data.files.map(async (file) => {
      return step.fetch(`https://api.example.com/files/${file.id}`)
    }))

    // Your Inngest function is resumed here with the responses
    await step.run("process-file", async (file) => {
      const body = await response.json()
      // body.files
    })
  }
)
```
Обратите внимание, что `step.fetch()`, как и все другие API `step`, соответствует конфигурации вашей функции, например [конкурентности](/docs/guides/concurrency) или [троттлингу](/docs/guides/throttling).

## Устойчивые HTTP-запросы сторонних библиотек с утилитой `fetch()`

Утилита `fetch()` Inngest может быть передана как пользовательский обработчик fetch, чтобы сделать все запросы сторонней библиотеки устойчивыми.

Например, можно передать утилиту `fetch()` в AI SDK или библиотеки OpenAI:

<Callout variant="warning">
  При использовании утилиты `fetch()` с AI SDK необходимо отключить встроенный механизм повторных попыток AI SDK и позволить Inngest обрабатывать повторные попытки.

  Ниже показано использование утилиты `fetch()` с AI SDK с отключением встроенного механизма повторных попыток AI SDK через `maxRetries: 0`.
</Callout>

<CodeGroup>
```typescript {{ title: "AI SDK" }}
import { fetch as inngestFetch } from 'inngest';
import { generateText } from 'ai';
import { createAnthropic } from '@ai-sdk/anthropic';

// Pass the Inngest fetch utility to the AI SDK's model constructor:
const anthropic = createAnthropic({
  fetch: inngestFetch,
});

const weatherFunction = inngest.createFunction(
  { id: "weather-function", triggers: { event: "weather/get" } },
  async ({ step }) => {
    // This request is offloaded to the Inngest platform
    // and it also retries automatically if it fails!
    const response = await generateText({
      model: anthropic('claude-3-5-sonnet-20240620'),
      prompt: `What's the weather in London?`,
      maxRetries: 0,
    });
  }
)
```
```typescript {{ title: "OpenAI SDK" }}
import { fetch } from "inngest";
import OpenAI from 'openai';

const client = new OpenAI({ fetch });

// use the global fetch
const completion = await client.chat.completions.create({
  model: "gpt-3.5-turbo",
  messages: [{ role: "user", content: "Hello, world!" }],
});

const weatherFunction = inngest.createFunction(
  { id: "weather-function", triggers: { event: "weather/get" } },
  async ({ step }) => {
    // The OpenAI request is automatically offloaded to the Inngest Platform
    const completion = await client.chat.completions.create({
      model: "gpt-3.5-turbo",
      messages: [{ role: "user", content: "What's the weather in London?" }],
    });
  }
)
```
