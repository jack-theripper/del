# Realtime: Стриминг обновлений из функций Inngest

Inngest Realtime позволяет стримить обновления из ваших функций, поддерживать живые UI и реализовывать двунаправленные рабочие процессы, такие как «Человек в цикле». Используйте каналы и топики для трансляции обновлений, стриминга логов или ожидания пользовательского ввода.

## Паттерн: Стриминг обновлений из одного запуска функции

Дайте пользователям возможность следить за прогрессом длительной задачи, стримя обновления из выделенного канала. Вот как запустить функцию и подписаться на её обновления:

```ts { title: "app/api/hello-world/route.ts" }
import crypto from "crypto";
import { inngest } from "@/inngest/client";
import { subscribe } from "@inngest/realtime";

export async function POST(req: Request) {
  const json = await req.json();
  const { prompt } = json;
  const uuid = crypto.randomUUID();

  await inngest.send({
    name: "hello-world/hello",
    data: { uuid },
  });

  const stream = await subscribe(inngest, {
    channel: `hello-world.${uuid}`,
    topics: ["logs"],
  });

  return new Response(stream.getEncodedStream(), {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      Connection: "keep-alive",
    },
  });
}
```

Затем ваша функция может публиковать обновления в этот канал:

```ts { title: "src/inngest/functions/hello-world.ts" }
import { Inngest } from "inngest";
import { realtimeMiddleware, channel, topic } from "@inngest/realtime";

const inngest = new Inngest({
  id: "my-app",
  middleware: [realtimeMiddleware()],
});

export const helloChannel = channel((uuid: string) => `hello-world.${uuid}`).addTopic(
  topic("logs").type<string>()
);

export const someTask = inngest.createFunction(
  { id: "hello-world", triggers: [{ event: "hello-world/hello" }] },
  async ({ event, step, publish }) => {
    const { uuid } = event.data;
    await publish(helloChannel(uuid).logs("Hello, world!"));
  }
);
```

Создавая канал с уникальным идентификатором, вы можете стримить обновления для конкретного запуска конечному пользователю.

## Паттерн: Стриминг обновлений из нескольких запусков функций

Realtime-канал можно использовать для стриминга обновлений из нескольких запусков функций. Здесь определим два канала: один глобальный и один для конкретного поста:

```ts {{ filename: "src/inngest/channels.ts" }}
import {
  channel,
  topic,
} from "@inngest/realtime";
import { z } from "zod";

export const globalChannel = channel("global").addTopic(topic("logs").type<string>());

export const postChannel = channel((postId: string) => `post:${postId}`)
  .addTopic(
    topic("updated").schema(
      z.object({
        id: z.string(),
        likes: z.number(),
      })
    )
  )
  .addTopic(
    topic("deleted").schema(
      z.object({
        id: z.string(),
        reason: z.string(),
      })
    )
  );
```

Наша функция `likePost` будет публиковать обновления в оба канала:

```ts {{ filename: "src/inngest/functions/likePost.ts" }}
import {
  channel,
  realtimeMiddleware,
  subscribe,
  topic,
} from "@inngest/realtime";
import { Inngest } from "inngest";
import { globalChannel, postChannel } from "../channels";

export const likePost = app.createFunction(
  {
    id: "post/like",
    retries: 0,
    triggers: [{ event: "app/post.like" }],
  },
  async ({
    event: {
      data: { postId = "123" },
    },
    step,
    publish,
  }) => {
    if (!postId) {
      await publish(
        globalChannel().logs("Missing postId when trying to like post")
      );
      throw new Error("Missing postId");
    }

    await publish(globalChannel().logs(`Liking post ${postId}`));

    // Fake a post update
    const post = await step.run("Update likes", async () => {
      const fakePost = {
        id: "123",
        likes: Math.floor(Math.random() * 10000),
      };

      return publish(postChannel(fakePost.id).updated(fakePost));
    });

    return post;
  }
);
```

`globalChannel` будет использоваться для стриминга обновлений для всех постов, а `postChannel` — для стриминга обновлений конкретных постов.

## Человек в цикле: Двунаправленные рабочие процессы

Объедините Realtime с `waitForEvent()` для создания рабочих процессов, требующих пользовательского ввода, например [шагов проверки или согласования](/docs/ai-patterns/human-in-the-loop). Вот как отправить сообщение пользователю и ожидать его подтверждения:

```ts { title: "src/inngest/functions/agentic-workflow.ts" }
import crypto from "crypto";
import { Inngest } from "inngest";
import { realtimeMiddleware, channel, topic } from "@inngest/realtime";

const inngest = new Inngest({
  id: "my-app",
  middleware: [realtimeMiddleware()],
});

export const agenticWorkflowChannel = channel("agentic-workflow").addTopic(
  topic("messages").schema(
    z.object({
      message: z.string(),
      confirmationUUid: z.string(),
    })
  )
);

export const agenticWorkflow = inngest.createFunction(
  { id: "agentic-workflow", triggers: [{ event: "agentic-workflow/start" }] },
  async ({ event, step, publish }) => {
    await step.run(/* ... */);
    const confirmationUUid = await step.run("get-confirmation-uuid", async () => crypto.randomUUID());
    await publish(agenticWorkflowChannel().messages({
      message: "Confirm to proceed?",
      confirmationUUid,
    }));
    const confirmation = await step.waitForEvent("wait-for-confirmation", {
      event: "agentic-workflow/confirmation",
      timeout: "15m",
      if: `async.data.confirmationUUid == \"${confirmationUUid}\"`,
    });
    if (confirmation) {
      // continue workflow
    }
  }
);
```

`confirmationUUid` связывает опубликованное сообщение с ответным событием, обеспечивая обработку правильного ответа пользователя.
