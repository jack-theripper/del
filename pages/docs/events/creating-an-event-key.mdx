# Создание ключа события

«Ключи событий» — это уникальные ключи, позволяющие приложениям отправлять (публиковать) события в Inngest. При использовании ключей событий с [Inngest SDK](/docs/events) вы можете настроить клиент `Inngest` двумя способами:

1. Установив ключ как переменную окружения [`INNGEST_EVENT_KEY`](/docs/sdk/environment-variables#inngest-event-key) в вашем приложении*
2. Передав ключ в качестве аргумента

```jsx
// Рекомендуется: установите переменную окружения INNGEST_EVENT_KEY для автоматической настройки:
const inngest = new Inngest({ name: "Your app name" });

// Или передайте eventKey явно в конструктор:
const inngest = new Inngest({ name: "Your app name", eventKey: "xyz..." });

// С ключом события вы готовы отправлять данные:
await inngest.send({ ... })
```

\* Наша [интеграция с Vercel](/docs/deploy/vercel) автоматически устанавливает [`INNGEST_EVENT_KEY`](/docs/sdk/environment-variables#inngest-event-key) как переменную окружения

<Callout>
  🙋 Ключи событий должны быть уникальны для конкретной <strong>среды</strong> (например, production, среды веток) и конкретного <strong>приложения</strong> (ваш API, ваше мобильное приложение и т.д.). Разделение ключей по приложениям упрощает управление ключами и их ротацию при необходимости.
</Callout>

<Callout>
  🔐 **Защита ключей событий** — Поскольку ключи событий используются для отправки данных в вашу среду Inngest, необходимо принять меры по их защите. Не храните их в исходном коде и по возможности сохраняйте ключи как секреты на вашей платформе.
</Callout>

## Создание нового ключа события

В панели управления Inngest Cloud ключи событий перечислены на вкладке «Manage»:

1. Нажмите «Manage» ([прямая ссылка](https://app.inngest.com/env/production/manage/keys))
2. Нажмите кнопку «+ Create Event Key» в правом верхнем углу
3. Укажите понятное название для ключа события и нажмите «Save changes»
4. Скопируйте созданный ключ с помощью кнопки «Copy»:

![Только что созданный ключ события в панели управления Inngest Cloud](/assets/docs/creating-an-event-key/new-event-key-2023.png)

🎉 Теперь вы можете использовать этот ключ события с Inngest SDK для отправки событий из любой кодовой базы. Вы также можете:

- Переименовать ключ события в любое время с помощью поля «Name», чтобы вы и ваша команда могли его идентифицировать
- Удалить ключ события, когда он больше не нужен
- Фильтровать события по имени или IP-адресам для расширенного контроля и безопасности

<Callout variant="warning">
  <span className="text-yellow-300">⚠️</span> Хотя технически _возможно_ использовать ключи событий для отправки событий из браузера, такая практика несёт риски: любой, кто изучит ваш клиентский код, сможет прочитать ваш ключ и отправлять события в вашу среду Inngest. Если вы хотите отправлять события со стороны клиента, рекомендуем создать конечную точку API или edge-функцию для проксирования отправки событий.
</Callout>
