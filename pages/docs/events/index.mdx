# Отправка событий

Для начала убедитесь, что вы [установили Inngest SDK](/docs/sdk/overview).

Для отправки событий необходимо создать экземпляр клиента `Inngest`. Рекомендуется сделать это в одном файле и экспортировать клиент, чтобы импортировать его в любом месте приложения. В production-среде вам понадобится ключ события, [который мы рассмотрим ниже](#setting-an-event-key).

<LanguageSelector>

<LanguageSection show="typescript">

```ts {{ filename: 'inngest/client.ts' }}
import { Inngest } from "inngest";

export const inngest = new Inngest({ id: "acme-storefront-app" }); // Используйте ID вашего приложения
```

Теперь с этим клиентом вы можете отправлять события из любого места приложения. Вы можете отправить одно событие или [несколько событий одновременно](#sending-multiple-events-at-once).

```ts {{ filename: 'app/api/checkout/route.ts' }}
import { inngest } from "../inngest/client";

// Отправляет событие в Inngest.
await inngest.send({
  // Название события
  name: "storefront/cart.checkout.completed",
  // Данные события
  data: {
    cartId: "ed12c8bde",
    itemIds: ["9f08sdh84", "sdf098487", "0fnun498n"],
    account: {
      id: 123,
      email: "test@example.com",
    },
  },
});
```

<Callout>

👉 `send()` — это асинхронный метод, возвращающий `Promise`. Всегда используйте `await` или `.then()`, чтобы убедиться, что метод завершил отправку события в Inngest. Бессерверные функции могут завершаться очень быстро, поэтому пропуск `await` может привести к тому, что события не будут отправлены.

</Callout>

</LanguageSection>

<LanguageSection show="python">

```python {{ filename: 'src/inngest/client.py' }}
import inngest

inngest_client = inngest.Inngest(app_id="acme-storefront-app")
```

Теперь с этим клиентом вы можете отправлять события из любого места приложения. Вы можете отправить одно событие или [несколько событий одновременно](#sending-multiple-events-at-once).

```python {{ filename: 'src/api/checkout/route.py' }}
import inngest
from src.inngest.client import inngest_client

# Отправляет событие в Inngest.
await inngest_client.send(
  inngest.Event(
    name="storefront/cart.checkout.completed",
    data={
      "cartId": "ed12c8bde",
      "itemIds": ["9f08sdh84", "sdf098487", "0fnun498n"],
      "account": {
        "id": 123,
        "email": "test@example.com",
      },
    },
  )
)
```

<Callout>

👉 `send()` предназначен для асинхронного вызова через `await`. Для синхронного кода [используйте метод `send_sync()` вместо него](/docs/reference/python/client/send).

Отправка этого события с именем `storefront/cart.checkout.completed` в Inngest выполняет два действия:

1. Автоматически запускает все [функции](/docs/functions), триггером которых является это конкретное событие, передавая полезную нагрузку события в аргументы функции.
2. Сохраняет полезную нагрузку события в Inngest Cloud. Её можно найти на вкладке **Events** в панели управления.

<Callout>

💡 Одно событие может запускать несколько функций, что позволяет обрабатывать одно событие несколькими способами. Это отличается от традиционных очередей сообщений, где только один обработчик может потреблять одно сообщение. Узнайте о [подходе fan-out здесь](/docs/guides/fan-out-jobs).

</Callout>

## Настройка ключа события

В production-среде вашему приложению потребуется «ключ события» для отправки событий в Inngest. Это секретный ключ, используемый для аутентификации вашего приложения и гарантирующий, что только ваше приложение может отправлять события в конкретную [среду](/docs/platform/environments) вашей учётной записи Inngest.

Вы можете узнать [как создать ключ события здесь](/docs/events/creating-an-event-key). Получив ключ, вы можете установить его одним из двух способов:

1. Установите переменную окружения `INNGEST_EVENT_KEY` со значением вашего ключа события. **Это рекомендуемый подход.**
2. Передайте ключ события в конструктор `Inngest` в качестве опции `eventKey`:

<LanguageSection show="typescript">
```ts {{ filename: 'inngest/client.ts' }}
import { Inngest } from "inngest";

// ПРИМЕЧАНИЕ — Не рекомендуется жёстко задавать ключ события в коде.
const inngest = new Inngest({ id: "your-app-id", eventKey: "xyz..." });
```
</LanguageSection>

<LanguageSection show="python">
```python {{ filename: 'src/inngest/client.py' }}
import inngest

# Не рекомендуется жёстко задавать ключ события в коде.
inngest_client = inngest.Inngest(app_id="your-app-id", event_key="xyz...")
```
</LanguageSection>

<Callout>

Ключи событий _не требуются_ при локальной разработке с [Inngest Dev Server](/docs/local-development). Вы можете опустить их в разработке, и события всё равно будут отправлены на Dev Server.

</Callout>

## Формат полезной нагрузки события

Полезная нагрузка события — это JSON-объект, который должен содержать свойства `name` и `data`.

Ознакомьтесь со всеми свойствами событий в [руководстве по формату полезной нагрузки событий](/docs/features/events-triggers/event-format).


## Отправка нескольких событий одновременно

Вы также можете отправить несколько событий в одном вызове `send()`. Это позволяет легко отправлять пакеты событий. Один запрос может содержать до `512 КБ` данных, что обычно позволяет отправить от 10 до 1000 стандартных полезных нагрузок. Это значение по умолчанию и может быть увеличено для вашей учётной записи.

<LanguageSection show="typescript">

```ts
await inngest.send([
  { name: "storefront/cart.checkout.completed", data: { ... } },
  { name: "storefront/coupon.used", data: { ... } },
  { name: "storefront/loyalty.program.joined", data: { ... } },
])
```

Это особенно удобно, если у вас есть массив данных и нужно отправить событие для каждого элемента:

```ts
// Этот вызов функции может вернуть десятки или сотни элементов,
// поэтому используем map для преобразования элементов в полезные нагрузки событий
// и передаём полученный массив в send:
const importedItems = await api.fetchAllItems();
const events = importedItems.map((item) => ({
  name: "storefront/item.imported",
  data: {
    ...item,
  }
}));
await inngest.send(events);
```

## Отправка событий из функций

Вы также можете отправлять события из своих функций с помощью `step.sendEvent()`, чтобы, например, запускать другие функции. Подробнее об [отправке событий из функций](/docs/guides/sending-events-from-functions). Внутри функций `step.sendEvent()` оборачивает запрос отправки события в `step`, обеспечивая надёжную доставку и предотвращая отправку дублирующихся событий. Рекомендуется использовать `step.sendEvent()` вместо `inngest.send()` внутри функций.

```ts
export default inngest.createFunction(
  { id: "user-onboarding", triggers: { event: "app/user.signup" } },
  async ({ event, step }) => {
    // Выполните какое-либо действие
    await step.sendEvent("send-activation-event", {
      name: "app/user.activated",
      data: { userId: event.data.userId },
    });
    // Выполните другое действие
  }
);
```

## Использование ID событий

Каждому событию, отправленному в Inngest, присваивается уникальный ID. Эти `ids` возвращаются из `inngest.send()` или `step.sendEvent()`. ID событий можно использовать для поиска события в панели управления Inngest или через [REST API](https://api-docs.inngest.com/docs/inngest-api/pswkqb7u3obet-get-an-event). Вы можете записывать или сохранять эти ID, если захотите найти их позже.

```ts
const { ids } = await inngest.send([
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e024befa68763f5b500" }
  },
  {
    name: "app/invoice.created",
    data: { invoiceId: "645e9e08f29fb563c972b1f7" }
  },
]);
/**
 * ids = [
 *   "01HQ8PTAESBZPBDS8JTRZZYY3S",
 *   "01HQ8PTFYYKDH1CP3C6PSTBZN5"
 * ]
 */
```

</LanguageSection>

```python
await inngest_client.send([

])
```

## Отправка событий из функций

Вы также можете отправлять события из своих функций с помощью `step.send_event()`, чтобы, например, запускать другие функции. Подробнее об [отправке событий из функций](/docs/guides/sending-events-from-functions). Внутри функций `step.send_event()` оборачивает запрос отправки события в `step`, обеспечивая надёжную доставку и предотвращая отправку дублирующихся событий. Рекомендуется использовать `step.send_event()` вместо `inngest.send()` внутри функций.

```python
import inngest
from src.inngest.client import inngest_client

@inngest_client.create_function(
    fn_id="my_function",
    trigger=inngest.TriggerEvent(event="app/my_function"),
)
async def fn(ctx: inngest.Context) -> list[str]:
    return await ctx.step.send_event("send", inngest.Event(name="foo"))
```

## Отправка событий через HTTP (Event API)

Вы можете отправлять события из любой системы или языка программирования с помощью нашего API и ключа события Inngest. API принимает как одну полезную нагрузку события, так и массив полезных нагрузок.

{/* ПРИМЕЧАНИЕ — Оставим другие SDK здесь пока, но со временем вместо этого сделаем всё руководство с примерами для Python и Go в каждом разделе выше */}

Чтобы отправить событие в конкретную [среду ветки](/docs/platform/environments#branch-environments), установите заголовок `x-inngest-env` равным имени вашей среды, например: `x-inngest-env: feature/my-branch`.

<CodeGroup tag="POST" label="inn.gs/e/:eventKey">

```bash {{ title: 'cURL' }}
curl -X POST https://inn.gs/e/$INNGEST_EVENT_KEY \
  -H 'Content-Type: application/json' \
  --data '{
    "name": "user.signup",
    "data": {
      "userId": "645ea8289ad09eac29230442"
    }
  }'
```

```php
$url = "https://inn.gs/e/{$eventKey}";
$content = json_encode([
    "name" => "user.signup",
    "data" => [
      "userId" => "645ea8289ad09eac29230442",
    ],
]);

$curl = curl_init($url);
curl_setopt($curl, CURLOPT_HEADER, false);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);
curl_setopt($curl, CURLOPT_HTTPHEADER, ["Content-type: application/json"]);
curl_setopt($curl, CURLOPT_POST, true);
curl_setopt($curl, CURLOPT_POSTFIELDS, $content);

$json_response = curl_exec($curl);
$status = curl_getinfo($curl, CURLINFO_HTTP_CODE);
if ($status != 200) {
    return [
        'status' => $status,
        'message' => "Error: call to URL $url failed with status $status, response $json_response, curl_error " . curl_error($curl) . ", curl_errno " . curl_errno($curl),
    ];
}

curl_close($curl);
$response = json_decode($json_response, true);
```

</CodeGroup>


<Note>
  При использовании [dev server](/docs/dev-server) используйте `http://localhost:8288/e/<fake-key>` в качестве конечной точки. При [самостоятельном размещении](/docs/self-hosting) замените на URL вашего экземпляра.
</Note>

Ответ будет содержать `ids` отправленных событий:

```json {{ title: 'Response' }}
{
  "ids": ["01H08W4TMBNKMEWFD0TYC532GG"],
  "status": 200
}
```

## Дедупликация

Нередко возникает необходимость предотвратить обработку дублирующихся событий в Inngest. Если ваша система может отправить одно и то же событие более одного раза, следует убедиться, что функции не запускаются повторно.

Чтобы предотвратить дублирующиеся запуски функций, добавьте параметр `id` в полезную нагрузку события. После получения события с `id` все события с тем же `id` будут проигнорированы независимо от их полезной нагрузки.

<LanguageSection show="typescript">

```ts
await inngest.send({
  // Ваш ID дедупликации должен быть специфичен для данной полезной нагрузки события.
  // Используйте что-то, что не будет повторяться в разных типах событий, а не универсальное значение вроде cartId
  id: "cart-checkout-completed-ed12c8bde",
  name: "storefront/cart.checkout.completed",
  data: {
    cartId: "ed12c8bde",
    // ...остальные данные полезной нагрузки...
  }
});
```
</LanguageSection>


<LanguageSection show="python">

```python
await inngest_client.send(
  inngest.Event(
    name="storefront/cart.checkout.completed",
    id="cart-checkout-completed-ed12c8bde",
    data={"cartId": "ed12c8bde"},
  )
)
```
</LanguageSection>


Подробнее об этом в [руководстве по обработке идемпотентности](/docs/guides/handling-idempotency).

<Callout variant="warning">
💡 Дедупликация предотвращает повторные запуски функций в течение 24 часов с момента получения первого события.
</Callout>

<Callout>

`id` является глобальным для всех типов событий, поэтому убедитесь, что ваш `id` не будет совпадать с ID других типов событий.

Например, для двух событий `storefront/item.imported` и `storefront/item.deleted` не используйте `id` самого `item` (`9f08sdh84`) в качестве ID дедупликации события. Вместо этого объедините ID элемента с типом события, чтобы он был уникален для данного события (например, `item-imported-9f08sdh84`).

</Callout>
