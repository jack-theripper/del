# Создание цикла инструментов агента (Agent Tool Loop)

В этом руководстве вы узнаете, как создать цикл агента в стиле ReAct (Рассуждение → Действие → Наблюдение → Повторение), где каждая итерация является надежным и отказоустойчивым шагом.

## Базовый цикл

Создайте функцию, которая принимает сообщение пользователя, вызывает LLM, выполняет запрошенные инструменты и повторяет процесс до тех пор, пока LLM не вернет окончательный ответ:

```ts
import { inngest } from "./client";
import Anthropic from "@anthropic-ai/sdk";

const anthropic = new Anthropic();

export const agent = inngest.createFunction(
  { id: "agent-loop", triggers: [{ event: "agent/message.received" }] },
  async ({ event, step }) => {
    const messages: Anthropic.MessageParam[] = [
	  // Подготовьте начальный массив сообщений с системными и пользовательскими промптами
      { role: "user", content: event.data.message },
    ];

    const MAX_ITERATIONS = 10;
    let iterations = 0;
    let done = false;

    while (!done && iterations < MAX_ITERATIONS) {
      iterations++;

      // 1. Рассуждение (Think) — спрашиваем LLM, что делать дальше
      const llmResult = await step.run(`think`, async () => {
        return await anthropic.messages.create({
          model: "claude-opus-4-6",
          max_tokens: 4096,
          // Используйте свой мастерски составленный промпт:
          system: "Вы — полезный помощник с доступом к инструментам.",
          messages,
          tools, // определения ваших инструментов
        });
      });

      // 2. Проверка, хочет ли LLM использовать инструменты
      const toolCalls = llmResult.content.filter(
        (block): block is Anthropic.ToolUseBlock => block.type === "tool_use"
      );

      if (toolCalls.length === 0) {
        // Инструменты не требуются — завершаем
        const text = llmResult.content.find(
          (b): b is Anthropic.TextBlock => b.type === "text"
        );
        done = true;
        return { response: text?.text ?? "", iterations };
      }

      // 3. Действие (Act) — выполнение каждого инструмента
      messages.push({ role: "assistant", content: llmResult.content });
      const toolResults: Anthropic.ToolResultBlockParam[] = [];

      for (const toolCall of toolCalls) {
        const result = await step.run(
          `tool-${toolCall.name}`,
          async () => executeTool(toolCall.name, toolCall.input)
        );
        toolResults.push({
          type: "tool_result",
          tool_use_id: toolCall.id,
          content: result,
        });
      }

      // 4. Наблюдение (Observe) — передаем результаты обратно и повторяем цикл
      messages.push({ role: "user", content: toolResults });
    }

    return { response: "Достигнут лимит итераций", iterations };
  }
);
```

Реализуйте `executeTool` любым удобным способом. Это может быть оператор `switch`, карта функций или система плагинов. Она принимает название инструмента и входные данные, а возвращает строковый результат.

Это весь паттерн. Ниже мы разберем каждую часть и способы ее настройки.

## Надежность вызовов LLM

Оборачивайте каждый вызов LLM в [`step.run()`](/docs/reference/typescript/v4/functions/step-run), чтобы сделать его отказоустойчивым и сохранять состояние:

```ts
// ✅ Надежно — повтор при сбое, результат сохраняется в контрольной точке
const result = await step.run("think", async () => {
  return await anthropic.messages.create({ /* ... */ });
});

// ❌ Ненадежно — при сбое вы потеряете всю предыдущую работу
const result = await anthropic.messages.create({ /* ... */ });
```

Когда вызов LLM завершается ошибкой (ограничение частоты, таймаут, сетевая ошибка), Inngest повторяет только этот шаг. Предыдущие итерации, результаты инструментов — всё остальное сохраняется.

## Выполнение каждого инструмента как шага

Оборачивайте выполнение каждого инструмента в собственный `step.run()`, чтобы инструменты повторялись независимо от вызова LLM и друг от друга. Использование `step.run()` обеспечивает:

- **Независимые повторы** — сбой инструмента не приводит к повторному вызову LLM.
- **Детальная наблюдаемость** — вы видите в панели управления, какой именно инструмент дал сбой.
- **Параллельное выполнение** — инструменты, не зависящие друг от друга, могут запускаться одновременно.

Последовательное выполнение (самый частый случай):

```ts
for (const toolCall of toolCalls) {
  const result = await step.run(
    `tool-${toolCall.name}`,
    async () => executeTool(toolCall.name, toolCall.input)
  );
  toolResults.push({ type: "tool_result", tool_use_id: toolCall.id, content: result });
}
```

Для параллельного запуска независимых инструментов используйте `Promise.all`. Добавьте индекс, чтобы сделать ID каждого шага уникальным и облегчить отладку параллельных вызовов:

```ts
const results = await Promise.all(
  toolCalls.map((toolCall, idx) =>
    step.run(`tool-${toolCall.name}-${idx}`, async () => ({
      toolUseId: toolCall.id,
      result: executeTool(toolCall.name, toolCall.input),
    }))
  )
);
```

## Управление циклом

### Ограничение количества итераций

Всегда устанавливайте лимит итераций. Выбирайте лимит в зависимости от сложности задачи: простым Q&A может хватить 2–3, а кодинг-агенту, который читает, редактирует и тестирует код, может понадобиться 15–20.

### Отслеживание использования токенов

Чтобы не превысить бюджет, накапливайте использование токенов по итерациям:

```ts
let totalInputTokens = 0;

while (!done && iterations < MAX_ITERATIONS) {
  const llmResult = await step.run(`think`, async () => {
    return await anthropic.messages.create({ /* ... */ });
  });

  totalInputTokens += llmResult.usage.input_tokens;
  if (totalInputTokens > 500_000) {
    return { response: "Превышен бюджет токенов", iterations };
  }

  // ... остальная часть цикла
}
```

### Обнаружение зацикливания

Если агент многократно вызывает один и тот же инструмент, прервите цикл:

```ts
if (iterations > 3 && lastThreeTools.every(t => t === prevTool)) {
  done = true;
}
```

## Сокращение контекста при росте цикла

По мере работы агента массив сообщений увеличивается с каждой итерацией. Чтобы не превысить лимит контекстного окна, сохраняйте исходное сообщение пользователя и последние сообщения, удаляя промежуточные:

```ts
function pruneMessages(messages, maxMessages) {
  if (messages.length <= maxMessages) return messages;
  const first = messages[0];
  const recent = messages.slice(-maxMessages + 1);
  return [first, ...recent];
}
```
