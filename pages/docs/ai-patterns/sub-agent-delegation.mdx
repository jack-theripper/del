# Делегирование задач суб-агентам

Делегирование даёт суб-агентам собственное контекстное окно, инструменты и бюджет токенов. Суб-агент можно смоделировать как отдельную функцию Inngest, которая запускает собственный [цикл агента](/docs/ai-patterns/agent-tool-loops) — родитель либо ожидает результата, либо запускает и забывает.

## Определение функции суб-агента

Суб-агент — это обычная функция Inngest. Она получает задачу, запускает цикл агента и возвращает результат:

```typescript
import { inngest } from "./client";
import { createAgentLoop } from "./agent-loop";

export const subAgent = inngest.createFunction(
  { id: "sub-agent", triggers: [{ event: "agent/sub-agent.spawn" }] },
  async ({ event, step, logger }) => {
    const { task, sessionId } = event.data;

    const systemPrompt = `You are a focused sub-agent. Complete the following task and return a clear, concise result.\n\nTask: ${task}`;

    const result = await runAgentLoop({
      step,
      systemPrompt,
      sessionId,
      tools: SUB_AGENT_TOOLS, // No delegation tools — see "Prevent recursion"
      maxIterations: 20,
    });

    return {
      response: result.response,
      iterations: result.iterations,
    };
  }
);
```

Суб-агент использует **ограниченный набор инструментов** без инструментов делегирования, чтобы предотвратить бесконечную рекурсию ([подробнее ниже](#prevent-recursion)).

## Определение инструмента делегирования

Предоставьте LLM родительского агента инструмент, который сделает делегирование естественным выбором:

```typescript
const delegateTaskTool = {
  type: "function",
  function: {
    name: "delegate_task",
    description:
      "Delegate a task to a sub-agent that will work on it independently and return a result. " +
      "Use this for tasks that require deep research, many tool calls, or focused work " +
      "that would clutter the current conversation.",
    parameters: {
      type: "object",
      properties: {
        task: {
          type: "string",
          description:
            "A clear, self-contained description of the task. Include all necessary context — " +
            "the sub-agent does not have access to this conversation's history.",
        },
      },
      required: ["task"],
    },
  },
};
```

## Синхронное делегирование с `step.invoke()`

Чтобы делегировать задачу и ожидать результата, используйте [`step.invoke()`](/docs/guides/invoking-functions-directly) — родитель блокируется до тех пор, пока суб-агент не вернёт результат:

```typescript
import { subAgent } from "./sub-agent";

export const parentAgent = inngest.createFunction(
  { id: "parent-agent", triggers: [{ event: "agent/task.received" }] },
  async ({ event, step }) => {
    let messages = [{ role: "system", content: SYSTEM_PROMPT }];
    let done = false;
    let i = 0;

    while (!done && i < 30) {
      const response = await step.run(`think`, async () => {
        return await callLLM(messages, TOOLS);
      });

      for (const toolCall of response.toolCalls) {
        let toolResult: string;

        if (toolCall.name === "delegate_task") {
          // Synchronous delegation — parent waits for the result
          const subResult = await step.invoke(`sub-agent`, {
            function: subAgent,
            data: {
              task: toolCall.arguments.task,
              sessionId: `sub-${event.data.sessionId}-${Date.now()}`,
            },
          });

          toolResult = subResult?.response ?? "(no response from sub-agent)";
        } else {
          toolResult = await step.run(`tool-${toolCall.name}`, async () => {
            return await executeTool(toolCall.name, toolCall.arguments);
          });
        }

        messages.push(
          { role: "assistant", content: null, tool_calls: [toolCall] },
          { role: "tool", tool_call_id: toolCall.id, content: toolResult }
        );
      }

      if (response.toolCalls.length === 0) {
        done = true;
      }
      i++;
    }

    return { response: messages[messages.length - 1].content };
  }
);
```

Поскольку `step.invoke()` является надёжным шагом, родитель приостанавливает выполнение и возобновляет его ровно с того места, где остановился, когда суб-агент завершается. LLM родителя видит только сводку суб-агента, но не его полный внутренний диалог.

## Асинхронное делегирование с `step.sendEvent()`

Чтобы делегировать без ожидания, используйте [`step.sendEvent()`](/docs/guides/sending-events-from-functions) — родитель отправляет событие и продолжает работу:

```typescript
if (toolCall.name === "delegate_background_task") {
  await step.sendEvent("spawn-background-task", {
    name: "agent/sub-agent.spawn",
    data: {
      task: toolCall.arguments.task,
      sessionId: `sub-${event.data.sessionId}-${Date.now()}`,
      isAsync: true,
      replyTo: {
        type: "webhook",
        url: event.data.callbackUrl,
      },
    },
  });

  toolResult = "Task delegated. The sub-agent is working on it in the background.";
}
```

Чтобы доставить результаты по завершении суб-агента, обработайте асинхронный режим в суб-агенте:

```typescript
export const subAgent = inngest.createFunction(
  { id: "sub-agent", triggers: [{ event: "agent/sub-agent.spawn" }] },
  async ({ event, step, logger }) => {
    const { task, sessionId, isAsync, replyTo } = event.data;

    const result = await runAgentLoop({
      step,
      systemPrompt: `Complete this task:\n\n${task}`,
      sessionId,
      tools: SUB_AGENT_TOOLS,
      maxIterations: 30,
    });

    if (isAsync && replyTo) {
      await step.run("deliver-result", async () => {
        if (replyTo.type === "webhook") {
          await fetch(replyTo.url, {
            method: "POST",
            headers: { "Content-Type": "application/json" },
            body: JSON.stringify({ response: result.response }),
          });
        }
      });
    }

    return result;
  }
);
```

В качестве альтернативы суб-агент может отправить событие с результатом, а отдельная функция займётся доставкой — это отделяет суб-агент от логики маршрутизации:

```typescript
// Sub-agent emits result as an event
if (isAsync) {
  await step.sendEvent("result-ready", {
    name: "agent/sub-agent.completed",
    data: {
      sessionId,
      parentSessionId: event.data.parentSessionId,
      response: result.response,
    },
  });
}
```

```typescript
// Separate function handles result delivery
export const deliverSubAgentResult = inngest.createFunction(
  { id: "deliver-sub-agent-result" },
  { event: "agent/sub-agent.completed" },
  async ({ event, step }) => {
    const { response, parentSessionId } = event.data;

    await step.run("deliver", async () => {
      await notifyUser(parentSessionId, response);
    });
  }
);
```

### Выбор между синхронным и асинхронным режимом

| | Синхронно (`step.invoke()`) | Асинхронно (`step.sendEvent()`) |
|---|---|---|
| **Родитель блокируется?** | Да — ожидает результата | Нет — продолжает немедленно |
| **Результат передаётся** | В вывод инструмента родительского агента | Через отдельную доставку (webhook, событие, уведомление) |
| **Лучше для** | Задач, где родителю нужен ответ для продолжения (исследование, поиск, анализ) | Длительных задач, о результате которых пользователь может быть уведомлен позже (отчёты, пакетная обработка) |
| **Тайм-аут** | Ограничен временем выполнения функции | Суб-агент работает по собственному расписанию |
| **Поведение при повторе** | При сбое суб-агента шаг родителя повторяется | Суб-агент повторяется независимо |

## Предотвращение рекурсии

Если у суб-агента есть инструменты делегирования, он может порождать суб-агентов бесконечно. Предотвратите это, ограничив набор инструментов:

```typescript
// Инструменты, доступные родительскому агенту
const PARENT_TOOLS = [
  searchTool,
  readFileTool,
  writeFileTool,
  delegateTaskTool,        // Может делегировать
  delegateBackgroundTool,  // Может делегировать асинхронное управление
];

// Инструменты, доступные субагентам - без делегирования полномочий
const SUB_AGENT_TOOLS = [
  searchTool,
  readFileTool,
  writeFileTool,
  // No delegate tools — sub-agents cannot spawn further sub-agents
];
```

Совместите ограничение инструментов с жёстким лимитом итераций для двухуровневой защиты:

```typescript
export const subAgent = inngest.createFunction(
  { id: "sub-agent", retries: 1 },
  { event: "agent/sub-agent.spawn" },
  async ({ event, step }) => {
    const result = await runAgentLoop({
      step,
      systemPrompt: `Complete this task:\n\n${event.data.task}`,
      sessionId: event.data.sessionId,
      tools: SUB_AGENT_TOOLS, // Restricted — always
      maxIterations: 20,      // Hard cap on iterations
    });

    return result;
  }
);
```

## Обработка сбоев суб-агента

`step.invoke()` повторяет суб-агент согласно его конфигурации `retries`. Если все попытки исчерпаны, ошибка передаётся родителю. Чтобы позволить LLM родителя адаптироваться, перехватите ошибку:

```typescript
let toolResult: string;
try {
  const subResult = await step.invoke(`sub-agent`, {
    function: subAgent,
    data: { task: toolCall.arguments.task, sessionId: subSessionId },
  });
  toolResult = subResult?.response ?? "(no response)";
} catch (error) {
  toolResult = `Sub-agent failed: ${error.message}. You may need to handle this task directly.`;
}
```

## Планирование суб-агента

Чтобы запустить суб-агент в будущем, включите [временную метку](/docs/guides/delayed-functions#schedule-a-function-for-later) (`ts`) при отправке события:

```typescript
await step.sendEvent("schedule-daily-report", {
  name: "agent/sub-agent.spawn",
  data: {
    task: "Generate the daily analytics summary report.",
    sessionId: `scheduled-${Date.now()}`,
    isAsync: true,
    replyTo: { type: "webhook", url: REPORT_WEBHOOK_URL },
  },
  ts: tomorrow9am.getTime(),
});
```

Для повторяющейся работы используйте вместо этого [функцию, запускаемую по расписанию cron](/docs/guides/scheduled-functions).

## Написание самодостаточных описаний задач

В этой схеме вы можете выбрать, как контекст передаётся между родительским и суб-агентами. При описанном подходе суб-агент получает основной контекст из «задачи», переданной ему. В этом случае необходимо включать в задачу всё необходимое:

```typescript
// ❌ Плохой — зависит от контекста, которого нет у субагента
{ task: "Summarize what we discussed above" }

// ✅ Хороший — самодостаточный, со всем необходимым контекстом
{ task: "Summarize the key findings from the Q4 2025 revenue report. Focus on: 1) YoY growth rate, 2) top performing segments, 3) areas of concern." }
```

## Универсальные и специализированные суб-агенты

Универсальные суб-агенты — отличный способ начать работу, и они подходят для многих сценариев. Если вашей системе требуются более специализированные наборы инструментов или разные модели для суб-агентов, рассмотрите создание специализированных суб-агентов.

Чтобы создать систему со специализированными суб-агентами, следуйте описанным выше шаблонам, но создайте несколько инструментов, каждый из которых вызывает собственного агента или «загрузчик» суб-агента, способный условно обрабатывать несколько типов агентов. Рекомендуется использовать отдельные инструменты для отдельных суб-агентов, а не один инструмент с разными параметрами выбора — LLM обычно справляются с этим лучше.

Описание задачи и доступные инструменты достаточны для специализации поведения.
